### 001. LeetCode 26 删除有序数组中的重复项

- 日期：2026-07-06
- 来源：LeetCode
- 章节：线性表
- 链接：[26 删除有序数组中的重复项](https://leetcode.cn/problems/remove-duplicates-from-sorted-array/)
- 状态：已做
- 复杂度：`O(n) / O(1)`
- 核心思路：双指针 `slow / fast`。`fast` 从前往后扫描有序数组，遇到和 `nums[slow]` 不同的新值时，先移动 `slow`，再把新值覆盖到 `nums[slow]`，让 `[0, slow]` 始终是去重后的有效前缀。
- 代码：

```cpp
class Solution {
public:
    int removeDuplicates(vector<int>& nums) {
        int slow = 0;
        int size = nums.size();
        for (int fast = 1; fast < size; fast++) {
            if (nums[slow] != nums[fast]) {
                slow++;
                nums[slow] = nums[fast];
            }
        }
        return slow + 1;
    }
};
```

### 002. LeetCode 27 移除元素

- 日期：2026-07-06
- 来源：LeetCode
- 章节：线性表
- 链接：[27 移除元素](https://leetcode.cn/problems/remove-element/)
- 状态：已做
- 复杂度：`O(n) / O(1)`
- 核心思路：同向双指针 `low / fast`。`fast` 扫描原数组，只有当 `nums[fast] != val` 时才把当前元素覆盖到 `nums[low]`，然后移动 `low`。这样 `[0, low)` 始终保存所有不等于 `val` 的有效元素。
- 代码：

```cpp
class Solution {
public:
    int removeElement(vector<int>& nums, int val) {
        int low = 0;
        int size = nums.size();
        for (int fast = 0; fast < size; fast++) {
            if (nums[fast] != val) {
                nums[low] = nums[fast];
                low++;
            }
        }
        return low;
    }
};
```

### 003. LeetCode 189 轮转数组

- 日期：2026-07-06
- 来源：LeetCode
- 章节：线性表
- 链接：[189 轮转数组](https://leetcode.cn/problems/rotate-array/)
- 状态：已做
- 复杂度：`O(n) / O(n)`
- 核心思路：使用辅助数组保存轮转后的结果。原下标 `i` 的元素右移 `k` 位后落到 `(i + k) % size`，先把所有元素放入 `temp`，再整体拷贝回 `nums`。
- 代码：

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int size = nums.size();
        std::vector<int> temp(size, 0);
        for (int i = 0; i < size; i++) {
            temp[(i + k) % size] = nums[i];
        }
        for (int i = 0; i < size; i++) {
            nums[i] = temp[i];
        }
    }
};
```

- 优化思路：三次翻转原地完成右轮转。先把 `k` 对数组长度取模，再整体翻转数组，使末尾 `k` 个元素到前面；随后分别翻转前 `k` 段和后半段，恢复两段内部顺序。(可以使用线性代数来理解)
- 优化复杂度：`O(n) / O(1)`
- 优化代码：

```cpp
class Solution {
public:
    void reverse(vector<int>& nums, int left, int right) {
        while (left < right) {
            std::swap(nums[left++], nums[right--]);
        }
    }

    void rotate(vector<int>& nums, int k) {
        int size = nums.size();
        k %= size;
        reverse(nums, 0, size - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, size - 1);
    }
};
```

### 004. LeetCode 41 缺失的第一个正数

- 日期：2026-07-06
- 来源：LeetCode
- 章节：线性表
- 链接：[41 缺失的第一个正数](https://leetcode.cn/problems/first-missing-positive/)
- 状态：已做
- 复杂度：`O(n) / O(1)`
- 核心思路：原地哈希，也就是把值 `x` 尽量放回下标 `x - 1`。遍历数组时，只处理范围在 `[1, n]` 内且还没归位的数，通过交换把它放到正确位置；最后再扫一遍，第一个 `nums[i] != i + 1` 的位置就是缺失的最小正数。
- 代码：

```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int size = nums.size();

        // 原地哈希：数字 x 应该放在下标 x - 1 的位置。
        for (int i = 0; i < size; i++) {
            while (nums[i] > 0 &&
                   nums[i] <= size &&
                   nums[i] != nums[nums[i] - 1]) {
                std::swap(nums[i], nums[nums[i] - 1]);
            }
        }

        // 找到第一个没有归位的位置。
        for (int i = 0; i < size; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }

        return size + 1;
    }
};
```

### 005. LeetCode 206 反转链表

- 日期：2026-07-06
- 来源：LeetCode
- 章节：线性表
- 链接：[206 反转链表](https://leetcode.cn/problems/reverse-linked-list/)
- 状态：已做
- 复杂度：`O(n) / O(1)`
- 核心思路：迭代三指针 `pre / cur / next`。`cur` 指向当前节点，先用 `next` 保存后继节点，再把 `cur->next` 改指向 `pre` 完成局部反转，最后整体向后移动 `pre = cur`、`cur = next`。循环结束时 `pre` 就是新头结点。
- 代码：

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* pre = nullptr;
        ListNode* next = nullptr;
        ListNode* cur = head;

        while (cur) {
            next = cur->next;
            cur->next = pre;
            pre = cur;
            cur = next;
        }

        return pre;
    }
};
```

### 006. LeetCode 876 链表的中间结点

- 日期：2026-07-07
- 来源：LeetCode
- 章节：线性表
- 链接：[876 链表的中间结点](https://leetcode.cn/problems/middle-of-the-linked-list/)
- 状态：已做
- 复杂度：`O(n) / O(1)`
- 核心思路：第一种思路为计数法。遍历一次计算链表长度 `n`；第二次遍历移动到第 `n / 2`（下取整）个节点。
- 代码：

```cpp
class Solution {
public:
    ListNode* middleNode(ListNode* head) {
        // 第一种思路 计数
        int n = 0;
        ListNode* cur = head;
        while(cur){
            n++;
            cur = cur->next;
        }
        
        int idx = 0;
        cur = head;
        while(idx < n / 2){
            idx++;
            cur = cur->next;
        }
        return cur;
    }
};
```

- 优化思路：快慢指针（双指针法）。`slow` 和 `fast` 均从 `head` 出发。`fast` 每次走两步，`slow` 每次走一步。当 `fast` 走到链表末尾（即 `fast` 或 `fast->next` 为空）时，`slow` 恰好指向链表的中间结点（对于奇数长度指向唯一的中间结点，对于偶数长度指向第二个中间结点），只需单次扫描。
- 优化复杂度：`O(n) / O(1)`
- 优化代码：

```cpp
class Solution {
public:
    ListNode* middleNode(ListNode* head) {
        ListNode* slow = head;
        ListNode* fast = head;
        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
        }
        return slow;
    }
};
```

- 关联概念：[[wiki/concepts/数据结构/线性表|线性表]]、[[wiki/concepts/数据结构/算法复杂度分析|算法复杂度分析]]

### 007. LeetCode 19 删除链表的倒数第 N 个结点

- 日期：2026-07-07
- 来源：LeetCode
- 章节：线性表
- 链接：[19 删除链表的倒数第 N 个结点](https://leetcode.cn/problems/remove-nth-node-from-end-of-list/)
- 状态：已做
- 复杂度：`O(n) / O(1)`
- 核心思路：双指针 + 虚拟头结点（Dummy Node）。由于可能删除头结点，故创建虚拟头结点 `dump` 指向 `head`。`fast` 与 `slow` 均初始化指向 `dump`。让 `fast` 先走 `n` 步。然后 `fast` 和 `slow` 同时向后移动，直至 `fast->next` 为空（此时 `fast` 指向尾节点，`slow` 刚好指向待删除节点的前驱节点）。最后通过 `slow->next = slow->next->next` 绕过并释放目标节点。
- 代码：

```cpp
class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
        // 双指针，快指针多走 N 步
        ListNode dump = ListNode(0, head);
        ListNode* fast = &dump;
        ListNode* slow = &dump;
        while(n--){
            fast = fast->next;
        }
        while(fast->next){
            fast = fast->next;
            slow = slow->next;
        }
        ListNode* temp = slow->next;
        slow->next = slow->next->next;
        delete temp;
        return dump.next;
    }
};
```

### 008. LeetCode 160 相交链表

- 日期：2026-07-07
- 来源：LeetCode
- 章节：线性表
- 链接：[160 相交链表](https://leetcode.cn/problems/intersection-of-two-linked-lists/)
- 状态：已做
- 复杂度：`O(m + n) / O(1)`
- 核心思路：双指针 + 长度对齐法。首先分别遍历两个链表计算长度 `lenA` 和 `lenB`。求出长度差值 `diff`。让较长链表的指针先向后移动 `diff` 步，使两链表剩余长度对齐。接着两指针以相同速度同步后移，若相交则会在首个交点处相遇（`cur1 == cur2`），若不相交则会同时指向空。
- 代码：

```cpp
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        // 计算两个链表的长度，让长的先走差值，最后一定时刻到达
        int lenA = 0;
        int lenB = 0;
        ListNode* curA = headA;
        ListNode* curB = headB;
        while(curA){
            lenA++;
            curA = curA->next;
        }
        while(curB){
            lenB++;
            curB = curB->next;
        }
        int diff = lenA - lenB;
        ListNode* cur1 = diff >= 0 ? headA : headB;
        ListNode* cur2 = diff >= 0 ? headB : headA;
        for(int i = 0; i < abs(diff); i++){
            cur1 = cur1->next;
        }
        while(cur1 && cur2 && cur1 != cur2){
            cur1 = cur1->next;
            cur2 = cur2->next;
        }
        return cur1;
    }
};
```

- 优化思路：双指针环形相遇法。定义指针 `pA` 和 `pB`。两个指针同步向后扫描。若 `pA` 扫描至尾端，则重定向到 `headB`；若 `pB` 扫描至尾端，则重定向到 `headA`。由于两指针走过的总路程均为两链表总长（`a + c + b` 与 `b + c + a`），若相交则一定会在交点相遇，若不相交则会同时指向 `nullptr` 退出。该法无需显式计算长度，代码更为简洁。
- 优化复杂度：`O(m + n) / O(1)`
- 优化代码：

```cpp
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode *pA = headA;
        ListNode *pB = headB;
        while (pA != pB) {
            pA = pA ? pA->next : headB;
            pB = pB ? pB->next : headA;
        }
        return pA;
    }
};
```
