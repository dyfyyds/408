# 计算机系统概述

## Definition

计算机系统由**硬件**和**软件**两大部分组成，采用**存储程序**的工作原理。软件和硬件在逻辑上是等效的（软硬件逻辑等效性）。

---

## 计算机的基本组成

### 冯·诺依曼结构基本思想

#### 核心原理：存储程序（Stored Program）
* **基本含义**：将编写好的程序和原始数据事先送入主存储器中。一旦程序启动，计算机便能**自动地、连续地**取出指令并执行，无需人工干预。其核心驱动方式是**控制流驱动**。

#### 五大部件
1. **运算器（ALU）**：执行算术运算和逻辑运算。
   > [!IMPORTANT]
   > 在早期的冯·诺依曼结构中，系统以**运算器为中心**，输入/输出设备与存储器的数据交换都要经过运算器。现代计算机则以**存储器为中心**。
2. **控制器（CU）**：负责从存储器中取出指令、分析指令，并向各部件发出控制信号以协调工作。
3. **存储器**：存放程序和数据，分为**主存储器（内存）**和**辅助存储器（外存）**。CPU 只能直接访问主存储器。
4. **输入设备**：将外部信息（如键盘输入、鼠标移动）转换为机器能识别的二进制代码送入存储器。
5. **输出设备**：将计算机内部的处理结果转换为人们或其他设备易于接受的形式输出。

#### 体系核心特征
* **同等存储**：指令和数据以同等地位存储在存储器中，形式上没有任何区别（都是二进制代码），并且都可以按地址寻址。
  > [!TIP]
  > **CPU 区分指令和数据的依据**：**指令周期的不同阶段**。在取指阶段从内存中取出来的是指令（送入 IR），在执行阶段从内存中取出来的是数据（送入通用寄存器或 ALU）。
* **二进制表示**：指令和数据均用二进制代码表示。指令由**操作码（OP）**和**地址码（AD）**组成，操作码指明操作类型，地址码指明操作数在存储器中的地址。

## 计算机语言与程序翻译

### 三个级别语言
1. **机器语言**：二进制代码表示，是计算机硬件**唯一可以直接识别和执行**的语言。
2. **汇编语言**：使用助记符表示操作码，用符号表示地址。与机器语言指令**一一对应**，但硬件不能直接识别，需通过**汇编程序**翻译。
3. **高级语言**：面向算法和用户的语言（如 C, C++, Java），具有良好的可读性和可移植性。

### 翻译程序的分类
* **汇编程序（Assembler）**：将汇编语言源程序翻译成机器语言目标程序。
* **编译程序（Compiler）**：将高级语言源程序**一次性全部**翻译成汇编语言或机器语言目标程序，并**生成目标文件**。程序执行时只需启动目标程序，运行效率高。
* **解释程序（Interpreter）**：将高级语言源程序逐条翻译成机器指令并**立刻执行**，**不生成目标文件**（如 Python, JavaScript）。

```mermaid
graph LR
    A[源程序.c] -- ① 预处理 --> B[预处理源程序.i]
    B -- ② 编译 --> C[汇编源程序.s]
    C -- ③ 汇编 --> D[可重定位目标文件.o]
    D -- ④ 链接 --> E[可执行文件.exe]
```
> [!NOTE]
> **编译系统的四个步骤详解：**
> 1. **预处理（Preprocessing）**：由预处理器（cpp）执行，主要处理以 `#` 开头的预处理指令（如展开 `#include` 头文件、替换 `#define` 宏定义、过滤注释等），生成 `.i` 文本文件。
> 2. **编译（Compilation）**：由编译器（cc1）执行，将预处理后的源文件进行语法、语义分析，生成对应的汇编语言 `.s` 文本文件。
> 3. **汇编（Assembly）**：由汇编器（as）执行，将汇编源程序翻译成二进制机器指令，生成可重定位的目标二进制文件 `.o` 或 `.obj`。
> 4. **链接（Linking）**：由链接器（ld）执行，将多个目标二进制文件以及它们调用的库函数（如 `libc.a`）合并链接，解决外部符号引用，最终生成可运行的二进制可执行文件（在 Windows 下为 `.exe`，Linux 下为可执行的 ELF 文件）。

## 字长、带宽与存储容量

### 字长的区分（必考概念）
* **机器字长**：指 CPU 内部用于整数运算的**数据通路的宽度**，通常等于 **ALU 的位数**和**通用寄存器的宽度**。
* **指令字长**：一条指令中包含的二进制代码的位数。
* **存储字长**：一个存储单元能够存储的二进制代码的长度，通常等于 **MDR 的位数**。
  > [!WARNING]
  > 指令字长和存储字长可以不等于机器字长，但它们必须是**字节（8 bits）的整数倍**。

### 带宽
* **数据通路带宽**：数据总线一次性所能传送信息的位数。
* **总线带宽**：总线每秒传送的最大字节数。$总线带宽 = 总线宽度（位） \times 总线工作频率（Hz） / 8$（单位为 B/s）。

### 主存容量计算
* **公式**：
  $$主存最大容量 = 存储单元个数 \times 存储字长$$
  * 存储单元个数由地址总线宽度（即 **MAR 的位数 $n$**）决定，最多为 $2^n$ 个存储单元。
  * 存储字长由 **MDR 的位数**决定。
  * 例如：MAR 为 32 位，MDR 为 8 位，则最大寻址容量为 $2^{32} \times 8 \text{ bits} = 4\text{G} \times 1\text{ Byte} = 4\text{GB}$。

## 计算机性能指标

### 时钟与速度指标
* **主频（时钟频率）**：CPU 内部主时钟脉冲的频率，单位为 Hz（如 3.6 GHz）。它代表 CPU 每秒产生的时钟周期数。
* **IPC（Instructions Per Cycle）**：每个时钟周期执行的指令条数，是 CPI 的倒数。
  $$IPC = \frac{1}{CPI}$$
* **CPU时钟周期**：主频的倒数（$1/主频$），是 CPU 处理动作的最小时间单位。
* **CPI（Cycles Per Instruction）**：执行一条指令所需的时钟周期数。对于包含不同指令的程序，CPI 是指平均 CPI。
* **IPS（Instructions Per Second）**：CPU 每秒执行的指令条数。
  $$IPS = \frac{指令条数}{CPU执行时间} = \frac{主频}{CPI} = 主频 \times IPC$$

### 执行时间
$$CPU执行时间 = \frac{指令条数 \times CPI}{主频} = 指令条数 \times CPI \times CPU时钟周期 = \frac{指令条数}{IPS}$$
> [!TIP]
> 如果题目给出的是以百万为单位的 **MIPS** 指标，则对应的 CPU 执行时间公式为：
> $$CPU执行时间 = \frac{指令条数}{MIPS \times 10^6}$$

> [!IMPORTANT]
> 主频高的 CPU 不一定比主频低的 CPU 快，必须同时综合考虑 **指令条数** 和 **CPI**。

### 计算能力指标（宏观速度指标）
* **KIPS、MIPS、GIPS（以 $10^3$ 进制递增）**：
  * **KIPS（Kilo IPS）**：每秒执行千条指令。$KIPS = IPS / 10^3 = \frac{主频}{CPI \times 10^3}$
  * **MIPS（Million IPS）**：每秒执行百万条指令。$MIPS = IPS / 10^6 = \frac{主频}{CPI \times 10^6}$
  * **GIPS（Giga IPS）**：每秒执行十亿条指令。$GIPS = IPS / 10^9 = \frac{主频}{CPI \times 10^9}$
* **FLOPS（Floating-point Operations Per Second）**：每秒执行浮点运算的次数（2011、2021 考点）。
  * **MFLOPS（Million FLOPS）**：百万（$10^6$）次浮点运算/秒。
  * **GFLOPS（Giga FLOPS）**：十亿（$10^9$）次浮点运算/秒。
  * **TFLOPS（Tera FLOPS）**：万亿（$10^{12}$）次浮点运算/秒。
  * **PFLOPS（Peta FLOPS）**：千万亿（$10^{15}$）次浮点运算/秒。
  * **EFLOPS（Exa FLOPS）**：百亿亿（$10^{18}$）次浮点运算/秒。
  * **ZFLOPS（Zetta FLOPS）**：十万亿亿（$10^{21}$）次浮点运算/秒。
  > *注：各速度单位之间以 $10^3$（千进制）递增。*

## 补充考研术语

* **系列机（兼容机）**：具有相同或相似的指令集架构（ISA），并运行相同操作系统的计算机系列。系列机通常满足**指令系统向后兼容**（即新开发的机器可以运行旧机器的程序，但反之不一定成立）。
* **兼容（Compatibility）**：指软件或硬件的通用性，即某些程序或硬件部件可以不经修改地在不同型号的计算机上运行。
* **软件可移植性（Portability）**：指将软件从一种计算机环境转移到另一种计算机环境运行而不需要做重大修改的方便程度。
* **固件（Firmware）**：将程序固定在 ROM 中的特殊软件（如 BIOS）。它在物理上是硬件，在逻辑上是软件，是“软硬件逻辑等效”的典型体现。

## 常见考点与陷阱

> [!WARNING]
> * **MIPS 的局限性**：MIPS 指标只能粗略比较**同一类指令系统**的计算机。不能脱离指令集，直接用 MIPS 比较不同指令集架构的机器性能（例如，RISC 和 CISC 之间）。
> * **系统软件的划分**：数据库管理系统（DBMS）是系统软件，但数据库系统（DBS）包含数据库、DBMS、硬件和管理员，不属于单纯的系统软件。
> * **时钟周期与指令周期**：一个指令周期通常包含多个机器周期，一个机器周期包含多个 CPU 时钟周期。
> * **数值前缀的二/十进制混淆陷阱（做题极高频死穴）**：
>   * **存储容量、文件大小**：前缀 K、M、G、T 基于 **2 的幂次**（即 $2^{10}$ 进制）。
>     * $1\text{ KB} = 2^{10}\text{ B} = 1024\text{ B}$
>     * $1\text{ MB} = 2^{20}\text{ B}$
>     * $1\text{ GB} = 2^{30}\text{ B}$
>     * $1\text{ TB} = 2^{40}\text{ B}$
>   * **传输速率、频率、速度指标（如主频、带宽、IPS、FLOPS）**：前缀 k、M、G、T 基于 **10 的幂次**（即 $10^3$ 进制）。
>     * $1\text{ kb/s} = 10^3\text{ b/s} = 1000\text{ b/s}$
>     * $1\text{ Mb/s} = 10^6\text{ b/s}$
>     * $1\text{ Gb/s} = 10^9\text{ b/s}$
>     * 主频 $1\text{ GHz} = 10^9\text{ Hz}$，总线带宽 $100\text{ MB/s} = 100 \times 10^6\text{ B/s}$。
>   * **书写习惯**：习惯上前者容量用大写 **K**，后者速率用小写 **k**（但 M、G、T 均大写）。做题时必须结合上下文精确判断到底是容量还是速率/频率，以防计算公式代入错误。
