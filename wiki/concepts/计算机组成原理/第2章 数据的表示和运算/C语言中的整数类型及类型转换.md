# C语言中的整数类型及类型转换

统考 408 大纲要求考生具备分析高级程序设计语言（如 C 语言）中相关机器级问题的能力。其中，**有符号与无符号数的解释方式**、**不同字宽变量之间的类型转换（位扩展与位截断）** 是高频常考重难点，必须深入掌握其底层的物理本质。

---

## 1. C语言中的整型数据类型

在 C 语言中，整型数据类型的具体长度依赖于编译器和目标平台，但在 408 考研及现代通用系统设计中，标准字宽约定如下：

| 数据类型 | 常见字宽（位） | 十六进制表示字宽 | 408 默认取值范围 |
| :---: | :---: | :---: | :--- |
| **字符型 `char`** | 8 位 | 2 位 | 有符号：$-128 \sim +127$<br>无符号：$0 \sim 255$ |
| **短整型 `short`** | 16 位 | 4 位 | 有符号：$-32768 \sim +32767$<br>无符号：$0 \sim 65535$ |
| **整型 `int`** | 32 位 | 8 位 | 有符号：$-2^{31} \sim 2^{31}-1$<br>无符号：$0 \sim 2^{32}-1$ |
| **长整型 `long`** | 32 / 64 位 | 8 / 16 位 | *注：32位系统下为32位，64位系统下通常为64位* |
| **双长整型 `long long`**| 64 位 | 16 位 | 有符号：$-2^{63} \sim 2^{63}-1$<br>无符号：$0 \sim 2^{64}-1$ |

### 核心设计机制
1. **符号修饰符**：类型前若添加 `unsigned` 关键字，则定义为**无符号类型**；未显式指定或使用 `signed` 则默认定为**有符号类型**。
2. **底层存储编码**：
   - **所有有符号整型**：在计算机内部均以**补码**形式存储（第一位为符号位，其余为数值位）。
   - **所有无符号整型**：全部二进制位均用于表示非负数值（无符号位），直接以**纯二进制无符号数**形式存储。

## 2. 整型数据的类型转换（核心物理本质）

C 语言中的类型转换在机器级表现为对寄存器内二进制位模式的重新解释或位宽调整。

### (1) 同字长转换（有符号 $\leftrightarrow$ 无符号）
* **物理本质**：**二进制位模式完全保持不变，仅改变对最高有效位（MSB）的解释权重**。
  - 对于 $n$ 位整型，最高有效位 $b_{n-1}$ 在有符号补码下代表的权值为 **$-2^{n-1}$**，而在无符号数下代表的权值为 **$+2^{n-1}$**。
* **转换公式**：
  设机器码为 $B = b_{n-1}b_{n-2}\dots b_0$：
  - **有符号转无符号**：
    $$U = S \pmod{2^n}$$
    即若 $S \ge 0$，则 $U = S$；若 $S < 0$，则 $U = 2^n + S$。
  - **无符号转有符号**：
    $$S = \begin{cases} U, & U < 2^{n-1} \\ U - 2^n, & U \ge 2^{n-1} \end{cases}$$
* **典型示例**：
  8 位二进制机器数 `1101 0011B`（十六进制为 `D3H`）：
  - 若作为有符号数 `char` 解释：最高位为 `1`，通过补码求真值（数值位取反加 1）得到真值为 **$-45$**。
  - 若作为无符号数 `unsigned char` 解释：真值为 $128 + 64 + 16 + 2 + 1 = 211$（即 $256 + (-45) = 211$）。

---

### (2) 短字长 $\to$ 长字长转换（位扩展）
当短整型变量转换为长整型变量时（如 `short` $\to$ `int`），为了填满长字宽寄存器，系统必须对高位进行填充。位扩展是**安全**的，转换后其代数真值能够完美保持不变。

#### ① 无符号数的零扩展（Zero Extension）
* **规则**：直接在高位填充 **`0`**。
* **示例**：
  `unsigned short x = 65531;` (十六进制机器数为 `FFFBH`)
  `unsigned int y = x;` (扩展为 32 位)
  - 机器高 16 位补零，转换后 `y` 在寄存器中的值为 `0000FFFBH`，真值仍为 $65531$。

#### ② 有符号数的符号扩展（Sign Extension）
* **规则**：在高位填充**原短字长机器数的符号位**（即原最高位）。
* **物理意义**：根据补码的数学性质，在符号位左边填充任意多个符号位字符，其代数真值大小保持恒等。
* **示例**：
  `short x = -5;` (16 位有符号补码为 `FFFBH`，最高位为 `1`)
  `int y = x;` (扩展为 32 位)
  - 机器检测到原最高位为 `1`，于是在高 16 位全部填充 `1`，转换后 `y` 为 `FFFF FFFBH`，真值仍然为 $-5$。

> [!WARNING] **408 高频易错点：有符号短整型 强转为 长无符号整型**
>
> 在 C 语言中，当进行如 `short` (有符号短整型) $\to$ `unsigned int` (无符号长整型) 这种**既涉及字宽变化又涉及符号变化**的强制转换时，编译器会严格遵循以下两步转换顺序：
> 1. **第一步：先进行位宽扩展**。因为源数据 `short` 是有符号数，因此按照**符号扩展**规则进行扩展（变成 32 位有符号 `int`）。
> 2. **第二步：再进行符号重新解释**。将得到的 32 位有符号 `int` 重新解释为 `unsigned int`。
>
> **典型真题推导（以 `short x = -1` 强转为 `unsigned int` 为例）**：
> 1. `short x = -1` 的 16 位补码机器数为 `FFFFH`。
> 2. **符号扩展**到 32 位：由于最高位为 `1`，高位补 `1`，得到 `FFFF FFFFH`（有符号 `int` 的 $-1$）。
> 3. **重新解释为无符号数**：直接将 `FFFF FFFFH` 读作无符号数，对应真值为 **$2^{32} - 1 = 4294967295$**。

---

### (3) 长字长 $\to$ 短字长转换（位截断）
当长整型变量转换为短整型变量时（如 `int` $\to$ `short`，或 `short` $\to$ `char`），会触发截断操作。

* **物理本质**：**系统直接强行丢弃高位，仅保留低位部分，原封不动放入目标短寄存器中**。
* **危害**：这种操作是**极不安全**的。因为丢弃了高位，剩余的低位如果重新解释，极易导致**数值发生严重的溢出**，或者**符号位发生改变**（正数变负数，或负数变正数）。
* **典型示例**：
  `int x = 32768;` (32 位有符号补码为 `0000 8000H`)
  `short y = (short)x;` (截断为 16 位)
  - 截断过程：直接丢弃高 16 位 `0000H`，保留低 16 位 `8000H`（即二进制 `1000 0000 0000 0000B`）。
  - 重新解释：由于 `y` 是有符号的 `short` 类型，最高位 `1` 被解释为负号，且该值恰好对应 `short` 所能表示的最小负数，其真值为 **$-32768$**。

## 3. 408 历年高频真题深度剖析

### (1) 2017 年 408 真题
**【题目】** 假设在 32 位计算机上运行以下 C 语言代码：
```c
int x = -134217728;
unsigned int y = (unsigned int)x;
```
若 `x` 的十六进制表示为 `F800 0000H`，则 `y` 的真值及十六进制表示为？

**【解析】**
1. **获取位模式**：有符号数 `x` 在寄存器中的二进制位模式为 `F800 0000H`。
2. **同字长强转**：`unsigned int y = (unsigned int)x` 属于同为 32 位的数据转换，位模式保持不变。
3. **解释为无符号数**：`y` 在寄存器中的二进制位模式仍为 `F800 0000H`。直接按无符号数权重展开计算真值：
   $$\text{F800 0000H} = 15 \times 16^7 + 8 \times 16^6 = 15 \times 2^{28} + 8 \times 2^{24} = 4026531840$$
   或利用公式：
   $$y = 2^{32} + x = 4294967296 - 134217728 = 4160749568$$
4. **结论**：`y` 的十六进制表示为 `F800 0000H`，真值为 `4160749568`（十进制）。

---

### (2) 2019 年 408 真题
**【题目】** 假设某计算机字长为 32 位，其中有符号整型变量 `x` 的值为 `-8`。请问将其强制类型转换为短整型（`short`）和双精度浮点型（`double`）后的机器数分别是什么？

**【解析】**
1. **写出原机器数**：
   `int x = -8;` 为 32 位有符号整型。
   - 真值绝对值为 $8 = 0000 0008H$。
   - 利用十六进制快捷口算，其 32 位补码机器数为 `FFFF FFF8H`。
2. **转换为短整型 `short`（位截断）**：
   - 截断规则：将 32 位的 `FFFF FFF8H` 直接截断高 16 位，保留低 16 位。
   - 得到 16 位 `short` 类型的机器数为 `FFF8H`。
3. **转换为双精度浮点型 `double`**：
   - 双精度浮点型遵循 IEEE 754 标准。
   - 真值 $-8 = -1.0 \times 2^3$。
   - 符号位 $S = 1$。
   - 阶码 $E = 3$，双精度偏置值为 $1023$，阶码移码表示为 $E_{移} = 3 + 1023 = 1026 = 100\ 0000\ 0010_2$（二进制）。
   - 尾数部分（隐藏前导 1）：尾数位全为 `0`。
   - 拼接得双精度 64 位表示为：
     $$\text{1 | 100 0000 0010 | 000...00} \implies \text{C020 0000 0000 0000H}$$
4. **结论**：截断为 `short` 的机器数为 `FFF8H`，转换为 `double` 的机器数为 `C020 0000 0000 0000H`。

---

### (3) 2024 年 408 真题
**【题目】** 设有有符号短整型变量 `short x = -2;`，将其强制转换为无符号整型（`unsigned int`）变量 `y`，则 `y` 的真值及十六进制机器数表示为？

**【解析】**
1. **写出原机器数**：
   `short x = -2` 为 16 位有符号整型。
   - 十六进制绝对值为 `0002H`。
   - 16 位补码机器数直接口算为 `FFFEH`（因为 `FFFEH + 0002H = 10000H`）。
2. **第一步：位宽扩展（符号扩展）**：
   - 因为 `short` 是有符号数，强制类型转换到 32 位时采用**符号扩展**。
   - 原最高位为 `1`（`F = 1111B`），因此高 16 位全部补充 `1` $\implies$ 得到 32 位整型机器数 `FFFF FFFEH`。
3. **第二步：类型重新解释**：
   - 将 32 位机器数 `FFFF FFFEH` 解释为无符号数 `unsigned int`。
   - 真值计算：利用模运算法，
     $$y = 2^{32} + x = 4294967296 - 2 = 4294967294$$
4. **结论**：`y` 的十六进制机器数表示为 `FFFF FFFEH`，真值为 `4294967294`。

## 4. 常见易错总结与解题Tips

> [!TIP] **强转大杀器：判断扩展类型的关键看“源类型”**
>
> 在进行字宽扩展时，很多同学会纠结到底使用“符号扩展”还是“零扩展”。记住一个绝对正确的黄金准则：
> **位扩展的类型（符号扩展或零扩展）完全由“转换前的源数据类型（Signed/Unsigned）”决定，与目标数据类型无关！**
> - 源数据类型是 **有符号** (`signed`) $\implies$ 一律进行**符号扩展**（高位补符号位）。
> - 源数据类型是 **无符号** (`unsigned`) $\implies$ 一律进行**零扩展**（高位补零）。
>
> *秒杀演练*：
> `unsigned short a = 65535;` (16位无符号 `FFFFH`)
> `int b = (int)a;`
> 转换分析：源类型 `unsigned short` 是**无符号数**，因此即使目标类型 `int` 是有符号数，也必须按**零扩展**进行扩展。
> 扩展后 `b = 0000 FFFFH` $\implies$ 真值仍然是正数 $65535$。如果是符号扩展就会错变成 `-1`。
