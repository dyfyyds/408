# 数据的宽度和存储

## 1. 数据宽度与存储单位

### 1.1 位、字节和数据宽度

- **位（bit）**：计算机中最小的数据单位，只能表示 0 或 1。
- **字节（Byte）**：通常由 8 个 bit 组成，是按字节编址系统中的基本存储单位。
- **数据宽度**：一个数据在机器中占用的 bit 数，例如 8 位、16 位、32 位和 64 位。
- **占用字节数**：

$$
\text{占用字节数}=\frac{\text{数据宽度（bit）}}{8}.
$$

常见对应关系：

| 数据宽度 | 占用字节数 | 常见称呼 |
| ---- | ---- | ---- |
| 8 bit | 1 Byte | 字节 |
| 16 bit | 2 Byte | 半字 |
| 32 bit | 4 Byte | 字 |
| 64 bit | 8 Byte | 双字 |

### 1.2 地址与多字节数据

在按字节编址的机器中，每个字节拥有一个独立地址。若一个 32 位数据从地址 1000H 开始存放，则它占用：

$$
1000\text{H},\ 1001\text{H},\ 1002\text{H},\ 1003\text{H}.
$$

数据的起始地址称为首地址，最后一个字节的地址为：

$$
\text{末地址}=\text{首地址}+\text{占用字节数}-1.
$$

> [!tip] 地址计算
> 题目给出首地址和数据宽度时，先把 bit 换算成 Byte，再用“首地址 + 字节数 - 1”求末地址。大小端只决定这些地址上分别放哪个字节。

## 2. 大小端存储

### 2.1 基本概念

当一个数据占用多个字节时，需要规定高有效字节和低有效字节在内存地址中的排列顺序，这种排列规则称为**字节序（Endianness）**。

- **大端存储（Big-Endian）**：高有效字节存放在低地址，低有效字节存放在高地址。
- **小端存储（Little-Endian）**：低有效字节存放在低地址，高有效字节存放在高地址。

这里的“高、低”指字节在整个数据中的有效位权，不是内存地址高低。

### 2.2 32 位数据示例

设 32 位数据为：

$$
12345678\text{H}=12\ 34\ 56\ 78\text{H},
$$

从低地址 1000H 开始存放。

| 地址 | 大端存储 | 小端存储 |
| ---- | ---- | ---- |
| 1000H | 12H（高有效字节） | 78H（低有效字节） |
| 1001H | 34H | 56H |
| 1002H | 56H | 34H |
| 1003H | 78H（低有效字节） | 12H（高有效字节） |

因此，从低地址到高地址观察：

$$
\begin{aligned}
\text{大端：}&\quad 12\ 34\ 56\ 78,\\
\text{小端：}&\quad 78\ 56\ 34\ 12.
\end{aligned}
$$

### 2.3 大小端的适用范围

#### 2.3.1 单个字节内部的位顺序不变

大小端针对的是**多个字节之间的排列顺序**，不是一个字节内部 bit 的颠倒。

例如，一个字节的二进制内容为：

$$
11000011.
$$

无论采用大端还是小端，这个字节在自己的 1 个 Byte 存储空间中仍然是 11000011。大小端规则不会把它改写成位逆序，也不会改变该字节内部各 bit 的相对位置。

因此：

- 只有数据占用两个及以上字节时，大小端才可能产生可观察差异；
- 8 位数据只有 1 个字节，不存在字节之间的端序排列问题；
- 端序判断应先按 Byte 拆分数据，而不是在单个字节内翻转 bit。

#### 2.3.2 字符串和字节数组不直接套用整数大小端

在常见单字节字符编码下，字符串本质上可以看作连续的字节数组。例如字符串 ABC 的 ASCII 编码依次为：

$$
\text{A}=41\text{H},\qquad
\text{B}=42\text{H},\qquad
\text{C}=43\text{H}.
$$

从低地址到高地址存放时为：

$$
41\text{H},\ 42\text{H},\ 43\text{H}.
$$

每个字符只占 1 个 Byte，通常按字符串顺序依次存放，不会因为整数采用大端或小端而把 ABC 自动排列为 CBA。

因此：

- 单字节字符数组通常按元素顺序处理；
- 不能把字符串当成一个整数，再机械套用整数大小端规则；
- 如果某种字符编码或文件格式明确规定了多字节字符的字节序，则应以该编码或格式的约定为准；
- 正确端序读取多字节整数时，真值不变；错误端序解析才会得到错误数值。

### 2.4 识别大小端的做题方法

给出连续内存内容和一个多字节数据时：

1. 确认数据宽度，按每 2 个十六进制数字拆成一个字节；
2. 找到最低地址；
3. 以最低地址为排列起点：大端放高有效字节，小端放低有效字节；
4. 地址每次增加 1 个 Byte；
5. 按题目指定的端序还原并拼接，检查能否得到原数据。

> [!warning] 高频混淆
> 大端不是“高字节放高地址”，而是“高有效字节放低地址”；小端正好相反。地址方向与数据位权方向不要混为一谈。

## 3. 大小端典型例题

### 3.1 2016 统考真题：double 的小端存储

**题目条件**：32 位计算机按字节编址，采用小端方式。一个 double 型变量的机器数为：

$$
1122\ 3344\ 5566\ 7788\text{H}.
$$

它从地址 0000 8040H 开始连续存放，求地址 0000 8046H 中的内容。

#### 解题过程

1. double 占 64 位，即 8 Byte。
2. 将机器数从高有效字节到低有效字节拆开：

$$
11\ 22\ 33\ 44\ 55\ 66\ 77\ 88.
$$

3. 小端存储要求低有效字节放在低地址，因此从首地址开始排列：

| 地址 | 存储字节 |
| ---- | ---- |
| 8040H | 88H |
| 8041H | 77H |
| 8042H | 66H |
| 8043H | 55H |
| 8044H | 44H |
| 8045H | 33H |
| 8046H | 22H |
| 8047H | 11H |

4. 地址 8046H 是首地址向后第 6 个字节，存放 22H。

**答案：A，22H。**

> [!tip] 地址定位公式
> 先算偏移量：目标地址 - 首地址 = 6。小端排列的第 6 个字节，是从低有效端数第 6 个字节，不要直接从机器数左侧数。

### 3.2 2018 统考真题：小端立即数存储

**题目条件**：32 位计算机按字节编址，采用小端方式。语句 int i=0; 对应机器代码：

$$
\text{C7 45 FC 00 00 00 00}.
$$

求语句 int i=-64; 对应的机器代码。

#### 解题过程

1. 指令前 3 个字节 C7 45 FC 是操作码和地址/寻址字段，本题只需修改后面的 32 位立即数。
2. $-64$ 的 32 位补码为：

$$
64=00000040\text{H},
$$

$$
[-64]_{\text{补}}=2^{32}-64=\text{FFFFFFC0H}.
$$

3. 立即数 FFFFFFC0H 按高有效字节到低有效字节拆分为：

$$
\text{FF FF FF C0}.
$$

4. 小端存储时，低有效字节 C0H 放在较低地址，内存顺序变为：

$$
\text{C0 FF FF FF}.
$$

5. 拼接指令前缀和立即数字节：

$$
\text{C7 45 FC C0 FF FF FF}.
$$

**答案：A。**

> [!warning] 易错点
> 小端只改变多字节立即数在内存中的排列，不改变指令前缀，也不能把负数的补码真值顺序直接倒写。应先求 32 位补码，再按字节反向存放。

## 4. 边界对齐

### 4.1 自然对齐与非对齐

在 C 语言的常见实现中，编译器会按照数据类型的对齐要求安排结构体布局，主要遵循两条规则：

1. 每个成员的起始地址必须是该成员对齐值的整数倍；
2. 整个结构体的大小必须是其最大成员对齐值的整数倍，不足时在结构体末尾填充。

这里的“对齐值”表示对象起始地址需要满足的边界要求，不等同于对象实际占用的字节数。它由数据类型、处理器和编译器共同决定。在 408 题目常用的简化模型中，基本类型的对齐值通常与其占用字节数相同：

| 数据类型 | 数据宽度 | 占用字节数 | 常见对齐值 |
| --- | ---: | ---: | ---: |
| char | 8 bit | 1 | 1 |
| short | 16 bit | 2 | 2 |
| int | 32 bit | 4 | 4 |
| long long / double | 64 bit | 8 | 8 |

例如，在对齐值为 4 的模型下，一个 4 字节对象从 1000H 开始存放属于自然对齐，从 1001H 开始存放则属于非对齐存储。

### 4.2 对齐地址判断

若某成员的对齐值为 $a$，其起始地址为 $addr$，则只有满足下式时，该成员才是自然对齐的：

$$
\text{addr}\bmod a=0.
$$

例如：

- 对齐值为 2 的成员：1002H 对齐，1001H 不对齐；
- 对齐值为 4 的成员：1000H、1004H 对齐，1002H 不对齐；
- 对齐值为 8 的成员：1000H、1008H 对齐，1004H 不对齐。

注意：在常见模型中“占用字节数”和“对齐值”数值相同，但概念不同。占用字节数决定对象占用多少存储空间，对齐值决定对象允许从哪些地址开始。

上述判断针对 408 题目中常用的自然对齐模型。实际程序的对齐要求还可能受到处理器、编译器选项和结构体打包指令的影响。

### 4.3 对齐、填充与跨边界访问

非对齐数据可能跨越处理器或存储器的访问边界，处理器可能需要拆成多次访问，再拼接出完整数据，从而降低访问效率。

结构体中常见的填充过程如下：

    struct Example {
        char c;
        int x;
    };

在常见的 4 字节结构体对齐模型下，每个成员的起始偏移必须满足自身的对齐值；所有成员排布完成后，结构体总大小还必须满足最大成员的对齐值：

- c 占用偏移 0；
- 1 到 3 可能作为填充字节；
- x 从偏移 4 开始；
- 结构体总大小通常向最大成员对齐值的整数倍取整，常见结果为 8 Byte。

### 4.4 数组与对齐

数组元素按相同类型连续排列。若元素类型本身需要对齐，数组首元素对齐后，后续元素通常按照元素占用空间或结构体大小递增：

$$
\text{第 }i\text{ 个元素地址}
=\text{首地址}+i\times\text{元素步长}.
$$

若元素是含填充的结构体，元素步长应取结构体实际大小，而不是成员大小的简单相加。

### 4.5 结构体成员顺序与填充演示

下面的示例采用 32 位 x86、GCC 的常见对齐规则，图片展示了同样成员的两种排列方式：

![[resources/计算机组成原理/数据宽度与存储/结构体对齐示例.png|900]]

设：

~~~c
struct A {
    int a;
    char b;
    short c;
};

struct B {
    char b;
    int a;
    short c;
};
~~~

在题图采用的常见规则下，int、short、char 分别占 4 Byte、2 Byte、1 Byte，对齐值也分别为 4、2、1。计算结构体布局时，应先按声明顺序确定每个成员的起始偏移和内部填充，再按最大成员对齐值检查结构体末尾是否需要填充。

对于 struct A：

1. int a 从偏移 0 开始，占用 0～3；
2. char b 放在偏移 4，占用 1 Byte；
3. short c 的对齐值为 2；偏移 5 不是 2 的倍数，因此在偏移 5 处补 1 Byte；
4. short c 从偏移 6 开始，占用 6～7；
5. 最大成员对齐值为 4，成员结束于偏移 8，因此总大小为 8 Byte。

对于 struct B：

1. char b 从偏移 0 开始，占用 0；
2. int a 的对齐值为 4；偏移 1 不是 4 的倍数，因此在偏移 1～3 补 3 Byte；
3. int a 从偏移 4 开始，占用 4～7；
4. short c 从偏移 8 开始，占用 8～9；
5. 最大成员对齐值为 4；当前结束位置为偏移 10，因此在偏移 10～11 补 2 Byte；
6. 总大小为 12 Byte。

因此：

$$
\operatorname{sizeof}(A)=8,\qquad
\operatorname{sizeof}(B)=12.
$$

> [!tip] 结构体对齐的核心
> 成员占用空间的简单相加，不一定等于结构体的实际大小。应先按声明顺序，根据每个成员的对齐值安排起始偏移和内部填充，最后再按最大成员对齐值补齐结构体尾部。

> [!warning] 做题边界
> 这里的 int=4 Byte、short=2 Byte、char=1 Byte，以及对齐值分别为 4、2、1，都是题目采用的常见模型，不是所有平台上的绝对结论。若题干给出其他平台、编译器或 #pragma pack 规则，应以题干为准。

> [!tip] 边界对齐判断模板
> 先确认数据类型和对齐值，再确认成员起始地址，最后计算“起始地址 mod 对齐值”。结构体总大小还要检查是否为最大成员对齐值的整数倍。不要先判断大小端；端序决定字节排列，对齐决定地址布局。

## 5. 综合解题模板

遇到“数据宽度、大小端、边界对齐”综合题时：

1. 将 bit 数换算为 Byte 数；
2. 根据首地址和占用字节数写出地址范围；
3. 用首地址和对齐数判断是否自然对齐；
4. 按大端或小端排列每个字节；
5. 若有结构体，先计算成员偏移，再补齐结构体总大小；
6. 最后检查地址是否连续、字节数量是否正确、端序是否反向。

## 6. 易错点

- 高有效字节不等于高地址字节；
- 大小端改变字节排列，不改变字节内部 bit 排列；
- 4 字节数据占用 4 个连续地址，不是 4 个 bit 地址；
- 端序判断必须以最低地址为起点；
- 对齐判断先看首地址，不由大小端决定；
- 结构体大小可能包含填充字节；
- 自然对齐是常见实现约定，具体题目仍以题干给出的平台规则为准。
