# 定点数的表示

定点数是指物理小数点位置固定不变的数。在计算机中，通常将小数点位置默认约定在数值的某一位上：
- **定点整数**：小数点约定在数值最低位的右侧（用来表示整数）。
- **定点小数**：小数点约定在数值符号位与数值最高位之间（用来表示纯小数）。

---

## 1. 无符号整数的表示与有符号整数的表示

### (1) 无符号整数的表示
当所有二进制位均用于表示数值（无符号位）时，该编码称为**无符号整数**，简称无符号数。
* **物理特征**：寄存器的所有位数全部用来表示数值本身，没有符号位，数值位隐含为非负整数。
* **表示范围**：对于 $n$ 位无符号整数，其表示范围为 **$0 \sim 2^n - 1$**。
* **主要特点**：由于无须保留符号位，在相同字长下，无符号整数能表示的最大值大于有符号整数。
* **应用场景**：适用于仅涉及非负整数且结果不会产生负值的场景。例如，无符号整数可用于进行**地址运算**，或用来**表示指针**。
* **典型字长（8位）对比**：
  - **8 位无符号整数**：最小值为 `0000 0000` ($0$)，最大值为 `1111 1111` ($2^8 - 1 = 255$)，表示范围为 **$0 \sim 255$**。
  - **8 位有符号整数（补码）**：最小值为 `1000 0000` ($-2^7 = -128$)，最大值为 `0111 1111` ($2^7 - 1 = 127$)，表示范围为 **$-128 \sim 127$**。
* **考点追踪：机器码、补码与无符号数之间的转换与多码制真值对比**
  - **多码制真值对比典型示例**：以 4 位机器码 `1101` 为例：
    - 若作为**无符号整数**，其真值为：
      $$2^3 + 2^2 + 0 \times 2^1 + 2^0 = 8 + 4 + 0 + 1 = 13$$
    - 若作为**有符号整数的原码**，最高位为 `1` 表示负数，其余数值位 `101B = 5`，对应的真值为 **$-5$**。
    - 若作为**有符号整数的补码**，其最高位具有负权值，对应的真值计算为：
      $$-2^3 + 4 + 0 + 1 = -3$$
  - **十六进制机器数正负快速判断秒杀规律**：
    对于常见 8 位、16 位、32 位等有符号机器数，可通过其最高位十六进制字符直接口算正负：
    - 若最高十六进制位 **$\le 7$**（最高二进制位为 `0`），说明对应的真值为**非负数**（正数或零）。
    - 若最高十六进制位 **$\ge 8$**（最高二进制位为 `1`），说明对应的真值为**负数**。

### (2) 有符号整数的表示
有符号整数通过在数值位前增设一位符号位（`0` 表示正，`1` 表示负）来表示正负。
虽然原码、反码和补码均可用于表示有符号整数，但**现代计算机统一采用补码**，主要原因在于补码具有以下三大独特优势：
1. **零的表示唯一**：消除了 $+0$ 与 $-0$ 的重复编码（例如，在 8 位补码中 $0$ 仅有 `0000 0000` 一种表示）。
2. **符号位可与数值位一同参与运算**：省去了单独的符号判断与处理硬件，使计算机的加减法运算能够统一为加法操作，极大简化了运算器（ALU）的内部电路设计。
3. **表示范围更大**：比原码和反码多表示一个最小负数（例如，8 位字长下原码/反码范围为 $-127 \sim +127$，而补码多表示一个最小负数 $-128$），具有更宽的动态范围。
* **表示范围**：对于 $n$ 位有符号整数（补码），其表示范围为 **$-2^{n-1} \sim 2^{n-1} - 1$**。

> [!NOTE] **C 语言中的整型表示与强转扩展机制**
> 在 C 语言中，无符号数与有符号数（补码）在进行强转或字宽变化时，会涉及到**位扩展**（零扩展/符号扩展）和**位截断**的底层物理操作。详细机器级机制与 408 历年真题剖析请参阅专属概念页面：[[wiki/concepts/计算机组成原理/第2章 数据的表示和运算/C语言中的整数类型及类型转换|C语言中的整数类型及类型转换]]。

## 2. 四种机器数的数学定义

在计算机中，有符号数有四种常用的编码表示方法：**原码**、**反码**、**补码**和**移码**。

### (1) 原码表示法（Sign-Magnitude）
原码是用最高位表示符号位，数值部分用绝对值二进制表示。

- **定点整数公式**：
  $$
  [x]_原 = \begin{cases} x, & 0 \le x < 2^n \\ 2^n - x = 2^n + |x|, & -2^n < x \le 0 \end{cases}
  $$
- **定点小数公式**：
  $$
  [x]_原 = \begin{cases} x, & 0 \le x < 1 \\ 1 - x = 1 + |x|, & -1 < x \le 0 \end{cases}
  $$
- **整数表示范围**：
  - 若机器字长为 $n+1$ 位（符号位占 1 位，数值位占 $n$ 位）：**$-(2^n - 1) \le x \le 2^n - 1$**。
  - 若机器字长为 $n$ 位（符号位占 1 位，数值位占 $n-1$ 位）：**$-(2^{n-1} - 1) \le x \le 2^{n-1} - 1$**。
  - **8位字长实例**：原码整数范围为 **$-127 \sim +127$**，其中最小负数机器数为 $11111111\text{B}$（真值为 $-127$），最大正数机器数为 $01111111\text{B}$（真值为 $+127$）。
- **特点**：
  - **简单直观**，真值与原码的数值位直接对应。
  - **$0$ 的表示不唯一**：有 $[+0]_原 = 0000\dots0$ 和 $[-0]_原 = 1000\dots0$ 两种形式。
  - 原码的加减运算复杂，符号位不能直接参与数值运算，需用硬件判断符号后再做加或减。
- **教材经典例题表示**（机器字长为 8 位，含 1 位符号位）：
  * $x_1 = +19 = +10011\text{B} \implies [x_1]_{\text{原}} = 0,0010011$
  * $x_2 = -19 = -10011\text{B} \implies [x_2]_{\text{原}} = 1,0010011$
  * $x_3 = +0.75 = +0.11\text{B} \implies [x_3]_{\text{原}} = 0.1100000$
  * $x_4 = -0.75 = -0.11\text{B} \implies [x_4]_{\text{原}} = 1.1100000$

### (2) 反码表示法（One's Complement）
反码是原码到补码的过渡形式。

- **定点整数公式**：
  $$
  [x]_反 = \begin{cases} x, & 0 \le x < 2^n \\ (2^{n+1} - 1) + x = (2^{n+1} - 1) - |x|, & -2^n < x \le 0 \end{cases}
  $$
- **定点小数公式**：
  $$
  [x]_反 = \begin{cases} x, & 0 \le x < 1 \\ (2 - 2^{-n}) + x = (2 - 2^{-n}) - |x|, & -1 < x \le 0 \end{cases}
  $$
- **整数表示范围**：
  - 若机器字长为 $n+1$ 位（符号位占 1 位，数值位占 $n$ 位）：**$-(2^n - 1) \le x \le 2^n - 1$**。
  - 若机器字长为 $n$ 位（符号位占 1 位，数值位占 $n-1$ 位）：**$-(2^{n-1} - 1) \le x \le 2^{n-1} - 1$**。
  - **8位字长实例**：反码整数范围为 **$-127 \sim +127$**，其中最小负数机器数为 $10000000\text{B}$（真值为 $-127$），最大正数机器数为 $01111111\text{B}$（真值为 $+127$）。
- **特点**：
  - **正数的反码与原码相同；负数的反码为符号位保持不变，数值位按位取反**。
  - **$0$ 的表示亦不唯一**：有 $[+0]_反 = 0000\dots0$ 和 $[-0]_反 = 1111\dots1$ 两种形式。
- **教材经典例题表示**（机器字长为 8 位，含 1 位符号位）：
  * $x_1 = +19 = +10011\text{B} \implies [x_1]_{\text{反}} = 0,0010011$
  * $x_2 = -19 = -10011\text{B} \implies [x_2]_{\text{反}} = 1,1101100$
  * $x_3 = +0.75 = +0.11\text{B} \implies [x_3]_{\text{反}} = 0.1100000$
  * $x_4 = -0.75 = -0.11\text{B} \implies [x_4]_{\text{反}} = 1.0011111$

### (3) 补码表示法（Two's Complement）
补码将减法运算统一为加法运算，是计算机内部存储和进行定点数算术运算的最常用码制。

- **定点整数公式**：
  $$
  [x]_补 = \begin{cases} x, & 0 \le x < 2^n \\ 2^{n+1} + x = 2^{n+1} - |x| \pmod{2^{n+1}}, & -2^n \le x \le 0 \end{cases}
  $$
- **定点小数公式**：
  $$
  [x]_补 = \begin{cases} x, & 0 \le x < 1 \\ 2 + x = 2 - |x| \pmod{2}, & -1 \le x \le 0 \end{cases}
  $$
- **整数表示范围**：
  - 若机器字长为 $n+1$ 位（符号位占 1 位，数值位占 $n$ 位）：**$-2^n \le x \le 2^n - 1$**。
  - 若机器字长为 $n$ 位（符号位占 1 位，数值位占 $n-1$ 位）：**$-2^{n-1} \le x \le 2^{n-1} - 1$**。
  - **8位字长实例**：补码整数范围为 **$-128 \sim +127$**，其中最小负数机器数为 $10000000\text{B}$（真值为 $-128$），最大正数机器数为 $01111111\text{B}$（真值为 $+127$）。
- **特点**：
  - **正数的补码与原码相同；负数的补码为其反码在最低位加 $1$**。
  - **$0$ 的表示唯一**：$[+0]_补 = [-0]_补 = 0000\dots0$。由于模运算丢弃溢出的高位，$-0$ 的补码 $1000\dots0 + 1$ 溢出后也变成了 $0000\dots0$。
  - **能多表示一个最小负数**：整数能表示 $-2^n$，小数能表示 $-1$（补码为 $1.000\dots0$）。
- **教材经典例题表示**（机器字长为 8 位，含 1 位符号位）：
  * $x_1 = +19 = +10011\text{B} \implies [x_1]_{\text{补}} = 0,0010011$
  * $x_2 = -19 = -10011\text{B} \implies [x_2]_{\text{补}} = 1,1101101$
  * $x_3 = +0.75 = +0.11\text{B} \implies [x_3]_{\text{补}} = 0.1100000$
  * $x_4 = -0.75 = -0.11\text{B} \implies [x_4]_{\text{补}} = 1.0100000$

### (4) 移码表示法（Offset Binary）
移码通常用于表示浮点数的阶码，便于利用硬件直接比较大小。

> ⚠️ **重要易错点：定点小数无移码！**
> 移码仅用于表示定点整数（例如浮点数的阶码），在 408 考研及王道教材中，**定点小数没有移码表示形式**。

- **定点整数公式**：
  $$
  [x]_移 = 2^n + x \quad (-2^n \le x < 2^n)
  $$
- **整数表示范围**：
  - 若机器字长为 $n+1$ 位（符号位占 1 位，数值位占 $n$ 位）：**$-2^n \le x \le 2^n - 1$**。
  - 若机器字长为 $n$ 位（符号位占 1 位，数值位占 $n-1$ 位）：**$-2^{n-1} \le x \le 2^{n-1} - 1$**。
  - **8位字长实例**：移码整数范围为 **$-128 \sim +127$**，其中最小负数机器数为 $00000000\text{B}$（真值为 $-128$），最大正数机器数为 $11111111\text{B}$（真值为 $+127$）。
- **特点**：
  - **移码与补码仅符号位相反**，数值位完全相同。例如，若 $[x]_补 = 10111$，则 $[x]_移 = 00111$。
  - 移码中 **$0$ 的表示唯一**：$[0]_移 = 2^n = 1000\dots0$。
  - 移码保持了真值的单调大小顺序，这使得硬件比较器只需按照无符号整数的比较逻辑就能直接判定两个移码的大小。
- **教材经典例题表示**（机器字长为 8 位，含 1 位符号位）：
  * $x_1 = +19 = +10011\text{B} \implies [x_1]_{\text{移}} = 1,0010011$
  * $x_2 = -19 = -10011\text{B} \implies [x_2]_{\text{移}} = 0,1101101$
  * *小数部分：无移码表示方式。*

## 3. 定点数表示范围与最值（按字长 $n$ 位与数值位 $n$ 位对照）

做题时必须仔细看清题干中对“机器字长为 $n$ 位”（总字长）还是“数值位为 $n$ 位”的约定。以下给出两种标准对照表：

### (1) 约定一：机器总字长为 $n$ 位（含 1 位符号位，数值位为 $n-1$ 位，王道教材与 408 真题最常用格式）

| 码制 | 定点整数表示范围 | 定点小数表示范围 | $0$ 的表示编码（以 8 位为例） |
| :--- | :--- | :--- | :--- |
| **无符号数** | $0 \le x \le 2^n - 1$ | $0 \le x \le 1 - 2^{-n}$ | （无符号位） |
| **原码** | $-(2^{n-1} - 1) \le x \le 2^{n-1} - 1$ | $-(1 - 2^{-(n-1)}) \le x \le 1 - 2^{-(n-1)}$ | $[+0]_{\text{原}} = 0,0000000$<br>$[-0]_{\text{原}} = 1,0000000$ |
| **反码** | $-(2^{n-1} - 1) \le x \le 2^{n-1} - 1$ | $-(1 - 2^{-(n-1)}) \le x \le 1 - 2^{-(n-1)}$ | $[+0]_{\text{反}} = 0,0000000$<br>$[-0]_{\text{反}} = 1,1111111$ |
| **补码** | $-2^{n-1} \le x \le 2^{n-1} - 1$ | $-1 \le x \le 1 - 2^{-(n-1)}$ | $[0]_{\text{补}} = 0,0000000$（唯一） |
| **移码** | $-2^{n-1} \le x \le 2^{n-1} - 1$ | （小数无移码） | $[0]_{\text{移}} = 1,0000000$（唯一） |

---

### (2) 约定二：数值位为 $n$ 位（含 1 位符号位，机器总字长为 $n+1$ 位，多用于理论分段函数定义）

| 码制 | 定点整数表示范围 | 定点小数表示范围 | $0$ 的表示编码（以 9 位为例） |
| :--- | :--- | :--- | :--- |
| **无符号数** | $0 \le x \le 2^{n+1} - 1$ | $0 \le x \le 1 - 2^{-(n+1)}$ | （无符号位） |
| **原码** | $-(2^n - 1) \le x \le 2^n - 1$ | $-(1 - 2^{-n}) \le x \le 1 - 2^{-n}$ | $[+0]_{\text{原}} = 0,00000000$<br>$[-0]_{\text{原}} = 1,00000000$ |
| **反码** | $-(2^n - 1) \le x \le 2^n - 1$ | $-(1 - 2^{-n}) \le x \le 1 - 2^{-n}$ | $[+0]_{\text{反}} = 0,00000000$<br>$[-0]_{\text{反}} = 1,11111111$ |
| **补码** | $-2^n \le x \le 2^n - 1$ | $-1 \le x \le 1 - 2^{-n}$ | $[0]_{\text{补}} = 0,00000000$（唯一） |
| **移码** | $-2^n \le x \le 2^n - 1$ | （小数无移码） | $[0]_{\text{移}} = 1,00000000$（唯一） |

> ❗ **考研重难点：为什么补码能多表示一个负数？**
> - 原码和反码中，有 $+0$ 和 $-0$ 两个编码，浪费了一个编码位置。
> - 补码中 $0$ 的编码唯一，因此腾出了一个编码给负数。
> - 对于机器总字长为 $n$ 位的定点整数，补码编码 `1000...0`（$1$ 后面跟 $n-1$ 个 $0$）在原码中代表 $-0$，但在补码中其代表最小负值 **$-2^{n-1}$**（如 8 位字长下为 $-128$），该值没有对应的原码和反码。
> - 对于机器总字长为 $n$ 位的定点小数，补码编码 `1.000...0` 的真值为 **$-1$**。

## 4. 机器数之间的转换与快捷判定

### (1) 原码与补码的互转（负数快捷求法）

### (2) 求相反数的补码

求一个 $n$ 位补码所表示真值的相反数，可以直接对全部位按“按位取反，再加 $1$”处理，并保留原机器字长的低 $n$ 位：

$$
[-x]_{\text{补}}=\overline{[x]_{\text{补}}}+1.
$$

#### 例 1：由正数求负数

4 位机器中，$+5$ 的补码为 0101：

$$
\overline{0101}=1010,\qquad 1010+1=1011.
$$

因此 1011 表示 $-5$。

#### 例 2：由负数求正数

已知 1011 表示 $-5$：

$$
\overline{1011}=0100,\qquad 0100+1=0101.
$$

所以它的相反数为 $+5$。

#### 例 3：最小负数的特殊情况

$n$ 位补码中，最小值 $-2^{n-1}$ 没有对应的正数。例如 4 位补码中：

$$
1000=-8.
$$

对 1000 求相反数：

$$
\overline{1000}=0111,\qquad 0111+1=1000.
$$

结果仍为 1000。这不是计算错误，而是因为 $+8$ 超出了 4 位补码的表示范围 $[-8,7]$，发生了特殊的回绕。

> [!tip] 记忆方法
> 从最低位开始向左找第一个 1：该位及其右侧保持不变，左侧所有位取反。这个方法与“按位取反再加 1”等价。

> [!warning] 字长必须固定
> 求相反数前先确认机器字长。若先扩展位数再取反，得到的结果可能与题目要求的固定字长补码不同。
*对于正数，原码与补码完全一致。*
对于负数，原码 $\leftrightarrow$ 补码转换存在一个非常适合手算秒杀的 “**求补**” 快捷规则：
- **规则**：**从数值位的最低位（最右端）开始往左看，找到第一个出现的 `1`。这个 `1` 及其右边所有的 `0`（如果有的话）保持不变。而这个 `1` 左边的所有数值位（不包含符号位）按位取反**。
- **示例**：设 $[-7]_原 = 1,0000111$：
  - 最右侧第一个 `1` 在最低位：$\dots 1$
  - 该 `1` 右侧无 `0`；该 `1` 左侧数值位为 $000011$，取反后为 $111100$
  - 符号位保持 $1$：$[ -7 ]_补 = 1,1111001$ ✓。
- **双向适用**：将该规则直接应用于负数的补码，也能以同样步骤求得其原码。

### (3) 负数真值与补码数值位互求的“求和全0”快速算法

对于负数，求其真值绝对值和补码数值位时，存在一个极具物理解释的“求和为全0”快速手算算法：
- **物理原理**：在模运算中，负数补码的数值位与该负数真值的绝对值之和正好等于模（对于 $n$ 位数值位，模为 $2^n$）。因此在 $n$ 位二进制加法中，**负数补码的数值位与它的绝对值二进制相加，其结果在 $n$ 位数值位内为“全0”（发生最高位进位溢出）**。
- **快速算法步骤**：
  1. **确定符号**：因为是负数，符号位固定为 `1`。
  2. **求绝对值/数值位**：两者相加为全 0。利用“从右往左遇到第一个 `1` 及右侧的 `0` 保持不变，左侧全部按位取反”规律可直接得出对方。
- **双向互求示例**：
  - **补码求真值**：已知 $[-x]_{\text{补}} = 1,1101101$，求其真值 $x$。
    1. 符号位为 `1`，说明 $x$ 是负数。
    2. 数值位为 `1101101`。
    3. 寻找一个数 $Y$（即绝对值 $|x|$）使得：
       ```text
         1101101
       +       Y
       ---------
        10000000  (数值位全0)
       ```
    4. 手算：最右侧为 `1` 保持不变，左侧 `110110` 全部按位取反得到 `001001`，故 $Y = 0010011_{\text{B}} = 19$。
    5. 结论：真值绝对值为 $19$，真值 $x = -19$。
  - **真值求补码**：已知负数真值 $x = -19$，求其 8 位补码 $[x]_{\text{补}}$（1位符号位，7位数值位）。
    1. 符号位固定为 `1`。
    2. 绝对值 $|x| = 19$，二进制展开为 `0010011`。
    3. 寻找数值位 $Y$，使两者相加结果在 7 位数值位内为全 0：
       ```text
         0010011
       +       Y
       ---------
        10000000  (数值位全0)
       ```
    4. 手算：最右位 `1` 保持不变，左侧 `001001` 取反得到 `110110`，故 $Y = 1101101_{\text{B}}$。
    5. 结论：补码 $[x]_{\text{补}} = 1,1101101$。

### (4) 正数真值与各机器数直接转换

对于正数，其原码、反码、补码三者完全相同，数值位均与其二进制绝对值一致，最高符号位为 `0`：
- **规则**：**如果真值 $x$ 为正数，求其原码、反码、补码时，符号位填 `0`，数值位直接填其二进制绝对值即可（如果字长不够，在符号位和数值最高位之间补 `0` 补齐字长）**。
- **示例**：设真值 $x = +19 = +10011\text{B}$，求其 8 位机器数：
  - 符号位填 `0`；
  - 绝对值二进制为 `10011`（5位），需要在符号位与数值位之间补 2 个前导零，补齐为 7 位数值位 `0010011`；
  - 拼接直接得出：$[x]_原 = [x]_反 = [x]_补 = 0,0010011$。

### (5) 高频真题实例与十六进制直接运算秒杀技巧
在 16 位字长（对应 4 位十六进制数）或 8 位字长（对应 2 位十六进制数）下，负数真值 $-x$ 与其补码之和在数值位内为模（如 $16^4 = 10000\text{H}$ 或 $16^2 = 100\text{H}$）。
因此，求负数补码或由补码求真值，**可以直接在十六进制下使用模减去绝对值（如 $10000\text{H} - |x|$）进行减法，完全无需转二进制**。

**十六进制直接“求和全0”快速手算口算规则**：
- 从右往左看，找到第一个非零的十六进制位。用 **$16$（即十六进制的 $10\text{H}$）减去该位的值**；
- 该位左侧的所有十六进制位全部用 **$15$（即十六进制的 $\text{FH}$）减去它们的值**；
- 该位右侧的所有 `0`（如果有）保持 `0` 不变。

**示例一：已知 16 位补码为 $\text{FFF8H}$，直接求其十进制真值**
1. 最高位是 `F`（二进制首位为 `1`），说明真值为负数。
2. 寻找与其相加为 `10000H` 的十六进制数 $Y$（即绝对值 $|x|$）：
   ```text
     F F F 8  (H)
   + Y Y Y Y  (H)
   ----------
   1 0 0 0 0  (H)
   ```
3. 从右往左直接口算：
   * 最右侧第一个非零位是 `8`：用 $16 - 8 = 8$；
   * 左侧所有位均为 `F`：用 $15 - 15 = 0$；
   * 直接得出绝对值表示为 $0008\text{H} = 8$。
4. 结论：真值 $x = -8$。

**示例二：已知负数真值 $x = -45$（十进制），直接求其 16 位补码（4位十六进制）**
1. **十进制转十六进制绝对值**：十进制绝对值 $|x| = 45 = 2 	imes 16 + 13 \implies 2	ext{DH}$。
2. **前导 0 补齐**：16 位字长对应 4 位十六进制数，表示为 `002DH`。
3. **寻找与 `002DH` 相加为 `10000H` 的数**：
   ```text
     0 0 2 D  (H)
   + Y Y Y Y  (H)
   ----------
   1 0 0 0 0  (H)
   ```
4. **十六进制直接口算**：
   - 最右侧首个非零位 `D`（十进制 13）：$16 - 13 = 3$；
   - 其左侧各位置，用 $15$ 减：
     - `2` 对应位置：$15 - 2 = 13 \implies 	ext{DH}$；
     - 右侧 `0` 对应位置：$15 - 0 = 15 \implies 	ext{FH}$；
     - 左侧 `0` 对应位置：$15 - 0 = 15 \implies 	ext{FH}$。
   - 直接求得 16 位补码十六进制表示为 **$\text{FFD3H}$**。

**示例三：已知负数真值 $x = -45$（十进制），直接求其 8 位补码（2位十六进制）**
1. **十进制转十六进制绝对值**：十进制绝对值 $|x| = 45 \implies 2	ext{DH}$，8 位字长对应 2 位十六进制，模为 $16^2 = 100\text{H}$。
2. **寻找与 `2DH` 相加为 `100H` 的数**：
   - 最右位 `D`（十进制 13）：$16 - 13 = 3$。
   - 左侧位 `2`：$15 - 2 = 13 \implies \text{D}$。
   - 直接求得 8 位补码十六进制表示为 **$\text{D3H}$**（即二进制 `1101 0011B`）。

> [!TIP] **补码中常见特值机器码**
>
> 在做数据表示、数据运算的算术溢出以及寄存器取值范围题目时，牢记以下三类补码特值（以 $n$ 位字长为例）能帮助你实现秒杀：
> 1. **全 0 补码（`0000...00`）**：
>    - **定点整数**：表示真值 **$0$**。
>    - **定点小数**：表示真值 **$0.0$**。
>    - *注：补码中 $0$ 的表示是唯一的，即 $[+0]_补 = [-0]_补 = 0000...00$*。
> 2. **全 1 补码（`1111...11`）**：
>    - **定点整数**：表示真值 **$-1$**。*(任何字长下，全 1 补码的真值永远是 $-1$)*。
>    - **定点小数**：表示真值 **$-2^{-(n-1)}$**。*(这是补码能表示的绝对值最小的负小数，即最贴近 $0$ 的负数)*。
> 3. **1 后面全 0 补码（`1000...00`）**：
>    - **定点整数**：表示真值 **$-2^{n-1}$**。*(即该字长下能表示的最小负数，没有相对应的原码和反码)*。
>    - **定点小数**：表示真值 **$-1$**。*(即小数补码所能表示的最小负值，同样没有原码和反码与其对应)*。

> [!TIP] **移码表示法与偏置值的物理原理**
>
> 移码（Offset Binary）常用于浮点数的阶码。其物理本质是通过**加上一个正的偏置值（Bias）**，将包含负数在内的真值区间平移映射到非负数区间，从而**保持真值与机器数在大小顺序上的一致性**，方便硬件直接比较大小。
> 1. **标准整数移码的偏置值**：通常约定为 **$2^{n-1}$**（其中 $n$ 为字长）。
> 2. **移码与补码的秒杀转换关系**：**数值位完全相同，最高符号位相反**。
>    - 公式：$[x]_移 = 2^{n-1} + x$；而由 $[x]_补$ 转换只需对最高位（符号位）取反即可。
> 3. **以 3 位字长（$n=3$，偏置值为 $2^{3-1} = 4$）为例对照演示**：
>
>    | 真值 $x$ | 补码 $[x]_补$ | 加上偏置量 $+4$ 后的值 | 移码 $[x]_移$ | 转换分析（最高位取反，数值位不变） |
>    | :---: | :---: | :---: | :---: | :---: |
>    | **$-4$** | `100` | $0$ | `000` | $[x]_补$ 的 `100` $	o$ 最高位 `1` 取反为 `0` $	o$ $[x]_移$ 为 `000` |
>    | **$-3$** | `101` | $1$ | `001` | $[x]_补$ 的 `101` $	o$ 最高位 `1` 取反为 `0` $	o$ $[x]_移$ 为 `001` |
>    | **$-2$** | `110` | $2$ | `010` | $[x]_补$ 的 `110` $	o$ 最高位 `1` 取反为 `0` $	o$ $[x]_移$ 为 `010` |
>    | **$-1$** | **`111`** | **$3$** | **`011`** | **$[x]_补$ 的 `111` $	o$ 最高位 `1` 取反为 `0` $	o$ $[x]_移$ 为 `011`** *(图示圆圈)* |
>    | **$0$** | `000` | $4$ | `100` | $[x]_补$ 的 `000` $	o$ 最高位 `0` 取反为 `1` $	o$ $[x]_移$ 为 `100` |
>    | **$1$** | `001` | $5$ | `101` | $[x]_补$ 的 `001` $	o$ 最高位 `0` 取反为 `1` $	o$ $[x]_移$ 为 `101` |
>    | **$2$** | **`010`** | **$6$** | **`110`** | **$[x]_补$ 的 `010` $	o$ 最高位 `0` 取反为 `1` $	o$ $[x]_移$ 为 `110`** *(图示圆圈)* |
>    | **$3$** | **`011`** | **$7$** | **`111`** | **$[x]_补$ 的 `011` $	o$ 最高位 `0` 取反为 `1` $	o$ $[x]_移$ 为 `111`** *(图示圆圈)* |

> [!TIP] **四种编码表示的特征与大小比较规律（考点追踪：补码大小的判断）**
>
> 对原码、反码、补码、移码的结构特征和大小比较规律进行如下系统归纳：
>
> 1. **正数编码的同异性**：
>    - 正数的**原码、反码、补码完全相同**；**移码则不同**（最高符号位与它们相反）。
> 2. **数轴对称性与零的表示唯一性**：
>    - **原码与反码**：在数轴上关于原点**对称**，都存在 $+0$ 与 $-0$ 两种编码形式（零的表示不唯一）。
>    - **补码与移码**：在数轴上**不对称**，零的表示**唯一**（消除了重复的零），且都比原码/反码多表示一个负数（对定点整数多表示 $-2^n$，而**定点小数由于无移码，仅补码多表示 $-1$**）。
> 3. **真值大小比较的快捷规律**：
>    - **原码**：可直观比较大小（符号位相同且为正数时，数值位越大真值越大；符号位相同且为负数时，数值位越大真值越小）。
>    - **负数补码/反码**：不能像原码那样直观判断。但它们存在一个极强的单调规律：
>      **在同为负数的前提下，补码或反码的数值部分（即除去符号位后的剩余位）越大，其对应的真值也越大**。
>      - *秒杀实例（8位补码）*：
>        $A = 1,1111110_2$（数值部分为大值 `126`） $\implies$ 真值为 $-2$
>        $B = 1,1111100_2$（数值部分为小值 `124`） $\implies$ 真值为 $-4$
>        因为数值位 $A > B$，所以对应的真值 $-2 > -4$，即真值 $A > B$。同样适用于反码。

## 5. 易错点总结

- **小数补码表示范围**：定点小数补码的最大正值为 $1 - 2^{-n}$（如 8 位为 $0.1111111_2 = 127/128$），但最小负值为 **$-1$**（编码为 $1.0000000$）。绝对不能写成 $-1 \sim 1$ 的对称区间。
- **移码的偏置常数**：移码是用真值加上一个偏置常数得到的。王道教材中整数移码的偏置常数默认是 **$2^n$**（即最高位加上 $1$），但在某些特殊协议（如 IEEE 754 浮点数阶码）中，偏置常数会有所变化（如单精度阶码字长 8 位，偏置常数是 127 而不是 128），需根据具体规定计算。
- **无符号数减法的溢出**：无符号数在计算机中没有符号位，因此它们相减如果结果为负，在硬件电路中表现为产生向最高位的借位，该结果以补码形式存放，但如果直接按无符号数解读会产生严重错误（即发生溢出）。
