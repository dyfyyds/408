# 数制与编码

## 1. 进位计数制

在进位计数制中，每个数位所能使用的字符个数称为**基数**（例如十进制的基数为 $10$，二进制的基数为 $2$）。处在不同位置的数位代表不同的值，即每个数位对应一个固定的权值，称为**权**。

任何一个 $r$ 进制数都可以表示为按权展开式：

$$
D = a_{n-1}r^{n-1} + a_{n-2}r^{n-2} + \cdots + a_0r^0 + a_{-1}r^{-1} + \cdots + a_{-m}r^{-m} = \sum_{i=-m}^{n-1} a_i r^i
$$

式中，$r$ 是基数，$r^i$ 是第 $i$ 位的权，$a_i$ 代表各数位上的字符，且满足 $0 \le a_i < r$。

### 进制表示符号
在计算机科学中，数制通常用括号和下标，或者对应的英文字母后缀来标明：
- **二进制（Binary）**：以 **B** 结尾，例如 $10111\text{B}$。
- **十进制（Decimal）**：以 **D** 结尾（或省略任何标识），例如 $23\text{D}$ 或 $23$。
- **八进制（Octal）**：以 **O** 结尾，例如 $27\text{O}$。
- **十六进制（Hexadecimal）**：以 **H** 结尾，例如 $17\text{H}$。
> 换算示例：$10111\text{B} = 23 = 17\text{H}$。

### 计算机内部采用二进制编码的原因
1. **物理器件极易实现**：二进制只有 $0$ 和 $1$ 两个字符，在物理上极易用具有两个稳定状态的物理器件（如电平高低、电容带电与否）实现。
2. **可靠性高且算法最简单**：二进制数的编码、写入、读取和传输等物理操作通道十分可靠。同时，二进制的算术规则极其简单，四则运算（尤其是乘除法）可以通过移位和加减法实现。
3. **与逻辑代数天然吻合**：二进制字符 $0$ 和 $1$ 刚好对应逻辑代数中的“假”和“真”，为计算机实现逻辑运算和逻辑判断提供了极大的方便。
4. **易于简化硬件结构**：乘除法能转换成加减移位，使得计算机中负责运算的算术逻辑部件（ALU）硬件结构十分简单。

## 2. 各种进制数之间的转换

### (1) $r$ 进制数转换为十进制数（按权展开求和）
将 $r$ 进制数按权展开，然后按十进制的运算规则求和即可。
- **示例**：$(10111)_2 = 1 \times 2^4 + 0 \times 2^3 + 1 \times 2^2 + 1 \times 2^1 + 1 \times 2^0 = 16 + 0 + 4 + 2 + 1 = 23$。

### (2) 十进制数转换为 $r$ 进制数（整数除基取余，小数乘基取整）
- **整数部分：除基取余法（倒序排列）**：
  用十进制整数连续除以基数 $r$，记录每次的余数。重复此步骤，直至商为 $0$。最先得到的余数为最低位（$a_0$），最后得到的余数为最高位，将余数**从下往上倒序排列**。
  - **示例**：十进制整数 $11$ 转换为二进制（$r=2$）：
    - $11 \div 2 = 5 \cdots \cdots \text{余 } 1$ （最低位 $a_0 = 1$）
    - $5 \div 2 = 2 \cdots \cdots \text{余 } 1$ （$a_1 = 1$）
    - $2 \div 2 = 1 \cdots \cdots \text{余 } 0$ （$a_2 = 0$）
    - $1 \div 2 = 0 \cdots \cdots \text{余 } 1$ （最高位 $a_3 = 1$）
    - 倒序结果：$(11)_{10} = (1011)_2$。

- **小数部分：乘基取整法（正序排列）**：
  用十进制纯小数连续乘以基数 $r$，记录每次乘积的整数部分。重复此步骤，用上一步的小数部分继续乘，直到乘积的小数部分为 $0$（或者达到题目要求的精度位数）。最先得到的整数部分为最高位（$b_{-1}$），将所得整数部分**从上往下正序排列**。
  - **示例**：十进制小数 $0.8125$ 转换为二进制（$r=2$）：
    - $0.8125 \times 2 = 1.625 \to \text{取整 } 1$ （最高位 $b_{-1} = 1$），余小数 $0.625$
    - $0.625 \times 2 = 1.25 \to \text{取整 } 1$ （$b_{-2} = 1$），余小数 $0.25$
    - $0.25 \times 2 = 0.5 \to \text{取整 } 0$ （$b_{-3} = 0$），余小数 $0.5$
    - $0.5 \times 2 = 1.0 \to \text{取整 } 1$ （$b_{-4} = 1$），余小数 $0.0$ （计算结束）
    - 正序结果：$(0.8125)_{10} = (0.1101)_2$。
  - **❗ 精度误差成因**：十进制小数（例如 $0.3$、$0.2$）在转换为二进制时往往是“乘不尽”的无限循环小数。由于计算机中的存储位数有限，必须对超出部分进行截断，这不可避免地带来了**精度舍入误差**，这是计算机处理浮点数时容易产生误差的物理根源。

### (3) 二进制与八/十六进制的互转（分组对应）
以**小数点**为界，整数部分向左分组，小数部分向右分组。
- **二进制转八进制**：每 $3$ 位二进制数分成一组，不足 $3$ 位时高位/低位**补 $0$**。
  - **示例**：$(11010111.01)_2 = (\underline{011}\ \underline{010}\ \underline{111} . \underline{010})_2 = (327.2)_8$。
- **二进制转十六进制**：每 $4$ 位二进制数分成一组，不足 $4$ 位时高位/低位**补 $0$**。
  - **示例**：$(11010111.01)_2 = (\underline{1101}\ \underline{0111} . \underline{0100})_2 = (\text{D}7.4)_{16}$。
  - **❗ 易错点**：在对小数部分 $(.01)_2$ 进行分组时，必须向右补 $0$ 凑成 $4$ 位（即 $.0100$ 对应十六进制的 $4$），绝不能写成向左补零的 $.0001$。
- **八/十六进制转二进制**：反向操作，直接将每一位八进制数对应展开为 $3$ 位二进制，或每一位十六进制对应展开为 $4$ 位二进制。
  - **示例**：$(\text{2B.C})_{16} = (0010\ 1011 . 1100)_2 = (101011.11)_2$。

## 3. BCD 码（二进制编码的十进制数）

人机交互中，常用 $4$ 位二进制数来表示 $1$ 位十进制数。$4$ 位二进制有 $16$ 种状态，表示十进制仅需 $10$ 种，因此需要舍弃 $6$ 种状态。由于编码规则不同，常用的 BCD 码有以下三种：

| 编码名称 | 编码类型 | 数位权值 | 编码特征 |
|---|---|---|---|
| **8421 码** | 有权码 | $8, 4, 2, 1$ | 权值与普通二进制前四位相同。 |
| **余3码** | 无权码 | 无固定权值 | 在 8421 码的基础上**加上 $(0011)_2$** 形成。 |
| **2421 码** | 有权码 | $2, 4, 2, 1$ | 存在重码（如 $5$ 可表示为 $1011$ 或 $0101$）。 |

### 2421 码的避免歧义规则
为了避免一数多码的歧义，在 2421 码中做出了如下硬性规定：
- **大于或等于 $5$** 的十进制数，其编码的最高位（$d_3$）**必须为 $1$**。
- **小于 $5$** 的十进制数，其编码的最高位（$d_3$）**必须为 $0$**。
  - **示例**：十进制数 $5$ 必须编码为 $1011$（因为 $5 \ge 5$），不能编码为 $0101$；十进制数 $4$ 则必须编码为 $0100$。

### 自补码特性（Self-complementing Code）
**余3码**和 **2421 码** 具有非常重要的自补码特性：
- **定义**：对于十进制数 $i$（其编码为 $A$），它关于 $9$ 的补数 $(9-i)$ 对应的编码刚好是 $A$ **按位取反** 的结果。
- **示例（2421码）**：
  - 十进制数 $4$ 的编码为 $0100$。
  - 其关于 $9$ 的补数为 $5$ ($9-4=5$)，对应的编码为 $1011$。
  - $0100$ 与 $1011$ 互为按位取反，体现了自补特性。这种特性在硬件设计中可以极大简化求补（如十进制减法运算）的电路结构。

### BCD 码加法修正
使用 8421 BCD 码进行加法运算时：
- 若两位相加之和**大于 $9$**（即落入 $1010_2 \sim 1111_2$ 的非法状态区），或者**产生向更高位的进位**，必须在该位加上 **$6$（即二进制 $0110_2$）** 进行进位修正。
- **示例**：$5 (0101) + 8 (1000) = 13 (1101)$：
  - 由于 $1101 > 1001$（大于9），属于非法编码。
  - 加上 6 修正：$1101 + 0110 = 10011$。
  - 结果为 $0001\ 0011$（十进制的 $1$ 和 $3$），正确得到了 $13$ 的 BCD 表示。

## 4. 字符与编码

### (1) ASCII 码（美国信息交换标准代码）
- 标准 ASCII 码使用 **$7$ 位二进制编码**，可以表示 $128$ 个字符（包含 $32$ 个非打印的控制字符和 $96$ 个可打印字符）。
- 在机器存储中，通常在最高位补 $0$，扩展为 **$8$ 位（即 $1$ 字节）** 存放。

### (2) 汉字编码（区位码、国标码与机内码的换算）
- **区位码**：由 $2$ 位区码和 $2$ 位位码组成（均为十进制形式，范围从 $01$ 到 $94$）。
- **国标码（交换码）**：将区位码的区码和位码分别转换为十六进制形式，并分别加上 $20\text{H}$（即十进制的 $32$）。
  $$
  \text{国标码} = \text{区位码(十六进制表示)} + 2020\text{H}
  $$
- **机内码（存储码）**：国标码的区码和位码的最高位均设为 $1$，以便与普通单字节 ASCII 码进行区分。即在国标码的基础上分别加上 $80\text{H}$。
  $$
  \text{机内码} = \text{国标码} + 8080\text{H} = \text{区位码(十六进制表示)} + \text{A}0\text{A}0\text{H}
  $$
- **换算示例**：已知某汉字的区位码是 $3012$（即第 $30$ 区，第 $12$ 位）：
  1. 将十进制的区码与位码分别转成十六进制：$30 \to 1\text{E}\text{H}$，$12 \to 0\text{C}\text{H}$。即区位码十六进制形式为 $1\text{E}0\text{C}\text{H}$。
  2. 计算国标码：$1\text{E}0\text{C}\text{H} + 2020\text{H} = 3\text{E}2\text{C}\text{H}$。
  3. 计算机内码：$3\text{E}2\text{C}\text{H} + 8080\text{H} = \text{B}\text{E}\text{A}\text{C}\text{H}$。

## 5. 易错点清单

- **十进制转二进制小数的精度**：要牢记除了能表示为 $k \cdot 2^{-n}$（$k, n$ 为整数）的十进制小数外，其余小数转换二进制都是无限循环小数，一定会产生精度损失。
- **2421 码大于等于 5 的最高位判定**：在手算 2421 码时，绝不能忘记 $\ge 5$ 时最高位必须为 1。例如 $5$ 必须为 $1011$。
- **自补码特征应用**：自补码求关于 9 的补数就是对每个二进制位按位取反，余3码和2421码是自补码，而 8421 码**不是**自补码。
- **机内码的标志位**：机内码两个字节的最高位都是 1（即 $\ge 80\text{H}$，或者说十六进制首位为字母），这是与单字节 ASCII 码区分的物理判定标志。
