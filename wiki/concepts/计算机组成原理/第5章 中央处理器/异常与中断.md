# 异常与中断

## 定义

异常是 CPU 执行指令过程中由内部事件引起的处理流程改变；中断通常由外部设备请求引起，使 CPU 暂停当前程序转去执行中断服务程序。

## 机制/流程

1. 中断源提出请求。
2. 中断判优，确定服务对象。
3. CPU 在满足条件时响应中断，保存断点。
4. 根据中断向量转入中断服务程序。
5. 服务结束后恢复现场并中断返回。

## 核心判断表

| 类型 | 来源 | 返回特点 |
|---|---|---|
| 故障 | 指令执行中检测到，可修复 | 通常返回原指令重新执行 |
| 自陷 | 程序有意触发 | 返回下一条指令或约定位置 |
| 终止 | 严重硬件故障 | 通常不返回 |
| 外中断 | 外设或外部事件 | 与具体指令不直接关联 |

## 考点题型

- 判断异常和中断的区别。
- 中断响应条件和中断隐指令内容。
- 多重中断与优先级。

## 易错点

- 中断隐指令通常由硬件完成，不是指令系统中的普通指令。
- 异常由 CPU 内部检测，中断需要外部中断请求信号。
