# 总线与I/O系统

## 总线（Bus）

### 什么是总线？
总线是一组能为多个部件分时共享的公共信息传送线路

### 总线的分类

#### 1. 按数据传送格式分类
- **并行总线**：同时传送多位数据
- **串行总线**：逐位传送数据

#### 2. 按时序控制方式分类
- **同步总线**：统一时钟控制
- **异步总线**：无统一时钟，用应答方式
- **半同步总线**：结合同步和异步

#### 3. 按功能分类
| 总线类型 | 功能 | 特点 |
|---------|------|------|
| **数据总线（DB）** | 传送数据信息 | 双向，位数与字长相同 |
| **地址总线（AB）** | 传送地址信息 | 单向，位数决定寻址范围 |
| **控制总线（CB）** | 传送控制信号 | 双向，实现总线仲裁 |

### 总线的特点

#### 共享性
- 多个部件分时共享总线
- 任何时刻只能有一个部件发送数据

#### 分时性
- 同一时刻只能有一个部件发送数据
- 但可以有多个部件接收数据

### 总线性能指标

#### 总线宽度（总线位数）
- 一次能同时传送的数据位数
- 数据总线宽度 = 字长

#### 总线带宽
$$总线带宽 = 总线宽度 \times 总线频率$$

单位：MB/s 或 GB/s

### 总线仲裁

#### 为什么需要总线仲裁？
- 多个部件同时申请使用总线
- 需要决定哪个部件获得总线使用权

#### 总线仲裁方式

##### 1. 集中式仲裁
- 总线控制器统一管理

**链式查询方式**
- 通过串行的菊花链传递总线授权信号
- 离总线控制器越近，优先级越高

**计数器定时查询方式**
- 总线控制器对所有设备进行计数查询
- 每次查询从某个计数值开始，按序号查询

**独立请求方式**
- 每个设备独立发送请求和接收授权信号
- 速度快，但连线多

##### 2. 分布式仲裁
- 无总线控制器，由各设备自己仲裁

### 总线操作和定时

#### 总线事务
- 请求总线 → 总线仲裁 → 总线分配 → 数据传送 → 总线释放

#### 总线定时方式

##### 同步定时
- 所有操作由统一时钟控制
- 优点：速度快，控制简单
- 缺点：时钟偏移问题

##### 异步定时
- 用应答信号（Request/Acknowledge）
- 优点：速度匹配灵活
- 缺点：控制复杂

## I/O系统

### I/O控制方式

#### 1. 程序查询方式（Programmed I/O）
- CPU主动查询I/O设备是否就绪
- CPU与I/O串行工作

```c
// 查询打印机是否就绪
while (!READY);  // 不断查询
OUT(Data, Printer);  // 输出数据
```

**特点：**
- 实现简单
- 效率低，CPU利用率低
- 适合简单、低速设备

#### 2. 程序中断方式（Interrupt-driven I/O）
- I/O设备就绪时向CPU发中断请求
- CPU暂停当前程序，转去执行中断服务程序

```c
// 中断服务程序
void interrupt_handler() {
    // 保存现场
    // 处理I/O
    // 恢复现场
    // 中断返回
}
```

**特点：**
- CPU与I/O并行工作
- 适合低速、中速设备
- 需要保存和恢复现场

#### 3. DMA方式（Direct Memory Access）
- I/O设备直接与主存交换数据
- 不需要CPU介入

**DMA的工作流程：**
1. CPU向DMA控制器发送DMA请求
2. DMA控制器向I/O设备发送命令
3. I/O设备准备数据
4. I/O设备向DMA控制器发送DMA响应
5. DMA控制器接管总线，直接与内存交换数据
6. 数据传输完成后，DMA控制器向CPU发送中断

**特点：**
- 数据传送速度快
- 适合高速、批量数据传送
- 减少了CPU的负担

### 三种I/O方式对比

| 对比维度 | 程序查询 | 程序中断 | DMA |
|---------|---------|---------|-----|
| **CPU介入** | 全程介入 | 中断服务 | 仅初始化和结束 |
| **并行性** | 串行 | 并行 | 并行 |
| **实时性** | 差 | 好 | 一般 |
| **适用设备** | 简单设备 | 低速/中速设备 | 高速设备 |
| **CPU效率** | 低 | 中 | 高 |

### I/O接口

#### I/O接口的功能
1. 设备选择
2. 数据缓冲（速度匹配）
3. 数据格式转换
4. 传送控制命令
5. 反映设备状态

#### I/O接口的类型
- **并行接口**：一次传送多位数据
- **串行接口**：逐位传送数据
- **同步接口**：同步传送
- **异步接口**：异步传送

### 中断系统

#### 中断的类型

##### 按中断来源分类
- **内中断**：来自CPU内部（异常、陷阱）
- **外中断**：来自外部设备（I/O中断、时钟中断）

##### 按中断响应方式分类
- **可屏蔽中断**：可以通过中断屏蔽字屏蔽
- **非屏蔽中断**：不能屏蔽，必须响应

#### 中断响应过程
1. 关中断
2. 保存断点（PC和PSW）
3. 识别中断源，获取中断向量
4. 转到相应的中断服务程序

#### 中断处理过程
1. 保存现场（通用寄存器、状态寄存器等）
2. 执行中断服务程序
3. 恢复现场
4. 开中断，返回断点

## 考试要点

### 高频考点
1. **总线的分类** — 数据总线、地址总线、控制总线
2. **总线仲裁** — 集中式仲裁的三种方式
3. **三种I/O方式** — 程序查询、程序中断、DMA的区别
4. **DMA的工作流程** — 何时需要CPU介入

### 常见陷阱
- 数据总线宽度 = 字长
- 地址总线宽度决定寻址范围（2^n 字节）
- DMA方式不需要CPU干预数据传送过程
