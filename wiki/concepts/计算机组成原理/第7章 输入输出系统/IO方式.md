# IO方式

## 定义

I/O 方式说明 CPU、主存、接口和外设之间如何配合完成数据输入输出。

## 核心判断表

| 方式 | CPU参与程度 | 特点 |
|---|---|---|
| 程序查询 | 最高 | CPU 反复查询状态，效率低 |
| 程序中断 | 中等 | 外设就绪后请求 CPU 服务 |
| DMA | 低 | DMA 控制器直接控制主存和外设成块传送 |
| 通道 | 更低 | 专门 I/O 处理器执行通道程序 |

## 考点题型

- 比较几种 I/O 方式效率和适用场景。
- 判断 CPU 在数据传送过程中的参与程度。
- 结合中断响应流程分析程序中断方式。

## 易错点

- 中断方式减少等待，但数据传送仍由 CPU 执行指令完成。
- DMA 适合高速批量数据，不适合每次少量且随机的低速事件。
