# 算法复杂度分析

## Definition

复杂度分析估计算法随问题规模增长所需的时间和额外空间，通常使用渐进记号。

## How It Works

- $O$：渐进上界。
- $\Omega$：渐进下界。
- $\Theta$：同阶精确界。
- 常见阶：$O(1)<O(\log n)<O(n)<O(n\log n)<O(n^2)<O(2^n)$。

## Exam Focus

- 单层/嵌套循环计数。
- 折半过程通常为 $O(\log n)$。
- 递归式的展开和主定理直觉。
- 最好、最坏、平均复杂度区别。

## 解答思路

复杂度题不要急着套答案，按下面顺序拆：

| 步骤 | 问题 | 操作 |
|---:|---|---|
| 1 | 问题规模是什么？ | 明确 $n$、$m$、顶点数 $V$、边数 $E$ 分别代表什么 |
| 2 | 基本操作是什么？ | 找最内层执行次数最高的语句，如比较、赋值、访问边 |
| 3 | 循环次数怎么变？ | 判断是线性递增、倍增、折半、平方嵌套还是三角形求和 |
| 4 | 多段代码如何合并？ | 顺序结构取最大阶，嵌套结构相乘，分支结构看最坏情况 |
| 5 | 是否有递归？ | 写出递归式 $T(n)$，再展开或用常见模板判断 |
| 6 | 空间是否额外申请？ | 只看辅助空间，输入本身不算额外空间 |

## 常见循环模板

| 代码结构 | 次数估计 | 复杂度 | 判断要点 |
|---|---:|---|---|
| `for i=1..n` | $n$ | $O(n)$ | 单层线性循环 |
| 双重循环 `i=1..n, j=1..n` | $n^2$ | $O(n^2)$ | 两层互相独立 |
| `for i=1..n, for j=1..i` | $\frac{n(n+1)}2$ | $O(n^2)$ | 三角形求和仍是平方级 |
| `for i=1; i<n; i*=2` | $\log_2 n$ | $O(\log n)$ | 循环变量倍增 |
| `while n>1: n/=2` | $\log_2 n$ | $O(\log n)$ | 折半过程 |
| `for i=1; i*i<=n; i++` | $\sqrt n$ | $O(\sqrt n)$ | 循环到平方根 |
| 两段顺序循环 $O(n)$ 与 $O(n^2)$ | 取最大项 | $O(n^2)$ | 顺序结构不是相加保留所有项 |

## 递归复杂度模板

| 递归式 | 常见来源 | 复杂度 |
|---|---|---|
| $T(n)=T(n-1)+O(1)$ | 每次规模减 1，每层常数工作 | $O(n)$ |
| $T(n)=T(n-1)+O(n)$ | 每次规模减 1，每层线性扫描 | $O(n^2)$ |
| $T(n)=T(n/2)+O(1)$ | 二分查找 | $O(\log n)$ |
| $T(n)=T(n/2)+O(n)$ | 每层线性处理，只递归一半 | $O(n)$ |
| $T(n)=2T(n/2)+O(1)$ | 二叉递归，每层节点数翻倍 | $O(n)$ |
| $T(n)=2T(n/2)+O(n)$ | 归并排序 | $O(n\log n)$ |

## 空间复杂度

| 场景 | 空间复杂度 | 说明 |
|---|---|---|
| 只使用若干变量 | $O(1)$ | 原地工作 |
| 新建长度为 $n$ 的数组 | $O(n)$ | 辅助数组计入空间 |
| 递归深度为 $h$ | $O(h)$ | 递归栈空间要计入 |
| 二叉树递归遍历 | $O(h)$ | 平衡树 $h=\log n$，最坏链状 $h=n$ |
| 图的邻接矩阵 | $O(V^2)$ | 与边数无关 |
| 图的邻接表 | $O(V+E)$ | 顶点表 + 边表 |

## 408 常考题型

| 题型 | 解题抓手 | 常见答案 |
|---|---|---|
| 给代码求时间复杂度 | 数最内层语句执行次数 | $O(1)$、$O(n)$、$O(n^2)$、$O(\log n)$ |
| 给递归函数求复杂度 | 写递归式再展开 | $O(n)$、$O(n\log n)$、$O(\log n)$ |
| 判断算法最好/最坏/平均 | 看输入是否影响循环提前结束 | 顺序查找最好 $O(1)$，最坏 $O(n)$ |
| 图算法复杂度 | 先看存储结构 | 邻接矩阵遍历 $O(V^2)$，邻接表遍历 $O(V+E)$ |
| 排序算法复杂度 | 记忆排序表并看特殊输入 | 快排最坏 $O(n^2)$，归并稳定 $O(n\log n)$ |

## Common Pitfalls

- 循环变量指数增长时不是 $O(n)$。
- 递归函数要同时考虑递归次数和每层代价。
- 顺序执行的两段代码取最高阶，不写成 $O(n+n^2)$。
- 嵌套循环不是永远 $O(n^2)$，要看内层是否与外层相关。
- 空间复杂度看额外空间，不把输入数组本身算进去。
- 图题必须区分邻接矩阵和邻接表。
