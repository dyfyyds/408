# 查找（Search）

## Definition

**查找**是在数据结构中找出满足某种条件的节点

**查找表（Search Table）**：由同一类型的数据元素（或记录）构成的集合

## 查找的基本概念

### 关键字（Key）
- 数据元素中某个数据项的值
- 可以唯一标识一个记录的关键字称为**主关键字**
- 可以标识若干个记录的关键字称为**次关键字**

### 查找成功
查找表中存在满足条件的记录

### 查找失败
查找表中不存在满足条件的记录

### 平均查找长度（ASL）
$$ASL = \sum_{i=1}^{n} P_i \cdot C_i$$

- $P_i$：查找第i个元素的概率
- $C_i$：找到第i个元素需要比较的次数

## 顺序查找（Sequential Search）

### 算法思想
从表的一端开始，依次扫描每个元素，直到找到目标或扫描完所有元素

### 算法实现
```c
int Search_Seq(SqList L, ElemType key) {
    for (int i = 0; i < L.length; i++) {
        if (L.data[i] == key)
            return i + 1; // 返回位置（1-based）
    }
    return 0; // 查找失败
}
```

### 性能分析
- **时间复杂度**：O(n)
- **空间复杂度**：O(1)
- **平均查找长度**：
  - 等概率：$(n+1)/2$
  - 最坏：n

### 顺序查找的特点
- 顺序存储或链式存储都可以
- 对表的结构无要求
- 算法简单，但效率低

## 二分查找（Binary Search）

### 算法思想
- 前提：**有序表**
- 每次将待查找区间缩小一半
- 时间复杂度：O(log n)

### 算法实现
```c
int Search_Bin(SqList L, ElemType key) {
    int low = 0, high = L.length - 1;
    int mid;
    
    while (low <= high) {
        mid = (low + high) / 2;
        
        if (L.data[mid] == key)
            return mid + 1;
        else if (L.data[mid] > key)
            high = mid - 1;
        else
            low = mid + 1;
    }
    
    return 0; // 查找失败
}
```

### 性能分析
- **时间复杂度**：O(log n)
- **空间复杂度**：O(1)
- **平均查找长度**：≈ log₂(n+1) - 1

### 二分查找的判定树
- 判定树：一棵二叉搜索树
- 树的高度：⌊log₂n⌋+1
- 比较次数不超过树的高度

### 二分查找的特点
- 必须是**顺序存储的有序表**
- 查找效率高，但插入删除不便
- 不适用于频繁变化的表

## 分块查找（Block Search）

### 算法思想
- 将查找表分成若干块（块内无序，块间有序）
- 先确定元素在哪一块（二分查找块）
- 再在块内顺序查找

### 性能分析
- **时间复杂度**：O(√n)
- 平均查找长度：√n + 1

## 散列查找（Hash Search）

### 基本概念

#### 散列表（Hash Table）
- 散列函数将关键字映射到存储位置
- 理想情况：直接寻址，时间复杂度O(1)

#### 散列函数
将关键字映射到散列表地址的函数

$$H(key) = key \mod m$$

### 散列函数的设计方法

#### 1. 直接定址法
$$H(key) = a \cdot key + b$$

#### 2. 除留余数法
$$H(key) = key \mod p$$

- p通常取质数或不超过n的最小质数
- 最常用的方法

#### 3. 数字分析法
- 抽取关键字的某几位作为地址

#### 4. 平方取中法
- 取关键字平方的中间几位作为地址

### 冲突处理方法

#### 1. 开放定址法
发生冲突时，寻找下一个空的散列地址

$$H_i = (H(key) + d_i) \mod m$$

其中$d_i$为增量序列

##### 增量序列的三种取法
1. **线性探测法**：$d_i = 0, 1, 2, ..., m-1$
2. **平方探测法**：$d_i = 0^2, 1^2, -1^2, 2^2, -2^2, ...$
3. **双散列法**：$d_i = i \cdot H_2(key)$

#### 2. 链地址法
- 将所有同义词存储在一个链表中
- 每个散列地址存储一个单链表的头指针

```c
typedef struct HashNode {
    int key;
    struct HashNode *next;
} HashNode;

typedef struct {
    HashNode *table[HashSize];
} HashTable;
```

#### 3. 再散列法
- 准备多个散列函数
- 发生冲突时，换一个散列函数

### 散列查找的性能分析

#### 装填因子（Load Factor）
$$\alpha = \frac{查找表中元素个数}{散列表的长度}$$

- $\alpha$越小，冲突可能性越小
- 一般取$\alpha \leq 0.75$

#### 平均查找长度
- 与**装填因子**有关，与元素个数无关
- 查找成功的ASL：
  - 线性探测：$(1 + 1/(1-\alpha)) / 2$
  - 链地址法：$1 + \alpha/2$

### 散列查找的特点
- **优点**：平均查找长度短，O(1)
- **缺点**：
  - 需要额外空间
  - 不支持范围查找
  - 存在冲突问题

## 各种查找方法的对比

| 查找方法 | 存储结构 | 平均查找长度 | 适用场景 |
|---------|---------|-------------|---------|
| **顺序查找** | 顺序/链式 | O(n) | 无序表 |
| **二分查找** | 有序顺序表 | O(log n) | 静态查找表 |
| **分块查找** | 顺序存储 | O(√n) | 中等规模 |
| **散列查找** | 散列表 | O(1) | 快速查找 |

## 考试要点

### 高频考点
1. **二分查找的判定树** — 比较次数和判定树高度的关系
2. **散列函数的构造** — 除留余数法、直接定址法
3. **冲突处理方法** — 开放定址法（线性探测、平方探测）、链地址法
4. **装填因子** — 对查找性能的影响

### 常见陷阱
- 二分查找必须是**有序表**
- 散列查找的ASL与元素个数无关，与装填因子有关
- 链地址法不会产生堆积现象
