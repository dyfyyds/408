# 图（Graph）

## Definition

**图（Graph）**是由非空的顶点集合和一个描述顶点之间关系的边集合组成的数据结构。

$$G = (V, E)$$

- $V$：顶点的非空有限集合
- $E$：边的有限集合

## 图的基本概念

### 无向图
- 边没有方向
- 边可以用无序对$(v, w)$表示
- 对称：$(v, w) = (w, v)$

### 有向图
- 边有方向
- 边用有序对$<v, w>$表示
- 不对称：$<v, w> \neq <w, v>$

### 网（带权图）
- 边带有权值的图

### 邻接矩阵
用矩阵表示顶点之间的关系

```c
#define MaxVertexNum 100
typedef struct {
    int Vex[MaxVertexNum];             // 顶点表
    int Edge[MaxVertexNum][MaxVertexNum]; // 邻接矩阵（边表）
    int vexnum, arcnum;                // 顶点数和边数
} MGraph;
```

### 邻接表
对每个顶点建立一个单链表

```c
typedef struct ArcNode {
    int adjvex;              // 邻接点的位置
    struct ArcNode *nextarc; // 指向下一个邻接点的指针
} ArcNode;

typedef struct VNode {
    int data;          // 顶点信息
    ArcNode *firstarc; // 指向第一个邻接点的指针
} VNode, AdjList[MaxVertexNum];

typedef struct {
    AdjList vertices;
    int vexnum, arcnum;
} ALGraph;
```

| 对比维度 | 邻接矩阵 | 邻接表 |
|---------|---------|--------|
| **空间复杂度** | O(n²) | O(n+m) |
| **判断边是否存在** | O(1) | O(degree) |
| **查找所有邻接点** | O(n) | O(degree) |

## 图的遍历

### 广度优先遍历（BFS）
类似**层序遍历**，使用**队列**

```c
bool visited[MaxVertexNum]; // 访问标记数组

void BFSTraverse(Graph G) {
    for (int i = 0; i < G.vexnum; i++)
        visited[i] = false;
    
    LinkQueue Q;
    InitQueue(Q);
    
    for (int i = 0; i < G.vexnum; i++) {
        if (!visited[i])
            BFS(G, i);
    }
}

void BFS(Graph G, int v) {
    visit(v);
    visited[v] = true;
    EnQueue(Q, v);
    
    while (!QueueEmpty(Q)) {
        DeQueue(Q, v);
        
        for (int w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
            if (!visited[w]) {
                visit(w);
                visited[w] = true;
                EnQueue(Q, w);
            }
        }
    }
}
```

**BFS算法性能分析**
- 时间复杂度：O(n+m)（邻接表）或O(n²)（邻接矩阵）
- 空间复杂度：O(n)（队列空间）

### 深度优先遍历（DFS）
类似**前序遍历**，使用**递归或栈**

```c
void DFSTraverse(Graph G) {
    for (int i = 0; i < G.vexnum; i++)
        visited[i] = false;
    
    for (int i = 0; i < G.vexnum; i++) {
        if (!visited[i])
            DFS(G, i);
    }
}

void DFS(Graph G, int v) {
    visit(v);
    visited[v] = true;
    
    for (int w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
        if (!visited[w])
            DFS(G, w);
    }
}
```

**DFS算法性能分析**
- 时间复杂度：O(n+m)
- 空间复杂度：O(n)（递归栈空间）

### 最小生成树（Minimum Spanning Tree，MST）

#### 定义
n个顶点的连通图的生成树是包含图中全部顶点的极小连通子图

#### 最小生成树的性质
1. 最小生成树必须包含全部顶点
2. 最小生成树的边数 = 顶点数 - 1
3. 最小生成树中不存在回路

#### 求最小生成树的方法

##### 1. Prim算法（普里姆算法）
- 思想：从某个顶点开始，逐步扩展，选择最小权值的边
- 时间复杂度：O(n²)
- 适合：**稠密图**

```c
void MiniSpanTree_Prim(MGraph G) {
    int adjvex[MaxVertexNum]; // 存储顶点下标
    int lowcost[MaxVertexNum]; // 存储最小权值
    
    // 从顶点0开始
    lowcost[0] = 0; // 起始顶点加入到U集合
    adjvex[0] = 0;
    
    for (int i = 1; i < G.vexnum; i++) {
        lowcost[i] = G.arc[0][i]; // 初始化
        adjvex[i] = 0;
    }
    
    for (int i = 1; i < G.vexnum; i++) {
        int min = INF;
        int k = 0;
        
        for (int j = 0; j < G.vexnum; j++) {
            if (lowcost[j] != 0 && lowcost[j] < min) {
                min = lowcost[j];
                k = j;
            }
        }
        
        printf("(%d,%d) ", adjvex[k], k);
        lowcost[k] = 0; // 将k加入U集合
        
        // 更新lowcost和adjvex
        for (int j = 0; j < G.vexnum; j++) {
            if (lowcost[j] != 0 && G.arc[k][j] < lowcost[j]) {
                lowcost[j] = G.arc[k][j];
                adjvex[j] = k;
            }
        }
    }
}
```

##### 2. Kruskal算法（克鲁斯卡尔算法）
- 思想：按边权值从小到大加入，避免形成回路
- 时间复杂度：O(eloge)（e为边数）
- 适合：**稀疏图**

```c
typedef struct {
    int begin, end;
    int weight;
} Edge;

// 快速排序边
void sort(Edge edges[], int e) {
    // 按权值从小到大排序
}

// 并查集操作
int find(int *parent, int f) {
    while (parent[f] != f)
        f = parent[f];
    return f;
}

void MiniSpanTree_Kruskal(MGraph G, Edge edges[], int e) {
    int parent[MaxVertexNum];
    
    for (int i = 0; i < G.vexnum; i++)
        parent[i] = i; // 初始化并查集
    
    sort(edges, e); // 按权值排序
    
    for (int i = 0; i < e; i++) {
        int n = find(parent, edges[i].begin);
        int m = find(parent, edges[i].end);
        
        if (n != m) { // 不会形成回路
            printf("(%d,%d) ", edges[i].begin, edges[i].end);
            parent[m] = n;
        }
    }
}
```

| 对比维度 | Prim算法 | Kruskal算法 |
|---------|---------|-------------|
| **时间复杂度** | O(n²) | O(eloge) |
| **适合图类型** | 稠密图 | 稀疏图 |
| **核心思想** | 顶点扩展 | 边扩展 |

### 最短路径

#### 1. Dijkstra算法（迪杰斯特拉算法）
- 求**单源最短路径**（一个顶点到其他所有顶点的最短路径）
- **不能处理带负权值的图**
- 时间复杂度：O(n²)

```c
void ShortestPath_Dijkstra(MGraph G, int v0) {
    int final[MaxVertexNum]; // final[i]=1表示已确定最短路径
    int dist[MaxVertexNum];  // 最短路径长度
    int path[MaxVertexNum];  // 路径上的前驱
    
    for (int i = 0; i < G.vexnum; i++) {
        final[i] = 0;
        dist[i] = G.arc[v0][i];
        path[i] = v0;
    }
    
    final[v0] = 1;
    dist[v0] = 0;
    
    for (int i = 1; i < G.vexnum; i++) {
        int min = INF;
        int k = -1;
        
        for (int j = 0; j < G.vexnum; j++) {
            if (!final[j] && dist[j] < min) {
                min = dist[j];
                k = j;
            }
        }
        
        final[k] = 1;
        
        for (int j = 0; j < G.vexnum; j++) {
            if (!final[j] && dist[k] + G.arc[k][j] < dist[j]) {
                dist[j] = dist[k] + G.arc[k][j];
                path[j] = k;
            }
        }
    }
}
```

#### 2. Floyd算法（弗洛伊德算法）
- 求**每对顶点之间的最短路径**
- 时间复杂度：O(n³)
- 可以处理带负权值的图（但不能有负权回路）

```c
void ShortestPath_Floyd(MGraph G, int Path[][MaxVertexNum], int Dist[][MaxVertexNum]) {
    // 初始化
    for (int i = 0; i < G.vexnum; i++) {
        for (int j = 0; j < G.vexnum; j++) {
            Dist[i][j] = G.arc[i][j];
            Path[i][j] = -1;
        }
    }
    
    // k为允许加入的中间顶点
    for (int k = 0; k < G.vexnum; k++) {
        for (int i = 0; i < G.vexnum; i++) {
            for (int j = 0; j < G.vexnum; j++) {
                if (Dist[i][j] > Dist[i][k] + Dist[k][j]) {
                    Dist[i][j] = Dist[i][k] + Dist[k][j];
                    Path[i][j] = k;
                }
            }
        }
    }
}
```

| 对比维度 | Dijkstra算法 | Floyd算法 |
|---------|-------------|-----------|
| **单源/多源** | 单源 | 多源（任意两点） |
| **负权值** | ❌ 不能处理 | ✅ 可以处理 |
| **时间复杂度** | O(n²) | O(n³) |

### 拓扑排序（AOV网）

#### 定义
有向无环图（DAG）的顶点的一种线性序列

#### 拓扑排序的作用
检测有向图中是否存在环

#### 拓扑排序的步骤（Kahn算法）
1. 选择一个入度为0的顶点并输出
2. 删除该顶点及其发出的所有边
3. 重复步骤1-2，直到所有顶点被输出

```c
bool TopologicalSort(Graph G) {
    int indegree[MaxVertexNum]; // 入度数组
    FindInDegree(G, indegree);  // 计算每个顶点的入度
    
    LinkStack S; // 存储入度为0的顶点
    InitStack(S);
    
    for (int i = 0; i < G.vexnum; i++) {
        if (indegree[i] == 0)
            Push(S, i);
    }
    
    int count = 0;
    
    while (!StackEmpty(S)) {
        int v;
        Pop(S, v);
        printf("%d ", v);
        count++;
        
        for (int w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
            indegree[w]--;
            if (indegree[w] == 0)
                Push(S, w);
        }
    }
    
    if (count < G.vexnum)
        return false; // 存在环
    else
        return true;  // 拓扑排序成功
}
```

### 关键路径（AOE网）

#### 定义
AOE网（Activity On Edge Network）中，从源点到汇点的**最长路径**

#### 关键路径的性质
- 关键路径上的活动称为**关键活动**
- 关键活动延期会导致整个工程延期
- 可以通过加速关键活动来缩短工期

## 考试要点

### 高频考点
1. **图的存储结构** — 邻接矩阵 vs 邻接表
2. **BFS和DFS** — 遍历顺序和过程
3. **最小生成树** — Prim和Kruskal算法的适用场景
4. **最短路径** — Dijkstra和Floyd算法的区别
5. **拓扑排序** — 判断有向图是否存在环

### 常见陷阱
- Dijkstra不能处理负权值
- Floyd算法的时间复杂度是O(n³)
- 拓扑排序不唯一
