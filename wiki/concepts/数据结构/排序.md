# 排序（Sort）

## Definition

**排序**是将一组杂乱无章的数据元素序列，按某个关键字递增或递减排列成有序序列的过程。

## 排序的分类

### 内部排序
数据全部在内存中进行的排序

### 外部排序
数据量很大，内存放不下，需要借助外存

## 排序算法的稳定性

### 稳定排序
如果排序前后，相同关键字的元素相对位置不变

### 不稳定排序
相同关键字的元素相对位置可能改变

| 排序算法 | 稳定性 |
|---------|--------|
| 冒泡排序 | ✅ 稳定 |
| 选择排序 | ❌ 不稳定 |
| 插入排序 | ✅ 稳定 |
| 归并排序 | ✅ 稳定 |
| 快速排序 | ❌ 不稳定 |
| 堆排序 | ❌ 不稳定 |

## 各种排序算法详解

### 1. 冒泡排序（Bubble Sort）

#### 算法思想
- 两两比较相邻元素，如果反序则交换
- 每一趟都将一个最大/最小元素放到正确位置

```c
void BubbleSort(ElemType A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false; // 优化：标记是否发生交换
        
        for (int j = n - 1; j > i; j--) {
            if (A[j - 1] > A[j]) {
                swap(A[j - 1], A[j]);
                flag = true;
            }
        }
        
        if (!flag) break; // 未发生交换，说明已有序
    }
}
```

#### 性能分析
- **时间复杂度**：
  - 最好：O(n)（已有序）
  - 平均：O(n²)
  - 最坏：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：稳定

### 2. 选择排序（Selection Sort）

#### 算法思想
- 每趟从待排序序列中选择最小（或最大）的元素
- 放到已排序序列的末尾

```c
void SelectSort(ElemType A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min = i; // 记录最小元素位置
        
        for (int j = i + 1; j < n; j++) {
            if (A[j] < A[min])
                min = j;
        }
        
        if (min != i)
            swap(A[i], A[min]);
    }
}
```

#### 性能分析
- **时间复杂度**：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：不稳定

### 3. 插入排序（Insertion Sort）

#### 算法思想
- 将未排序序列中的元素，逐个插入到已排序序列的正确位置

```c
void InsertSort(ElemType A[], int n) {
    for (int i = 1; i < n; i++) {
        ElemType temp = A[i];
        int j = i - 1;
        
        while (j >= 0 && A[j] > temp) {
            A[j + 1] = A[j];
            j--;
        }
        
        A[j + 1] = temp;
    }
}
```

#### 性能分析
- **时间复杂度**：
  - 最好：O(n)（已有序）
  - 平均：O(n²)
  - 最坏：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：稳定

### 4. 希尔排序（Shell Sort）

#### 算法思想
- 插入排序的改进
- 先将序列分成若干子序列，分别进行插入排序
- 逐步缩小增量，最后对整个序列进行一次插入排序

```c
void ShellSort(ElemType A[], int n) {
    for (int dk = n / 2; dk >= 1; dk = dk / 2) {
        for (int i = dk; i < n; i++) {
            if (A[i] < A[i - dk]) {
                ElemType temp = A[i];
                int j = i - dk;
                
                while (j >= 0 && A[j] > temp) {
                    A[j + dk] = A[j];
                    j -= dk;
                }
                
                A[j + dk] = temp;
            }
        }
    }
}
```

#### 性能分析
- **时间复杂度**：
  - 最好：O(n)
  - 平均：O(n^1.5) 或 O(n log²n)
  - 最坏：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：不稳定

### 5. 归并排序（Merge Sort）

#### 算法思想
- 分治思想
- 将序列分成两个子序列，分别排序
- 将两个有序子序列合并为一个有序序列

```c
void Merge(ElemType A[], int low, int mid, int high) {
    ElemType B[high - low + 1];
    int i = low, j = mid + 1, k = 0;
    
    while (i <= mid && j <= high) {
        if (A[i] <= A[j])
            B[k++] = A[i++];
        else
            B[k++] = A[j++];
    }
    
    while (i <= mid) B[k++] = A[i++];
    while (j <= high) B[k++] = A[j++];
    
    for (int i = low, k = 0; i <= high; i++)
        A[i] = B[k++];
}

void MergeSort(ElemType A[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;
        MergeSort(A, low, mid);
        MergeSort(A, mid + 1, high);
        Merge(A, low, mid, high);
    }
}
```

#### 性能分析
- **时间复杂度**：
  - 最好：O(n log n)
  - 平均：O(n log n)
  - 最坏：O(n log n)
- **空间复杂度**：O(n)（需要辅助数组）
- **稳定性**：稳定

### 6. 快速排序（Quick Sort）

#### 算法思想
- 分治思想
- 选择一个基准元素（pivot）
- 将序列分为两部分：小于基准的和大于基准的
- 递归排序两部分

```c
int Partition(ElemType A[], int low, int high) {
    ElemType pivot = A[low]; // 基准元素
    
    while (low < high) {
        while (low < high && A[high] >= pivot)
            high--;
        A[low] = A[high];
        
        while (low < high && A[low] <= pivot)
            low++;
        A[high] = A[low];
    }
    
    A[low] = pivot;
    return low;
}

void QuickSort(ElemType A[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(A, low, high);
        QuickSort(A, low, pivotpos - 1);
        QuickSort(A, pivotpos + 1, high);
    }
}
```

#### 性能分析
- **时间复杂度**：
  - 最好：O(n log n)
  - 平均：O(n log n)
  - 最坏：O(n²)（已有序或逆序）
- **空间复杂度**：
  - 最好：O(log n)（递归栈深度）
  - 最坏：O(n)
- **稳定性**：不稳定

#### 快速排序的优化
1. **三数取中法**：选择三个元素的中位数作为基准
2. **小数组优化**：当n较小时（如n<20），使用插入排序
3. **三路快速排序**：处理大量重复元素

### 7. 堆排序（Heap Sort）

#### 算法思想
- 利用堆这种数据结构进行排序
- 将序列构建成大顶堆（或小顶堆）
- 反复取出堆顶元素放到已排序序列的末尾

```c
// 堆调整（大顶堆）
void HeapAdjust(ElemType A[], int k, int len) {
    A[0] = A[k]; // 暂存根节点
    
    for (int i = 2 * k; i <= len; i *= 2) {
        if (i < len && A[i] < A[i + 1])
            i++; // 选择较大的子节点
        
        if (A[0] >= A[i])
            break;
        
        A[k] = A[i];
        k = i;
    }
    
    A[k] = A[0];
}

// 建立大顶堆
void BuildMaxHeap(ElemType A[], int len) {
    for (int i = len / 2; i > 0; i--)
        HeapAdjust(A, i, len);
}

// 堆排序
void HeapSort(ElemType A[], int len) {
    BuildMaxHeap(A, len);
    
    for (int i = len; i > 1; i--) {
        swap(A[1], A[i]);      // 堆顶元素与最后一个元素交换
        HeapAdjust(A, 1, i-1); // 调整堆
    }
}
```

#### 性能分析
- **时间复杂度**：
  - 最好：O(n log n)
  - 平均：O(n log n)
  - 最坏：O(n log n)
- **空间复杂度**：O(1)
- **稳定性**：不稳定

#### 堆的性质
- 大顶堆：父节点 ≥ 子节点
- 小顶堆：父节点 ≤ 子节点
- 完全二叉树，可以用数组存储

## 排序算法的对比

| 排序算法 | 最好时间 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|---------|---------|---------|---------|------|--------|
| **冒泡排序** | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| **选择排序** | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| **插入排序** | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| **希尔排序** | O(n) | O(n^1.5) | O(n²) | O(1) | ❌ |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ |

## 选择排序算法的依据

1. **数据量n较小**
   - 选择插入排序或冒泡排序（O(n²)可以接受）

2. **数据量n较大**
   - 选择快速排序（平均性能最好）

3. **要求稳定性**
   - 选择归并排序

4. **数据量大且存储受限**
   - 选择堆排序（空间O(1)）

5. **数据大部分有序**
   - 选择插入排序或冒泡排序（可以优化到O(n)）

## 考试要点

### 高频考点
1. **排序算法的比较** — 时间空间复杂度、稳定性
2. **快速排序的划分过程** — 基准元素的选择
3. **堆排序的调整过程** — 堆的建立和调整
4. **归并排序的合并过程** — 两路归并

### 常见陷阱
- 时间复杂度不是排序算法的唯一判断标准
- 快速排序的最坏情况（O(n²)）
- 堆排序的空间复杂度是O(1)（原地排序）
