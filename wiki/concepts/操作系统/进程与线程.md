# 进程与线程

## Definition

进程是程序在某个数据集合上的一次运行活动，是系统进行资源分配和调度的基本单位。线程是进程内的执行流，是 CPU 调度的基本单位。

## How It Works

- PCB：进程存在的唯一标志，保存进程标识、状态、程序计数器、寄存器、调度信息、资源清单等。
- 进程状态：创建、就绪、运行、阻塞、终止。
- 状态转换：就绪获得 CPU 变运行；运行等待事件变阻塞；阻塞事件完成变就绪。
- 线程共享进程资源，但拥有自己的程序计数器、寄存器和栈。

## Exam Focus

- 进程与程序、线程的区别。
- 五状态模型和状态转换原因。
- 进程切换与线程切换开销。
- 用户级线程和内核级线程的优缺点。

## Common Pitfalls

- 阻塞态不能直接变运行态，必须先回到就绪态。
- 时间片用完是运行态到就绪态，不是阻塞态。
- 进程是资源分配单位，线程是调度执行单位。
- 用户级线程切换快，但一个线程阻塞可能导致整个进程阻塞。
