# 分段存储管理

> 按程序的逻辑模块分段，段内地址连续，段间地址不连续。详见 [[内存管理]]。

---

## 基本原理

- 程序按逻辑模块划分为若干段（代码段、数据段、堆栈段等）
- 每个段有自己的段名和段长
- 段内地址连续，段间地址不连续

## 地址转换

```
逻辑地址 = 段号 + 段内偏移
  ↓
段表查询 → 段基址 + 段长
  ↓
检查：段内偏移 < 段长？
  ↓
物理地址 = 段基址 + 段内偏移
```

## 段表

| 字段 | 说明 |
|------|------|
| 段基址 | 段在内存中的起始地址 |
| 段长 | 段的长度 |
| 存在位 | 该段是否在内存中 |
| 保护位 | 读写权限 |

## 分页 vs 分段

| 对比维度 | 分页 | 分段 |
|----------|------|------|
| 地址空间 | 一维 | 二维 |
| 信息保护 | 不方便 | 方便（按段保护） |
| 共享 | 不方便 | 方便（按段共享） |
| 碎片 | 内部碎片 | 外部碎片 |
| 对用户透明 | 是 | 否（用户可见） |

## 段页式存储管理

- 先分段，再分页
- 每个段再划分为若干页
- 逻辑地址 = 段号 + 页号 + 页内偏移
