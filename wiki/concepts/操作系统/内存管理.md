# 内存管理

## 内存管理的基本概念

### 内存管理的功能
1. **内存空间的分配与回收**
2. **地址转换**（逻辑地址 → 物理地址）
3. **内存空间的共享与保护**
4. **内存扩充**（虚拟存储器）

### 程序执行过程

```
源程序 → 编译 → 链接 → 装入 → 执行
```

#### 链接的三种方式
1. **静态链接**：程序运行前完成链接
2. **装入时动态链接**：程序装入时链接
3. **运行时动态链接**：程序运行时链接

#### 装入的三种方式

##### 1. 绝对装入
- 编译时就知道物理地址
- 装入时直接装入绝对地址

##### 2. 静态重定位（可重定位装入）
- 装入时一次性完成地址转换
- 程序运行时不能移动

##### 3. 动态重定位（动态运行时装入）
- 运行时进行地址转换
- 需要**重定位寄存器**
- 程序可以移动（便于虚拟存储器的实现）

### 逻辑地址与物理地址

#### 逻辑地址（相对地址、虚地址）
- 程序中的地址
- 由段号和段内偏移组成

#### 物理地址（绝对地址、实地址）
- 内存单元的实际地址
- CPU通过地址总线发出的地址

#### 地址转换
```
逻辑地址 → MMU（内存管理单元） → 物理地址
```

## 连续分配方式

### 单一连续分配
- 内存分为系统区和用户区
- 只能有一道用户程序
- 适合单道批处理系统

### 固定分区分配
- 内存分为若干个固定大小的分区
- 每个分区装入一个作业
- **内部碎片**：分区内部未被利用的空间

### 动态分区分配

#### 分配算法
1. **首次适应算法（First Fit）**
   - 从低地址开始查找，找到第一个满足大小的分区
   - 优点：保留高地址的大分区
   - 缺点：低地址碎片多

2. **最佳适应算法（Best Fit）**
   - 找到最小的满足大小的分区
   - 优点：尽量不浪费空间
   - 缺点：碎片多

3. **最坏适应算法（Worst Fit）**
   - 找到最大的分区
   - 优点：避免产生太多小碎片
   - 缺点：大作业可能无法装入

#### 分区分配操作
- **分配**：将分区分成两个，一个已分配，一个空闲
- **回收**：
  - 上邻接：合并成一个更大分区
  - 下邻接：合并成一个更大分区
  - 上下都邻接：合并成一个更大分区

#### 外部碎片
- 动态分区分配产生的外部碎片可以通过**紧凑（Compaction）**解决
- 紧凑：移动内存中的数据，使分散的空闲区合并

## 非连续分配方式

### 基本分页存储管理

#### 基本原理
- 将内存和进程空间划分为**大小相同**的块（页框/页帧）
- 进程空间划分的块称为**页**
- 通过**页表**实现地址转换

#### 页面大小
- 通常为2的幂次（如4KB、8KB、16KB）
- 太小：页表太长
- 太大：内存碎片增加

#### 地址转换
```
逻辑地址 = 页号 + 页内偏移
  ↓
页表查询 → 页框号 + 页内偏移
  ↓
物理地址 = 页框号 × 页面大小 + 页内偏移
```

#### 页表
| 页号 | 页框号 | 存在位 | 访问位 | 修改位 | 保护位 |
|------|--------|--------|--------|--------|--------|

- **存在位**：该页是否在内存中
- **访问位**：该页是否被访问过
- **修改位**：该页是否被修改过
- **保护位**：读写权限

#### 两级页表
- 解决单级页表占用空间大的问题

```
页目录 → 页表 → 物理页框
```

#### 快表（TLB，Translation Lookaside Buffer）
- 页表的高速缓存
- 存放最近访问的页表项
- 命中率可达90%以上

### 基本分段存储管理

#### 基本原理
- 按程序的逻辑模块分段
- 每个段有自己的段名和段长
- 段内地址连续，段间地址不连续

#### 地址转换
```
逻辑地址 = 段号 + 段内偏移
  ↓
段表查询 → 段基址 + 段长
  ↓
检查：段内偏移 < 段长？
  ↓
物理地址 = 段基址 + 段内偏移
```

### 段页式存储管理

#### 基本原理
- 先分段，再分页
- 每个段再划分为若干页
- 结合了段和页的优点

#### 地址转换
```
逻辑地址 = 段号 + 页号 + 页内偏移
  ↓
段表 → 页表 → 物理页框 + 页内偏移
  ↓
物理地址
```

## 虚拟存储器

### 什么是虚拟存储器？
- 将主存和辅存统一编址，构成一个大的存储空间
- 程序可以使用比物理主存更大的地址空间

### 虚拟存储器的实现方式

#### 1. 请求分页
- 页表增加存在位、访问位、修改位
- 缺页中断处理

#### 2. 请求分段
- 段表增加存在位、增长位等

### 缺页中断处理

```
访问页面 → 检查存在位
  ├── 存在 → 地址转换，访问内存
  └── 不存在 → 触发缺页中断
        ↓
    分配页框
        ↓
    从辅存调入页面
        ↓
    修改页表
        ↓
    重新执行指令
```

### 页面置换算法

#### 1. 最佳置换算法（OPT）
- 置换最长时间不再被访问的页面
- 理论最优算法，无法实现（需要预知未来）

#### 2. 先进先出算法（FIFO）
- 置换最早进入内存的页面
- **Belady异常**：分配的页框越多，缺页率反而越高

#### 3. 最近最少使用算法（LRU）
- 置换最长时间未被访问的页面
- 需要硬件支持（计数器或栈）

#### 4. 时钟置换算法（Clock）
- LRU的近似算法
- 用访问位和修改位判断

#### 5. 改进型Clock算法
- 综合考虑访问位和修改位
- 优先淘汰：(0, 0) → (0, 1) → (1, 0) → (1, 1)

## 磁盘调度

### 磁盘的结构

#### 盘片
- 磁盘由多个盘片组成
- 每个盘片有两个盘面

#### 磁道
- 盘面上的同心圆
- 编号：0, 1, 2, ...

#### 扇区
- 磁道划分为多个扇区
- 每个扇区512字节或4KB

#### 柱面
- 所有盘面上的同一磁道组成的圆柱面

### 磁盘访问时间

$$磁盘访问时间 = 寻道时间 + 旋转延迟时间 + 传输时间$$

#### 1. 寻道时间
- 磁头移动到目标磁道的时间

#### 2. 旋转延迟时间（延迟时间）
- 磁头到达目标磁道后，等待目标扇区到达磁头下的时间
- 平均旋转延迟 = 半圈时间 = (60 / 转速) / 2

#### 3. 传输时间
- 数据从一个扇区传送到内存的时间

### 磁盘调度算法

#### 1. 先来先服务（FCFS）
- 按请求到达顺序处理
- 公平但性能较差

#### 2. 最短寻道时间优先（SSTF）
- 优先处理距当前磁头最近的请求
- 可能出现饥饿现象

#### 3. 扫描算法（SCAN，电梯算法）
- 磁头从一端移动到另一端，沿途响应请求
- 到达一端后反向移动

#### 4. 循环扫描算法（C-SCAN）
- 磁头只沿一个方向扫描
- 到达一端后直接返回到起点

#### 5. LOOK和C-LOOK算法
- SCAN和C-SCAN的改进
- 磁头移动到最远请求处就反向

## 考试要点

### 高频考点
1. **逻辑地址 vs 物理地址** — 地址转换过程
2. **分页与分段** — 基本原理、优缺点对比
3. **页面置换算法** — FIFO、LRU、Clock的特点
4. **磁盘调度算法** — FCFS、SSTF、SCAN、C-SCAN

### 常见陷阱
- 分页有内部碎片，分段有外部碎片
- FIFO算法可能出现Belady异常
- 磁盘访问时间的三部分构成

### 真题示例
- 2012年408真题：逻辑地址到物理地址的转换
- 2015年408真题：页面置换算法的缺页率计算
- 2018年408真题：磁盘调度算法的比较
