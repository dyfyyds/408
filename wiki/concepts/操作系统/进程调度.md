# 进程调度（Process Scheduling）

## 调度的基本概念

### 什么是进程调度？
按照一定的策略，从就绪队列中选择一个进程，将CPU分配给它

### 调度的层次

#### 1. 高级调度（作业调度）
- 从外存将作业调入内存
- 发生频率最低

#### 2. 中级调度（内存调度）
- 将内存中的进程换出到外存（挂起）
- 内存紧张时进行

#### 3. 低级调度（进程调度）
- 从就绪队列中选择进程分配CPU
- 发生频率最高
- **也称为**：进程调度

## 进程调度的时机

### 需要进行调度的情况

1. **进程主动放弃CPU**
   - 正常结束
   - 执行阻塞操作（I/O、信号量）
   - 主动让出CPU

2. **进程被动放弃CPU**
   - 时间片用完
   - 被更高优先级进程抢占

### 不能调度的情况

1. **处理中断时**
   - 中断处理程序执行期间

2. **在操作系统内核的临界区**
   - 内核临界区禁止进程调度

3. **原子操作执行期间**
   - 执行原语期间

## 调度算法的评价指标

### 1. CPU利用率
$$CPU利用率 = \frac{CPU忙碌时间}{总时间}$$

### 2. 系统吞吐量
$$系统吞吐量 = \frac{完成作业的数量}{总时间}$$

### 3. 周转时间
$$周转时间 = 作业完成时间 - 作业提交时间$$

#### 带权周转时间
$$带权周转时间 = \frac{周转时间}{服务时间}$$

### 4. 等待时间
$$等待时间 = 周转时间 - 服务时间$$

### 5. 响应时间
$$响应时间 = 首次响应时间 - 提交时间$$

## 各种调度算法

### 1. 先来先服务算法（FCFS，First Come First Served）

#### 算法思想
按照作业/进程到达的先后顺序进行调度

#### 实现方式
- **先来先服务**：非抢占式
- 先到的先服务，后来的排队

#### 优缺点
- **优点**：实现简单，公平
- **缺点**：
  - 平均等待时间长
  - 不利于短作业
  - 不利于I/O型作业

#### 例题
```
作业序列：J1(服务时间20), J2(8), J3(10), J4(5)
到达时间：J1(0), J2(1), J3(2), J4(3)

FCFS调度：
J1: 0 → 20，完成时间20
J2: 20 → 28，完成时间28
J3: 28 → 38，完成时间38
J4: 38 → 43，完成时间43

平均周转时间 = (20 + 27 + 36 + 40) / 4 = 30.75
平均带权周转时间 = (1.0 + 3.375 + 3.6 + 8.0) / 4 = 3.99
```

### 2. 短作业优先算法（SJF，Shortest Job First）

#### 算法思想
选择服务时间最短的作业优先执行

#### 实现方式
- **非抢占式SJF**：一旦分配给进程，直到完成
- **抢占式SJF（SRTN，最短剩余时间优先）**：新到达作业服务时间比当前运行作业短时，抢占CPU

#### 优缺点
- **优点**：
  - 平均等待时间最短
  - 吞吐量高
- **缺点**：
  - 长作业可能饥饿
  - 作业长度难以预知
  - 不利于I/O型作业

#### 例题
```
作业序列：J1(7), J2(4), J3(1), J4(4)
到达时间：J1(0), J2(2), J3(3), J4(4)

SJF调度：
J1: 0 → 7，完成时间7
J3: 7 → 8，完成时间8
J2: 8 → 12，完成时间12
J4: 12 → 16，完成时间16

平均周转时间 = (7 + 10 + 5 + 12) / 4 = 8.5
```

### 3. 优先级调度算法（Priority Scheduling）

#### 算法思想
为每个作业/进程设置优先级，选择优先级最高的先执行

#### 优先级设置
- **静态优先级**：创建进程时确定，运行期间不变
- **动态优先级**：运行期间动态调整

#### 实现方式
- **非抢占式**：进程执行完毕或主动放弃
- **抢占式**：新到达进程优先级更高时抢占

#### 优先级设置策略
- 系统进程 > 用户进程
- 交互型进程 > 非交互型进程
- I/O型进程 > CPU型进程

#### 优缺点
- **优点**：可以区分轻重缓急
- **缺点**：
  - 低优先级进程可能饥饿
  - 需要解决优先级反转问题

### 4. 时间片轮转算法（RR，Round Robin）

#### 算法思想
每个进程轮流执行一个时间片，时间片用完则排队

#### 时间片的选择
- **太长**：退化为FCFS
- **太短**：进程切换频繁，开销大
- **经验值**：10-100ms

#### 实现方式
- 采用**抢占式**
- 使用**队列**（FIFO）

#### 时间片轮转的调度过程
```
就绪队列：[J1, J2, J3, J4]
时间片 = 1

时间0：J1执行1个时间片 → 排队到队尾
时间1：J2执行1个时间片 → 排队到队尾
时间2：J3执行1个时间片 → 排队到队尾
时间3：J4执行1个时间片 → 排队到队尾
...
```

#### 优缺点
- **优点**：
  - 公平，无饥饿
  - 响应快，适合分时系统
- **缺点**：
  - 平均周转时间长
  - 时间片选择困难

### 5. 多级反馈队列算法（MFQ，Multilevel Feedback Queue）

#### 算法思想
- 设置多个就绪队列，每个队列优先级不同
- 每个队列采用不同的时间片
- 进程在队列间动态调整

#### 多级反馈队列的规则
1. 设置多个就绪队列，优先级从高到低
2. 高优先级队列的时间片小，低优先级队列的时间片大
3. 新进程先进入最高优先级队列
4. 时间片用完，降低优先级，进入下一级队列
5. 进程主动阻塞后唤醒，进入最高优先级队列

#### 多级反馈队列的优缺点
- **优点**：
  - 兼顾公平性和效率
  - 短作业响应快
  - 无饥饿问题
- **缺点**：
  - 实现复杂
  - 参数（队列数、时间片）难以调整

## 调度算法对比

| 算法 | 类型 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **FCFS** | 非抢占 | 实现简单、公平 | 平均等待时间长 | 批处理 |
| **SJF** | 非抢占 | 平均等待时间最短 | 长作业饥饿 | 批处理 |
| **优先级** | 可抢占 | 区分轻重缓急 | 低优先级饥饿 | 实时系统 |
| **RR** | 抢占 | 响应快、公平 | 平均周转时间长 | 分时系统 |
| **MFQ** | 抢占 | 兼顾公平和效率 | 实现复杂 | 通用系统 |

## 实时调度

### 什么是实时系统？
要求系统在指定时间内完成特定任务

### 实时任务的分类

#### 硬实时任务
- 必须在截止时间前完成
- 否则会造成严重后果（如飞机控制）

#### 软实时任务
- 尽量在截止时间前完成
- 偶尔错过可以接受（如视频播放）

### 最早截止时间优先算法（EDF，Earliest Deadline First）
- 选择截止时间最早的进程优先执行
- 适用于软实时系统

### 最低松弛度优先算法（LLF，Least Laxity First）
- 选择松弛度（截止时间 - 当前时间 - 执行时间）最小的进程

## 考试要点

### 高频考点
1. **FCFS算法** — 先来先服务的调度过程
2. **SJF算法** — 短作业优先的计算
3. **时间片轮转** — 时间片大小的选择
4. **多级反馈队列** — 各队列的时间片和优先级

### 常见陷阱
- FCFS不利于短作业
- SJF可能造成饥饿
- 时间片不是越小越好
- RR算法的时间片选择
