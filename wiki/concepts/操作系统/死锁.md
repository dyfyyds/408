# 死锁（Deadlock）

## 什么是死锁？

### 死锁的定义
多个进程因竞争资源而陷入一种僵局，若无外力作用，这些进程都无法向前推进

### 死锁的示例
```
哲学家进餐问题：
5个哲学家，5根筷子，每人左右各一根筷子
如果每个人都拿起左边的筷子，等待右边的筷子
→ 所有人都等待 → 死锁
```

## 死锁产生的四个必要条件

### 1. 互斥条件（Mutual Exclusion）
- 进程独占资源
- 一个资源每次只能被一个进程使用

### 2. 不剥夺条件（No Preemption）
- 进程已获得的资源，在未使用完之前，不能被剥夺
- 只能由该进程自己释放

### 3. 请求与保持条件（Hold and Wait）
- 进程在等待新资源时，不释放已占有的资源

### 4. 循环等待条件（Circular Wait）
- 存在进程-资源的循环等待链

#### 示例
```
P1 → P2 → P3 → P1

P1等待P2的资源
P2等待P3的资源
P3等待P1的资源
```

**四个条件缺一不可**，只要破坏其中一个，就可以预防死锁

## 死锁的处理策略

### 1. 死锁预防（Prevention）
**破坏死锁的四个必要条件之一**

#### 破坏互斥条件
- 将独占设备改为共享设备
- 但某些设备（如打印机）必须互斥使用

#### 破坏不剥夺条件
- 当一个进程请求新资源失败时，必须释放已占有的资源
- 缺点：实现复杂，可能导致前功尽弃

#### 破坏请求与保持条件
- 采用**静态分配策略**
- 进程在运行前一次性申请所有需要的资源
- 缺点：资源利用率低，可能导致饥饿

#### 破坏循环等待条件
- 采用**有序资源分配法**
- 给所有资源编号，进程必须按序号申请资源
- 优点：简单、安全、有效

### 2. 死锁避免（Avoidance）
**在资源分配时，判断是否安全，避免进入不安全状态**

#### 安全状态
系统能按某种进程推进顺序为每个进程分配资源

#### 不安全状态
不存在一种安全的进程推进顺序

### 银行家算法（Banker's Algorithm）

#### 算法思想
资源分配时，先判断分配后是否安全，若安全则分配，否则等待

#### 数据结构
- **Available**：可用资源向量
- **Max**：最大需求矩阵
- **Allocation**：已分配矩阵
- **Need**：需求矩阵 = Max - Allocation

#### 算法流程
```
1. 检查请求是否合法（Request ≤ Need）
2. 检查资源是否可用（Request ≤ Available）
3. 试分配资源
4. 执行安全性算法
5. 若安全，正式分配；若不安全，恢复状态
```

#### 安全性算法
```
1. 设置Work = Available, Finish[i] = false
2. 找到Finish[i] = false且Need[i] ≤ Work的进程Pi
3. 若找到，Work = Work + Allocation[i]，Finish[i] = true，重复步骤2
4. 若所有Finish[i] = true，则安全；否则不安全
```

#### 例题
```
系统有5个资源，3个进程：

进程  最大需求  已分配  需求
P1    3 2 2    1 0 0   2 2 2
P2    6 1 3    3 1 1   3 0 2
P3    3 1 4    1 1 1   2 0 3

可用资源：2 1 1

步骤1：P1可以满足（Need ≤ Available）
        Work = 2 1 1 + 1 0 0 = 3 1 1

步骤2：P2可以满足（Need = 3 0 2 ≤ Work = 3 1 1）
        Work = 3 1 1 + 3 1 1 = 6 2 2

步骤3：P3可以满足（Need = 2 0 3 ≤ Work = 6 2 2）
        安全序列：P1 → P2 → P3
```

### 3. 死锁检测与解除（Detection and Recovery）

#### 检测时机
- 资源利用率低时
- 进程长时间不推进时

#### 检测方法

##### 简化资源分配图
```
进程 → 资源 → 进程 → 资源

若所有进程都从图中消去，则无死锁
若有进程无法消去，则存在死锁
```

#### 死锁解除方法

##### 1. 资源剥夺法
- 抢占某些进程的资源，分配给死锁进程

##### 2. 进程终止法
- 终止一个或多个死锁进程
- **终止所有死锁进程**
- **逐个终止死锁进程**

##### 3. 进程回退法
- 将进程回退到安全状态

## 死锁预防 vs 死锁避免

| 对比维度 | 死锁预防 | 死锁避免 |
|---------|---------|---------|
| **策略** | 破坏必要条件 | 动态检测 |
| **资源利用率** | 低 | 高 |
| **实现难度** | 简单 | 复杂 |
| **实时性** | 好 | 一般 |
| **适用场景** | 系统简单 | 系统复杂 |

## 鸵鸟算法

### 什么是鸵鸟算法？
假装死锁不会发生，不做任何处理

### 适用场景
- 死锁极少发生
- 处理死锁的开销大于死锁造成的损失

### 采用鸵鸟算法的系统
- UNIX
- Linux
