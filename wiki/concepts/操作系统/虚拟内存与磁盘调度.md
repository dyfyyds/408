# 虚拟内存与磁盘调度

## 虚拟内存（Virtual Memory）

### 什么是虚拟内存？
将主存和辅存统一编址，构成一个大的存储空间，程序可以使用比物理主存更大的地址空间

### 虚拟内存的特征

#### 1. 多次性
一个作业不需要全部装入内存就可以运行

#### 2. 对换性
作业运行过程中，可以在内存和辅存之间对换

#### 3. 虚拟性
从逻辑上扩充内存容量

### 虚拟内存的实现方式

#### 1. 请求分页存储管理
- 将虚拟空间和主存空间都划分为大小相同的页
- 通过页表实现地址转换
- 页表增加存在位、访问位、修改位

#### 2. 请求分段存储管理
- 按程序的逻辑模块分段
- 通过段表实现地址转换

### 缺页中断处理

#### 缺页中断流程
```
访问页面 → 检查存在位
  ├── 存在 → 地址转换，访问内存
  └── 不存在 → 触发缺页中断
        ↓
    分配页框
        ↓
    从辅存调入页面
        ↓
    修改页表
        ↓
    重新执行指令
```

#### 缺页中断的特点
- 属于**内部中断**
- 在指令执行期间产生
- 中断处理后重新执行被中断的指令

### 页面置换算法

#### 1. 最佳置换算法（OPT，Optimal）

##### 思想
置换最长时间不再被访问的页面

##### 特点
- **理论最优算法**
- 无法实现（需要预知未来）
- 用于评价其他算法

##### 例题
```
页面访问序列：7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1
物理块数：3

步骤：
7: [7] [ ] [ ] → 缺页
0: [7] [0] [ ] → 缺页
1: [7] [0] [1] → 缺页
2: [2] [0] [1] → 缺页（7最长时间不再访问）
0: [2] [0] [1] → 命中
3: [2] [0] [3] → 缺页（1最长时间不再访问）
0: [2] [0] [3] → 命中
4: [2] [4] [3] → 缺页（0最长时间不再访问）
2: [2] [4] [3] → 命中
3: [2] [4] [3] → 命中
0: [0] [4] [3] → 缺页（2最长时间不再访问）
3: [0] [4] [3] → 命中
2: [0] [2] [3] → 缺页（4最长时间不再访问）
1: [0] [2] [1] → 缺页（3最长时间不再访问）

缺页次数：9次
缺页率：9/14 ≈ 64.3%
```

#### 2. 先进先出算法（FIFO，First In First Out）

##### 思想
置换最早进入内存的页面

##### 实现
- 使用**队列**
- 先进入的先被替换

##### 特点
- 实现简单
- 不符合局部性原理
- 可能出现**Belady异常**

###### Belady异常
```
增加物理块数，缺页率反而增加
```

**示例：**
```
页面访问序列：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

3块时缺页次数：9次
4块时缺页次数：10次（增加了！）
```

#### 3. 最近最少使用算法（LRU，Least Recently Used）

##### 思想
置换最长时间未被访问的页面

##### 实现
- 使用**栈**或**计数器**
- 每次访问时更新时间戳

##### 特点
- 最接近OPT算法
- 需要硬件支持
- **不会出现Belady异常**

##### 例题
```
页面访问序列：7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1
物理块数：3

步骤：
7: [7] [ ] [ ] → 缺页
0: [7] [0] [ ] → 缺页
1: [7] [0] [1] → 缺页
2: [2] [0] [1] → 缺页（7最长时间未访问）
0: [2] [0] [1] → 命中
3: [3] [0] [1] → 缺页（2最长时间未访问）
0: [3] [0] [1] → 命中
4: [4] [0] [1] → 缺页（3最长时间未访问）
2: [2] [0] [1] → 缺页（4最长时间未访问）
3: [3] [0] [1] → 缺页（2最长时间未访问）
0: [3] [0] [1] → 命中
3: [3] [0] [1] → 命中
2: [3] [2] [1] → 缺页（0最长时间未访问）
1: [3] [2] [1] → 命中

缺页次数：8次
缺页率：8/14 ≈ 57.1%
```

#### 4. 时钟置换算法（Clock）

##### 思想
LRU的近似算法，使用**访问位**和**修改位**

##### 实现
1. 将所有页框组织成环形链表
2. 指针指向下一个待检查的页框
3. 访问位=1的页框不替换，访问位置0
4. 访问位=0的页框替换

##### 改进型Clock算法
优先淘汰：(0, 0) → (0, 1) → (1, 0) → (1, 1)

### 页面置换算法对比

| 算法 | 实现复杂度 | 缺页率 | Belady异常 | 实际应用 |
|------|-----------|--------|-----------|---------|
| **OPT** | 无法实现 | 最低 | 否 | 理论基准 |
| **FIFO** | 简单 | 高 | 可能 | 简单系统 |
| **LRU** | 复杂 | 低 | 否 | 广泛使用 |
| **Clock** | 中等 | 较低 | 否 | 实际系统 |

## 磁盘调度

### 磁盘访问时间

$$磁盘访问时间 = 寻道时间 + 旋转延迟时间 + 传输时间$$

#### 1. 寻道时间（Seek Time）
- 磁头移动到目标磁道的时间

#### 2. 旋转延迟时间（Latency）
- 等待目标扇区到达磁头下的时间
- 平均旋转延迟 = 半圈时间 = (60 / 转速) / 2

#### 3. 传输时间（Transfer Time）
- 数据从一个扇区传送到内存的时间

### 磁盘调度算法

#### 1. 先来先服务算法（FCFS）

##### 思想
按请求到达顺序处理

##### 优缺点
- **优点**：公平、简单
- **缺点**：寻道时间长，性能较差

##### 示例
```
磁头起始位置：100
请求序列：55, 58, 39, 18, 90, 160, 150, 38, 184

FCFS调度：
100 → 55 → 58 → 39 → 18 → 90 → 160 → 150 → 38 → 184

总寻道距离：|100-55| + |55-58| + |58-39| + |39-18| + |18-90| + |90-160| + |160-150| + |150-38| + |38-184|
          = 45 + 3 + 19 + 21 + 72 + 70 + 10 + 112 + 146
          = 498
```

#### 2. 最短寻道时间优先算法（SSTF，Shortest Seek Time First）

##### 思想
优先处理距当前磁头最近的请求

##### 优缺点
- **优点**：平均寻道时间短
- **缺点**：
  - 可能出现饥饿
  - 不公平

##### 示例
```
磁头起始位置：100
请求序列：55, 58, 39, 18, 90, 160, 150, 38, 184

SSTF调度：
100 → 90 → 58 → 55 → 39 → 38 → 18 → 150 → 160 → 184

总寻道距离：|100-90| + |90-58| + |58-55| + |55-39| + |39-38| + |38-18| + |18-150| + |150-160| + |160-184|
          = 10 + 32 + 3 + 16 + 1 + 20 + 132 + 10 + 24
          = 248
```

#### 3. 扫描算法（SCAN，电梯算法）

##### 思想
磁头从一端移动到另一端，沿途响应请求

##### 实现
- 磁头向一个方向移动，响应途中所有请求
- 到达一端后反向移动

##### 优缺点
- **优点**：性能较好，无饥饿
- **缺点**：
  - 两端请求响应慢
  - 磁头可能空转

##### 示例
```
磁头起始位置：100，方向向大
请求序列：55, 58, 39, 18, 90, 160, 150, 38, 184

SCAN调度：
100 → 150 → 160 → 184（到达端点） → 90 → 58 → 55 → 39 → 38 → 18

总寻道距离：|100-150| + |150-160| + |160-184| + |184-90| + |90-58| + |58-55| + |55-39| + |39-38| + |38-18|
          = 50 + 10 + 24 + 94 + 32 + 3 + 16 + 1 + 20
          = 250
```

#### 4. 循环扫描算法（C-SCAN，Circular SCAN）

##### 思想
磁头只沿一个方向扫描，到达端点后直接返回起点

##### 实现
- 磁头向一个方向移动，响应途中所有请求
- 到达端点后，快速返回起点，不响应途中请求

##### 优缺点
- **优点**：响应时间更均匀
- **缺点**：磁头空转时间长

#### 5. LOOK和C-LOOK算法

##### LOOK算法
- 磁头移动到最远的请求处就反向
- 不需要到达端点

##### C-LOOK算法
- 磁头移动到最远的请求处后，直接返回起点

### 磁盘调度算法对比

| 算法 | 寻道时间 | 公平性 | 饥饿 | 实际应用 |
|------|---------|--------|------|---------|
| **FCFS** | 长 | 好 | 否 | 批处理 |
| **SSTF** | 短 | 差 | 可能 | 较少 |
| **SCAN** | 中 | 中 | 否 | 广泛使用 |
| **C-SCAN** | 中 | 好 | 否 | 广泛使用 |
| **LOOK** | 中 | 中 | 否 | 实际系统 |
| **C-LOOK** | 中 | 好 | 否 | 实际系统 |

## 考试要点

### 高频考点
1. **页面置换算法** — OPT、FIFO、LRU的执行过程
2. **Belady异常** — 何时发生，为何发生
3. **磁盘调度算法** — FCFS、SSTF、SCAN、C-SCAN的对比
4. **缺页中断** — 缺页中断的处理流程

### 常见陷阱
- LRU不会出现Belady异常
- FIFO可能出现Belady异常
- 磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间
