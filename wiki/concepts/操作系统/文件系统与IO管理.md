# 文件系统与I/O管理

## 文件系统

### 文件的基本概念

#### 什么是文件？
文件是操作系统用于存储和管理信息的基本逻辑单位

#### 文件的分类
| 分类方式 | 类型 | 说明 |
|---------|------|------|
| **按用途** | 系统文件、用户文件、库文件 | 文件的功能 |
| **按保护级别** | 只读文件、读写文件、可执行文件、不保护文件 | 文件权限 |
| **按信息流向** | 输入文件、输出文件、输入输出文件 | 文件用途 |
| **按设备类型** | 普通文件、设备文件、管道文件、目录文件 | 文件存储设备 |

#### 文件的属性
1. **文件名**：用户可读的名称
2. **标识符**：系统内部的唯一标识
3. **类型**：文件类型
4. **位置**：文件在外存上的位置
5. **大小**：文件长度
6. **保护信息**：访问权限
7. **时间信息**：创建、访问、修改时间

### 文件的组织结构

#### 逻辑结构
- **无结构文件**（流式文件）：字节流
- **有结构文件**（记录式文件）：由记录组成

#### 物理结构（存储结构）
- **连续分配**：文件占用连续的磁盘块
- **链接分配**：通过链表连接磁盘块
- **索引分配**：为每个文件建立索引块

### 目录结构

#### 单级目录结构
- 所有文件在同一目录下
- 查找速度慢，不允许重名

#### 两级目录结构
- 主目录（MFD）：用户目录
- 用户目录（UFD）：用户文件目录
- 允许不同用户使用相同文件名

#### 树形目录结构
- 根目录 → 子目录 → 文件
- 允许不同目录有同名文件
- 支持路径（绝对路径、相对路径）

#### 图形目录结构（无环图结构）
- 允许一个文件出现在多个目录中（共享）
- 可以用链接实现

### 文件共享

#### 共享方式
1. **硬链接** — 多个目录项指向同一个物理块（索引节点）
2. **符号链接（软链接）** — 创建一个新的文件，内容是目标文件的路径

### 文件存储空间管理

#### 1. 空闲表法
- 类似于动态分区的分配
- 空闲盘区组成空闲表

#### 2. 空闲链表法
- 所有空闲块链接成链表

#### 3. 位示图法
- 用二进制位表示磁盘块的使用情况
- 0：空闲；1：已分配

```
位示图 = 行号 × 总列数 + 列号

块号 = 字号 × 位数 + 位号
```

#### 4. 成组链接法
- UNIX系统使用
- 空闲块每100块为一组，每块的第一项指向下一组

### 文件系统调用

#### 创建文件
```c
create(pathname, mode);
```

#### 删除文件
```c
unlink(pathname);
```

#### 打开文件
```c
fd = open(pathname, flags);
```

#### 关闭文件
```c
close(fd);
```

#### 读文件
```c
read(fd, buffer, count);
```

#### 写文件
```c
write(fd, buffer, count);
```

### 文件系统的安全性

#### 磁盘转储
- 全量转储：定期备份整个文件系统
- 增量转储：只备份上次转储后修改的文件

#### 文件系统一致性检查
- 检查文件系统的数据结构是否一致

## I/O管理

### I/O设备分类

#### 按传输速率分类
- **低速设备**：键盘、鼠标（几到几百B/s）
- **中速设备**：打印机、激光打印机（几千到几万B/s）
- **高速设备**：磁盘、网卡（几十万到几MB/s）

#### 按信息交换单位分类
- **块设备**：磁盘、SD卡（以块为单位传输）
- **字符设备**：键盘、鼠标、显示器（以字符为单位传输）

#### 按设备共享属性分类
- **独占设备**：打印机（一段时间内只能由一个进程独占）
- **共享设备**：磁盘（可被多个进程同时访问）
- **虚拟设备**：通过SPOOLing技术实现的虚拟设备

### I/O控制方式

#### 1. 程序查询方式
- CPU主动查询I/O设备状态
- 串行工作，CPU利用率低

#### 2. 程序中断方式
- I/O设备就绪时向CPU发中断
- CPU与I/O并行工作

#### 3. DMA方式
- I/O设备直接与内存交换数据
- CPU仅在初始化和结束时介入

#### 4. 通道控制方式
- 引入专门的I/O处理机（通道）
- CPU只需发I/O指令，通道完成数据交换
- 适合大规模I/O系统

### 缓冲技术

#### 为什么需要缓冲？
- 解决速度不匹配问题
- 减少中断次数
- 提高CPU和I/O设备的并行性

#### 缓冲的类型

##### 1. 单缓冲
- 在用户区和设备之间设置一个缓冲区
- 缓冲时间：$T = max(C, T') + M$
  - $C$：CPU计算时间
  - $T'$：I/O传输时间
  - $M$：从缓冲区到内存的传输时间

##### 2. 双缓冲
- 设置两个缓冲区
- 一个缓冲区输入时，另一个缓冲区输出
- 吞吐率提高一倍

##### 3. 循环缓冲
- 多个缓冲区组成循环队列
- 适合生产者和消费者速度不匹配的情况

##### 4. 缓冲池（Buffer Pool）
- 系统设置多个公共缓冲区
- 可以动态分配
- 提高缓冲区利用率

### 设备独立性

#### 什么是设备独立性？
应用程序独立于具体使用的物理设备

#### 逻辑设备名与物理设备名
- **逻辑设备名**：应用程序使用的名称
- **物理设备名**：实际设备的名称

#### 设备控制表（DCT）
- 记录设备的状态和控制参数

#### 设备分配
- 先分配设备，再分配控制器，最后分配通道

### SPOOLing技术（Simultaneous Peripheral Operations On-Line）

#### 思想
- 用磁盘模拟打印机
- 应用程序输出时写入磁盘，打印机慢慢打印

#### 组成
1. **输入井**：磁盘上的输入缓冲区
2. **输出井**：磁盘上的输出缓冲区
3. **输入进程**：将输入数据写入输入井
4. **输出进程**：从输出井读取数据输出到打印机

#### 特点
- 实现了虚拟设备功能
- 提高了I/O速度
- 将独占设备变成共享设备

## 考试要点

### 高频考点
1. **文件的逻辑结构** — 流式文件 vs 记录式文件
2. **文件的物理结构** — 连续、链接、索引分配
3. **位示图法** — 空闲块管理
4. **目录结构** — 单级、两级、树形、图形目录
5. **I/O控制方式** — 程序查询、中断、DMA、通道
6. **缓冲技术** — 单缓冲、双缓冲、循环缓冲、缓冲池

### 常见陷阱
- 连续分配支持随机访问，链接分配不支持随机访问
- 位示图中，块号 = 字号 × 位数 + 位号
- 双缓冲的吞吐率 ≈ CPU处理速度（CPU和I/O并行）

- 2012年408真题：位示图计算空闲块号
- 2015年408真题：磁盘空间管理
- 2017年408真题：I/O控制方式的对比
