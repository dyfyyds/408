# 网络层（Network Layer）

## 网络层的功能

### 网络层的核心功能
1. **路由选择**：选择最佳路径将数据包送达目的地
2. **分组转发**：将数据包从输入链路转移到输出链路
3. **网络互连**：连接异构网络

### 数据交换方式

#### 1. 电路交换
- 面向连接，建立专用物理路径
- 优点：传输时延小
- 缺点：线路利用率低

#### 2. 报文交换
- 整个报文一起传输
- 存储转发，线路利用率高
- 实时性差

#### 3. 分组交换
- 将报文分成多个分组
- 独立传输，动态选路
- 现代计算机网络采用的方式

## IP协议

### IP地址

#### IPv4地址
- 32位二进制数
- 点分十进制表示：192.168.1.1

#### IP地址的分类

| 类别 | 前缀 | 网络号 | 主机号 | 范围 |
|------|------|--------|--------|------|
| **A类** | 0 | 8位 | 24位 | 1.0.0.0 ~ 126.255.255.255 |
| **B类** | 10 | 16位 | 16位 | 128.0.0.0 ~ 191.255.255.255 |
| **C类** | 110 | 24位 | 8位 | 192.0.0.0 ~ 223.255.255.255 |
| **D类** | 1110 | 多播地址 | - | 224.0.0.0 ~ 239.255.255.255 |
| **E类** | 1111 | 保留地址 | - | 240.0.0.0 ~ 255.255.255.255 |

#### 特殊IP地址

| 网络号 | 主机号 | 含义 |
|--------|--------|------|
| 全0 | 全0 | 本网络上的本主机 |
| 网络号 | 全0 | 本网络上的主机 |
| 网络号 | 全1 | 广播地址（直接广播） |
| 127 | 任意 | 环回地址（localhost） |

### 子网划分

#### 为什么需要子网划分？
- 缓解IP地址空间不足
- 便于管理

#### 子网划分原理
- 从主机号中借用若干位作为**子网号**
- 子网号 + 主机号

#### 子网掩码
- 子网掩码中1对应网络号+子网号，0对应主机号
- 例如：192.168.1.10/24 → 子网掩码：255.255.255.0

#### 子网划分示例

**原始C类地址：**
```
192.168.1.0/24
```

**借用2位作为子网号：**
```
192.168.1.0/26
192.168.1.64/26
192.168.1.128/26
192.168.1.192/26
```

每个子网可容纳主机数：$2^{(32-26)} - 2 = 62$ 台

### CIDR（无分类域间路由）

#### 什么是CIDR？
- 消除了传统的A、B、C类地址的概念
- 使用**斜线记法**：192.168.1.0/24

#### CIDR地址块
- 将网络前缀相同的连续IP地址组成一个CIDR地址块
- 例如：192.168.0.0/16 包含 192.168.0.0 ~ 192.168.255.255

#### 最长前缀匹配
- 当路由表中有多个匹配项时，选择**最长前缀**的路由

### 私有IP地址

| 地址范围 | 说明 |
|---------|------|
| 10.0.0.0 ~ 10.255.255.255 | A类私有地址 |
| 172.16.0.0 ~ 172.31.255.255 | B类私有地址 |
| 192.168.0.0 ~ 192.168.255.255 | C类私有地址 |

### NAT（网络地址转换）

#### 什么是NAT？
- 将私有IP地址转换为公有IP地址
- 解决IPv4地址不足的问题

#### 静态NAT
- 一对一映射

#### 动态NAT
- 多对多映射，从公有地址池中分配

#### NAPT（Network Address Port Translation）
- 多对一映射，通过端口号区分不同的主机
- 最常用

## ARP协议（地址解析协议）

### 什么是ARP？
将IP地址解析为MAC地址

### ARP的工作原理

```
已知IP地址 → 广播ARP请求：谁有IP？ → 收到ARP应答 → 得到MAC地址
```

1. 主机A知道主机B的IP地址
2. 主机A广播ARP请求："谁有IP地址B？请告诉我MAC地址"
3. 主机B收到ARP请求，发送ARP应答："我的IP是B，MAC是XX:XX:XX:XX:XX:XX"
4. 主机A收到ARP应答，更新ARP缓存表

### ARP报文格式

```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│ 硬件类型 │ 协议类型 │ 硬件地址长度 │ 协议地址长度 │ 操作码 │ 发送方MAC │ 发送方IP │ 目标方MAC │ 目标方IP │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

- 硬件类型：1表示以太网
- 协议类型：0x0800表示IP
- 操作码：1表示ARP请求，2表示ARP应答

## ICMP协议（Internet控制报文协议）

### 什么是ICMP？
ICMP用于网络设备之间传递控制消息和差错报告

### ICMP的应用

#### 1. Ping命令（ICMP回显请求/应答）
- 测试网络连通性
- ICMP类型：8（请求），0（应答）

#### 2. Traceroute命令（ICMP超时）
- 跟踪数据包经过的路由
- 通过逐渐增加TTL，使路由器返回ICMP超时

### ICMP报文类型

| 类型 | 代码 | 说明 |
|------|------|------|
| 0 | 0 | 回显应答（Ping应答） |
| 3 | - | 目的不可达 |
| 4 | 0 | 源抑制（拥塞控制） |
| 5 | - | 重定向 |
| 8 | 0 | 回显请求（Ping请求） |
| 11 | - | 超时 |

## 路由协议

### 什么是路由？
在数据包从源地址到目的地址的传输过程中，经过的路由器的转发过程

### 路由表
路由器根据路由表进行转发

| 目的网络 | 子网掩码 | 下一跳 | 接口 |
|---------|---------|--------|------|

### 路由协议的分类

#### 内部网关协议（IGP）

##### 1. RIP（Routing Information Protocol）
- 距离向量路由协议
- 最大跳数15，16表示不可达
- 周期广播路由表（30秒）
- **缺点**：收敛慢，网络规模受限

##### 2. OSPF（Open Shortest Path First）
- 链路状态路由协议
- 使用Dijkstra算法计算最短路径
- **优点**：收敛快，支持大型网络

#### 外部网关协议（EGP）

##### BGP（Border Gateway Protocol）
- 用于自治系统（AS）之间的路由
- 路径向量协议
- 基于策略的路由选择

### IP数据报的转发

#### 路由器转发流程
```
收到IP数据包 → 检查数据链路层帧头
  ↓
提取IP数据包
  ↓
检查IP版本、校验和
  ↓
查路由表，找到匹配项（最长前缀匹配）
  ↓
TTL减1
  ↓
重新计算校验和
  ↓
根据路由表，发送到下一跳
  ↓
封装成新的数据链路层帧
```

## 考试要点

### 高频考点
1. **IP地址** — 分类、特殊地址、私有地址
2. **子网划分** — 子网掩码、子网号计算
3. **CIDR** — 无分类编址、最长前缀匹配
4. **ARP协议** — IP地址到MAC地址的解析
5. **ICMP协议** — Ping、Traceroute

### 常见陷阱
- 子网划分时，主机号不能全0或全1
- 私有地址不能在公网上路由
- 路由器隔离广播域
