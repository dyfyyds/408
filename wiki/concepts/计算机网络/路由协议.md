# 路由协议

## 路由的基本概念

### 什么是路由？
在数据包从源地址到目的地址的传输过程中，经过的路由器的转发过程

### 路由表
路由器根据路由表进行转发决策

#### 路由表的结构
```
目的网络 | 子网掩码 | 下一跳 | 接口 | 度量值（跳数）
```

### 路由的分类

#### 按路由的归属分类
- **默认路由**：0.0.0.0/0，转发到某个默认接口
- **特定路由**：特定网络的路由

#### 按路由的获取方式分类
- **静态路由**：管理员手动配置
- **动态路由**：路由协议自动计算

### 路由协议的分类

#### 1. 内部网关协议（IGP，Interior Gateway Protocol）
- 用于**同一个自治系统（AS）**内的路由
- 例如：RIP、OSPF

#### 2. 外部网关协议（EGP，Exterior Gateway Protocol）
- 用于**不同自治系统（AS）**之间的路由
- 例如：BGP

## 自治系统（AS）

### 什么是自治系统？
- 具有统一管理机构、统一路由策略的网络
- 用AS号（ASN）标识
- 范围：1 ~ 65535（16位），扩展支持32位

## 距离向量路由协议

### 什么是距离向量？
- 每个路由器只知道到目的网络的**距离**和**方向**（下一跳）
- 通过邻居交换路由信息

### RIP（Routing Information Protocol）

#### 什么是RIP？
- 最古老的IGP之一
- 基于**距离向量**算法

#### RIP的特点

##### 度量标准
- **跳数（Hop Count）**：经过的路由器数量
- 最大跳数：15，16表示不可达

##### 更新方式
- 每30秒广播一次路由表
- 路由表包含：目的网络、跳数、下一跳

##### 路由更新
```
路由器A收到邻居B的路由表
  ↓
   对于每个目的网络N：
     如果 B→N的路径更短
     更新 A→N的路径 = A→B的路径 + B→N的路径
```

#### RIP的版本

| 版本 | 说明 |
|------|------|
| **RIPv1** | 有类路由协议，不支持VLSM，广播更新 |
| **RIPv2** | 无类路由协议，支持VLSM/CIDR，组播更新 |

#### RIP的优缺点

**优点：**
- 实现简单
- 配置简单

**缺点：**
- **收敛慢**：网络变化后，需要较长时间才能达成一致
- **最大跳数15**：不适合大规模网络
- **跳数作为度量**：不考虑带宽、延迟
- **周期广播**：即使没有变化，也定期广播路由表

#### RIP的坏消息传播慢
```
网络拓扑：
A → B → C → D

C到D的链路断开
1. C将D标记为不可达（16跳）
2. B收到C的路由表，发现D变为16跳，更新
3. A收到B的路由表，发现D变为17跳，但最大16，标记为不可达
4. 最终A也知道D不可达

需要传播多轮才能让所有路由器知道
```

## 链路状态路由协议

### 什么是链路状态？
- 每个路由器了解整个网络的拓扑结构
- 使用**最短路径算法**（Dijkstra）计算路由

### OSPF（Open Shortest Path First）

#### 什么是OSPF？
- 基于**链路状态**算法
- 开放式最短路径优先

#### OSPF的特点

##### 度量标准
- **开销（Cost）**：基于链路带宽
- Cost = 参考带宽 / 链路带宽

##### 路由器类型
- **内部路由器**：区域内路由器
- **区域边界路由器（ABR）**：连接骨干区域和其他区域
- **骨干路由器**：属于Area 0（骨干区域）
- **自治系统边界路由器（ASBR）**：连接外部AS

##### 区域（Area）
- OSPF将AS划分为多个区域
- 减少LSA泛洪范围
- 提高可扩展性

##### 路由器类型
| 类型 | 说明 |
|------|------|
| **IR（Internal Router）** | 所有接口在同一区域 |
| **ABR（Area Border Router）** | 连接骨干区域和其他区域 |
| **BR（Backbone Router）** | 至少一个接口在Area 0 |
| **ASBR（AS Boundary Router）** | 连接外部AS |

##### 路由器类型
| 类型 | 说明 |
|------|------|
| **IR（Internal Router）** | 所有接口在同一区域 |
| **ABR（Area Border Router）** | 连接骨干区域和其他区域 |
| **BR（Backbone Router）** | 至少一个接口在Area 0 |
| **ASBR（AS Boundary Router）** | 连接外部AS |

#### OSPF的报文类型

| 类型 | 说明 |
|------|------|
| **Hello** | 发现和维护邻居关系 |
| **DBD（Database Description）** | 描述本地LSDB |
| **LSR（Link State Request）** | 请求特定LSA |
| **LSU（Link State Update）** | 发送LSA |
| **LSAck（Link State Ack）** | 确认收到LSA |

#### OSPF的邻接关系
- 路由器之间必须建立邻接关系
- 交换链路状态信息

#### OSPF的工作流程
```
1. 发现邻居（Hello报文）
2. 建立邻接关系
3. 交换链路状态信息（LSA）
4. 计算最短路径（Dijkstra算法）
5. 生成路由表
```

#### OSPF的优缺点

**优点：**
- **收敛快**：网络变化后，快速更新
- **支持大规模网络**：支持区域划分
- **无跳数限制**：使用Cost作为度量
- **无类路由**：支持VLSM/CIDR

**缺点：**
- 配置复杂
- 资源消耗大

## BGP（Border Gateway Protocol）

### 什么是BGP？
- 用于**自治系统之间**的路由
- 路径向量协议

### BGP的特点

#### 度量标准
- **路径属性**：AS路径、下一跳、本地优先级等

#### 路径选择原则
- 基于**策略**的路由选择
- 不单纯考虑最短路径

#### 更新方式
- 增量更新
- 只发送变化的路由

#### BGP报文类型
| 类型 | 说明 |
|------|------|
| **OPEN** | 建立BGP连接 |
| **UPDATE** | 发送路由更新 |
| **KEEPALIVE** | 保持BGP连接 |
| **NOTIFICATION** | 差错报告 |

### BGP的特点

**优点：**
- 支持大规模网络（Internet）
- 基于策略的路由

**缺点：**
- 配置复杂
- 收敛慢

## 路由协议对比

| 协议 | 类型 | 度量 | 适用规模 | 收敛速度 |
|------|------|------|---------|---------|
| **RIP** | 距离向量 | 跳数 | 小 | 慢 |
| **OSPF** | 链路状态 | Cost | 大 | 快 |
| **BGP** | 路径向量 | 策略 | 极大（Internet） | 慢 |

## 路由聚合

### 什么是路由聚合？
将多个路由条目合并为一个更大的路由条目

### 路由聚合的优点
- 减少路由表大小
- 减少路由更新流量
- 提高路由查询效率

### 示例
```
路由表：
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
192.168.3.0/24

聚合为：192.168.0.0/22
```

## 考试要点

### 高频考点
1. **RIP协议** — 距离向量、最大跳数、收敛速度
2. **OSPF协议** — 链路状态、Dijkstra算法、区域划分
3. **BGP协议** — 路径向量、策略路由
4. **路由协议对比** — 各种协议的适用场景

### 常见陷阱
- RIP最大跳数15
- OSPF使用Cost作为度量
- BGP用于AS之间的路由
