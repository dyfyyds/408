# UDP协议（用户数据报协议）

## 什么是UDP？

UDP（User Datagram Protocol）是**无连接的、不可靠的、面向报文**的传输层协议

## UDP的特点

### 无连接
- 通信前不需要建立连接
- 发送数据前不需要握手

### 不可靠
- 尽最大努力交付
- 不保证数据到达
- 不保证顺序

### 面向报文
- 保留应用层报文的边界
- 应用层报文多大，UDP就发送多大

### 低开销
- 首部只有8字节
- 无流量控制、拥塞控制
- 速度快

## UDP报文段结构

### 首部格式（8字节）

```
┌──────┬──────┬────────┬──────┐
│ 源端口 │ 目的端口 │  数据长度 │ 校验和 │
│ 16位  │ 16位   │  16位   │ 16位 │
└──────┴──────┴────────┴──────┘
```

#### 字段说明

| 字段 | 大小 | 功能 |
|------|------|------|
| **源端口** | 16位 | 源端口号 |
| **目的端口** | 16位 | 目的端口号 |
| **数据长度** | 16位 | UDP用户数据报长度（字节） |
| **校验和** | 16位 | 用于差错校验 |

### UDP数据报

```
UDP报文段 = UDP首部（8字节） + UDP数据

最大数据长度：65535 - 8 = 65527字节
```

## UDP校验和

### 校验范围
- UDP伪首部 + UDP首部 + UDP数据

### 伪首部格式

```
┌──────┬──────┬──────┬──────┬──────┬──────┐
│ 源IP地址（4字节）                        │
├──────────────────────────────────────────┤
│ 目的IP地址（4字节）                       │
├──────────────────────────────────────────┤
│ 全0     │ 协议   │  UDP长度    │
│ 8位     │ 8位    │  16位      │
└──────────────────────────────────────────┘
```

### 校验和的计算
- 采用**二进制反码求和**
- 结果取反作为校验和
- 若校验和为全0，则置为全1

## TCP与UDP对比

| 对比维度 | TCP | UDP |
|---------|-----|-----|
| **连接性** | 面向连接 | 无连接 |
| **可靠性** | ✅ 可靠 | ❌ 不可靠 |
| **流量控制** | ✅ 有 | ❌ 无 |
| **拥塞控制** | ✅ 有 | ❌ 无 |
| **传输效率** | 低 | 高 |
| **数据格式** | 面向字节流 | 面向报文 |
| **首部大小** | 20-60字节 | 8字节 |
| **传输模式** | 单播 | 单播、多播、广播 |
| **适用场景** | 网页、邮件、文件 | 视频、语音、DNS、DHCP |

## UDP的应用场景

### 适合使用UDP的场景

1. **实时性要求高**
   - 视频会议
   - 语音通话
   - 在线游戏

2. **广播/多播**
   - DNS查询
   - DHCP
   - RIP路由协议

3. **小数据量传输**
   - DNS查询
   - SNMP

4. **可靠性由应用层保证**
   - TFTP
   - QUIC

### 不适合使用UDP的场景
- 需要可靠传输的场景（文件传输、网页）
- 需要流量控制、拥塞控制的场景

## 基于UDP的应用层协议

| 协议 | 端口 | 说明 |
|------|------|------|
| **DNS** | 53 | 域名解析 |
| **DHCP** | 67/68 | 动态主机配置 |
| **RIP** | 520 | 路由协议 |
| **TFTP** | 69 | 简单文件传输 |
| **SNMP** | 161/162 | 简单网络管理 |
| **NTP** | 123 | 网络时间协议 |
