# 计算机网络体系结构

## 计算机网络概述

### 什么是计算机网络？
计算机网络是将地理位置不同的、具有独立功能的多台计算机及其外部设备通过通信线路连接起来，实现资源共享和信息传递的系统。

### 计算机网络的功能
1. **资源共享**：硬件、软件、数据资源共享
2. **信息传输**：实现计算机之间的信息传递
3. **分布式处理**：协同工作完成复杂任务
4. **负载均衡**：提高系统可靠性

### 计算机网络的分类

#### 按地理覆盖范围分类
- **局域网（LAN）**：几公里以内，如校园网
- **城域网（MAN）**：几公里到几十公里，如城市宽带网
- **广域网（WAN）**：几十公里到几千公里，如Internet

#### 按拓扑结构分类
- **总线型拓扑**：所有节点共享一条通信线路
- **星型拓扑**：所有节点连接到一个中心节点
- **环型拓扑**：节点形成一个环
- **网状型拓扑**：节点之间有多条路径

## 计算机网络体系结构

### 为什么需要分层？
- 降低复杂度
- 各层独立，便于维护
- 接口标准化，促进互操作

### OSI参考模型（七层）

```
应用层（Application）
  ↓
表示层（Presentation）
  ↓
会话层（Session）
  ↓
传输层（Transport） ← 端到端
  ↓
网络层（Network）    ← 点到点
  ↓
数据链路层（Data Link）
  ↓
物理层（Physical）
```

#### 各层的功能

| 层次 | 功能 | 协议数据单元（PDU） |
|------|------|-------------------|
| **应用层** | 为用户提供网络服务 | 报文（Message） |
| **表示层** | 数据格式转换、加密解密、压缩解压 | 报文 |
| **会话层** | 建立、管理、终止会话 | 报文 |
| **传输层** | 端到端的可靠/不可靠数据传输 | 段（Segment）、数据报（Datagram） |
| **网络层** | 路由选择、逻辑寻址、分组转发 | 分组（Packet）、数据报（Datagram） |
| **数据链路层** | 相邻节点间可靠传输、帧定界、差错控制 | 帧（Frame） |
| **物理层** | 比特流传输、物理介质、信号编码 | 比特（Bit） |

### TCP/IP参考模型（四层）

```
应用层（Application）
  ↓
传输层（Transport）
  ↓
网际层（Internet）
  ↓
网络接口层（Network Interface）
```

#### 与OSI模型的对比

| OSI模型 | TCP/IP模型 | 说明 |
|---------|-----------|------|
| 应用层 | 应用层 | 合并 |
| 表示层 | 应用层 | 合并 |
| 会话层 | 应用层 | 合并 |
| 传输层 | 传输层 | 相同 |
| 网络层 | 网际层 | 基本相同 |
| 数据链路层 | 网络接口层 | 合并 |
| 物理层 | 网络接口层 | 合并 |

### 五层参考模型（教学常用）

```
应用层（Application）
  ↓
传输层（Transport）
  ↓
网络层（Network）
  ↓
数据链路层（Data Link）
  ↓
物理层（Physical）
```

### 各层的核心问题

| 层次 | 核心问题 | 关键协议 |
|------|---------|---------|
| **应用层** | 为用户提供网络服务 | HTTP、FTP、SMTP、DNS |
| **传输层** | 端到端的通信 | TCP、UDP |
| **网络层** | 路由选择和转发 | IP、ICMP、RIP、OSPF |
| **数据链路层** | 相邻节点可靠传输 | PPP、CSMA/CD |
| **物理层** | 比特流传输 | 以太网、光纤 |

## 数据封装与解封装

### 发送过程（封装）
```
应用层：报文
  ↓ 加首部
传输层：段（Segment）
  ↓ 加首部
网络层：数据包（Packet）
  ↓ 加首部和尾部
数据链路层：帧（Frame）
  ↓ 转换
物理层：比特流
```

### 接收过程（解封装）
```
物理层：比特流
  ↓ 转换
数据链路层：帧（Frame）
  ↓ 去掉首尾部
网络层：数据包（Packet）
  ↓ 去掉首部
传输层：段（Segment）
  ↓ 去掉首部
应用层：报文
```

### 各层的PDU（协议数据单元）

| 层次 | PDU |
|------|-----|
| **应用层** | Message（报文） |
| **传输层** | Segment（TCP）/ Datagram（UDP） |
| **网络层** | Packet（IP数据报） |
| **数据链路层** | Frame（帧） |
| **物理层** | Bit（比特） |

## 考试要点

### 高频考点
1. **五层模型 vs OSI vs TCP/IP** — 各层的对比和关系
2. **各层的功能** — 每层解决的核心问题
3. **数据封装与解封装** — 每层的PDU
4. **协议数据单元** — 报文、段、包、帧、比特

### 常见陷阱
- 网际层 = 网络层
- 传输层是唯一实现端到端通信的层次
- 数据链路层的PDU是帧，网络层的PDU是数据包
