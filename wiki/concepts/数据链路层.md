# 数据链路层（Data Link Layer）

## 数据链路层概述

### 数据链路层的功能

#### 1. 为网络层提供服务
- 无确认无连接
- 有确认无连接
- 有确认有连接

#### 2. 链路管理
- 连接的建立
- 连接的维持
- 连接的释放

#### 3. 组帧
- 将网络层的数据封装成帧

#### 4. 流量控制
- 协调发送方和接收方的速度

#### 5. 差错控制
- 检测和纠正传输错误

## 组帧

### 什么是组帧？
将网络层的数据封装成数据链路层的帧

### 帧的组成部分
```
┌──────────┬──────────────────┬──────────┐
│ 帧首部   │ 数据部分          │ 帧尾部   │
└──────────┴──────────────────┴──────────┘
```

### 组帧的四种方法

#### 1. 字符计数法
- 在帧首部使用一个字段，以字符为单位表示帧的长度

#### 2. 字符填充法
- 使用特殊字符（DLE、SOH、EOT）作为帧定界符
- 数据中出现特殊字符时，进行转义

#### 3. 零比特填充法
- 使用**标志字段**（01111110）作为帧定界符
- 数据中出现5个连续1时，插入一个0
- 接收端删除插入的0

#### 4. 违规编码法
- 使用物理层编码中不会出现的码字作为定界符

## 差错控制

### 什么是差错？
数据传输过程中，比特位发生翻转

### 差错的类型

#### 1. 突发差错
- 多个连续比特出错

#### 2. 随机差错
- 单个或少数比特出错

### 差错检测方法

#### 1. 奇偶校验
- 添加一个校验位
- 使1的个数为奇数（奇校验）或偶数（偶校验）
- **缺点**：只能检测奇数位错误

#### 2. 循环冗余校验（CRC，Cyclic Redundancy Check）

##### 原理
- 发送端：$T(x) = x^n M(x) \div G(x)$
- 接收端：$R(x) = x^n M(x) + T(x)$，若$R(x) \div G(x) = 0$，则正确

##### 步骤
1. 选择生成多项式$G(x)$
2. 发送端计算校验码$T(x) = x^n M(x) \div G(x)$
3. 发送数据：$x^n M(x) + T(x)$
4. 接收端用$G(x)$除接收到的数据，若余数为0，则正确

##### 特点
- 可以检测所有单比特错误
- 可以检测所有双比特错误
- 可以检测所有奇数位错误
- 可以检测所有长度小于n的突发错误

### 纠错方法

#### 1. 前向纠错（FEC，Forward Error Correction）
- 接收端检测并纠正错误
- 需要发送额外的冗余信息

#### 2. 自动请求重传（ARQ，Automatic Repeat reQuest）
- 接收端检测错误，请求发送端重传
- 需要反馈信道

## 流量控制

### 什么是流量控制？
协调发送方和接收方的速度，避免接收方缓冲区溢出

### 数据链路层的流量控制

#### 1. 停止-等待协议
- 发送方发送一帧后，等待接收方确认
- 发送下一帧

#### 2. 滑动窗口协议

##### 后退N帧协议（GBN，Go-Back-N）
- 发送窗口 > 1
- 接收窗口 = 1
- 出错时，重传出错帧及之后的所有帧

##### 选择重传协议（SR，Selective Repeat）
- 发送窗口 > 1
- 接收窗口 > 1
- 只重传出错帧

#### 滑动窗口协议对比

| 协议 | 发送窗口 | 接收窗口 | 重传范围 | 效率 |
|------|---------|---------|---------|------|
| **停止-等待** | 1 | 1 | 出错帧 | 低 |
| **GBN** | >1 | 1 | 出错帧及之后 | 中 |
| **SR** | >1 | >1 | 仅出错帧 | 高 |

## PPP协议（点对点协议）

### 什么是PPP？
PPP（Point-to-Point Protocol）是**点对点**的数据链路层协议

### PPP的特点
- **简单**：提供不可靠的数据链路服务
- **封装成帧**：使用字符填充法
- **透明性**：采用零比特填充法
- **多种网络层协议**：支持多种网络层协议
- **多种类型链路**：串行、并行、同步、异步
- **差错检测**：使用CRC
- **连接管理**：支持网络层地址协商
- **网络层地址协商**：知道对方的网络层地址
- **数据压缩协商**：可选的数据压缩
- **身份验证**：支持PAP和CHAP

### PPP的帧格式

```
┌──────┬──────┬──────┬──────────┬──────────┬──────┐
│ 标志  │ 地址  │ 控制  │  协议     │  数据     │ FCS  │ 标志 │
│ 7E   │ FF   │ 03   │  2字节   │  变长     │ 2字节│ 7E  │
└──────┴──────┴──────┴──────────┴──────────┴──────┘
```

#### 字段说明
- **标志字段**：01111110（0x7E），帧定界符
- **地址字段**：11111111（0xFF），广播地址
- **控制字段**：00000011（0x03），无意义
- **协议字段**：标识数据字段的类型
  - 0x0021：IP数据报
  - 0xC021：LCP数据
  - 0xC023：PAP认证
  - 0xC223：CHAP认证

### 零比特填充
- 发送端：遇到5个连续1，插入1个0
- 接收端：遇到5个连续1，删除后面的0

## CSMA/CD（载波监听多路访问/冲突检测）

### 什么是CSMA/CD？
**总线型局域网**中使用的介质访问控制方法

### CSMA/CD的工作原理

#### 1. 载波监听（Carrier Sense）
- 发送前先监听信道
- 信道空闲才发送

#### 2. 多路访问（Multiple Access）
- 多个站点共享总线

#### 3. 冲突检测（Collision Detection）
- 发送时同时监听信道
- 检测到冲突立即停止发送
- 等待随机时间后重试

### CSMA/CD的流程

```
1. 适配器从网络层获得分组
2. 封装成以太网帧
3. 监听信道：
   ├── 信道空闲 → 立即发送
   └── 信道忙 → 持续监听直到空闲
4. 发送过程中持续监听
   ├── 未发生冲突 → 发送完成
   └── 发生冲突 → 立即停止发送
       ↓
   发送一个强化冲突的Jam信号
       ↓
   等待随机时间（二进制指数退避）
       ↓
   重新尝试发送（最多16次）
```

### 二进制指数退避算法

#### 基本退避时间
$$2\tau = 2 \times 往返时间 = 2 \times 争用期$$

#### 从[0, 2^k-1]中随机选择一个数r
- k = min(重传次数, 10)

#### 等待时间
$$r \times 2\tau$$

#### 重传次数限制
- 最多重传16次
- 16次后丢弃帧，向上报告错误

### 争用期（Contention Period）
- 以太网中，争用期 = 51.2μs
- 10Mbps以太网，争用期内发送64字节

### 以太网帧格式

```
┌──────┬──────┬──────┬──────────┬──────┐
│ 目的地址 │ 源地址 │ 类型  │  数据     │ FCS  │
│ 6字节  │ 6字节 │ 2字节 │ 46-1500字节│ 4字节│
└──────┴──────┴──────┴──────────┴──────┘
```

#### 字段说明
- **目的地址**：6字节MAC地址
- **源地址**：6字节MAC地址
- **类型**：标识上层协议（0x0800=IP，0x86DD=IPv6）
- **数据**：46-1500字节（MTU=1500）
- **FCS**：帧检验序列（CRC）

### 最短帧长
$$最短帧长 = 64字节 = 512bit$$

- 目的地址6字节 + 源地址6字节 + 类型2字节 + 数据46字节 + FCS 4字节
- 保证在争用期内能够检测到冲突

### 最大帧长
$$最大帧长 = 1518字节 = 12144bit$$

## 考试要点

### 高频考点
1. **组帧方法** — 字符计数法、字符填充法、零比特填充法
2. **CRC校验** — 生成多项式、校验码计算
3. **CSMA/CD** — 工作原理、二进制指数退避
4. **以太网帧格式** — 帧格式、最短帧长、MTU

### 常见陷阱
- 最短帧长64字节（确保在争用期内能检测到冲突）
- MTU=1500字节
- 零比特填充法用于PPP协议
- 二进制指数退避的随机数范围
