# 串（String）

## Definition

**串（String）**是由零个或多个字符组成的有限序列。

串是数据元素为单个字符的特殊线性表。

## 串的基本概念

### 串的名称
- 串中字符的个数称为**串的长度**
- 由一个或多个空格组成的串称为**空格串**
- 零个字符的串称为**空串（ε）**

### 串的比较
两个串相等当且仅当两个串的**长度相等**，并且**每个对应位置的字符都相同**

### 串的存储结构

#### 1. 定长顺序存储
```c
#define MaxSize 255
typedef struct {
    char data[MaxSize + 1];  // 存储字符
    int length;              // 串的实际长度
} SqString;
```

#### 2. 堆分配存储
```c
typedef struct {
    char *ch;   // 指向字符数组的指针
    int length; // 串的实际长度
} HString;
```

#### 3. 块链存储
- 每个块可以存放多个字符
- 最后一个块可以存不满

## 串的基本操作

### 赋值操作
```c
bool StrAssign(HString &T, char *chars);
// 生成一个值为chars的串T
```

### 求串的长度
```c
int StrLength(HString S);
// 返回串S的元素个数
```

### 串比较
```c
int StrCompare(HString S, HString T);
// 若S>T，返回值>0
// 若S=T，返回值=0
// 若S<T，返回值<0
```

### 串的连接
```c
bool Concat(HString &T, HString S1, HString S2);
// 将串S1和S2连接起来，并用T返回新串
```

### 求子串
```c
bool SubString(HString &Sub, HString S, int pos, int len);
// 用Sub返回串S的第pos个字符开始，长度为len的子串
```

### 串的模式匹配

#### 1. 暴力匹配算法（Brute Force，BF算法）

**思想**：从主串的每一个字符开始，依次与模式串比较

```c
int BF(char *S, char *T) {
    int i = 0, j = 0;
    while (S[i] != '\0' && T[j] != '\0') {
        if (S[i] == T[j]) {
            i++;
            j++;
        } else {
            i = i - j + 1;  // i回溯
            j = 0;          // j回溯
        }
    }
    if (T[j] == '\0')
        return i - j;  // 匹配成功，返回起始位置
    else
        return -1;     // 匹配失败
}
```

**时间复杂度**：
- 最好情况：O(n+m)
- 最坏情况：O(n×m)（n为主串长度，m为模式串长度）

#### 2. KMP算法（改进的模式匹配算法）

**思想**：利用模式串的部分匹配信息，避免主串指针的回溯

##### next数组的计算
`next[i]`表示模式串的前i个字符中，最长公共前后缀的长度

```c
void getNext(char *T, int *next) {
    int i = 0, j = -1;
    next[0] = -1;
    
    while (T[i] != '\0') {
        if (j == -1 || T[i] == T[j]) {
            i++;
            j++;
            next[i] = j;
        } else {
            j = next[j];  // j回溯
        }
    }
}
```

##### KMP匹配算法
```c
int KMP(char *S, char *T) {
    int i = 0, j = 0;
    int next[100];
    
    getNext(T, next);  // 先计算next数组
    
    while (S[i] != '\0' && T[j] != '\0') {
        if (j == -1 || S[i] == T[j]) {
            i++;
            j++;
        } else {
            j = next[j];  // 利用next数组，j不回溯到-1
        }
    }
    
    if (T[j] == '\0')
        return i - j;
    else
        return -1;
}
```

**时间复杂度**：O(n+m)
- 主串扫描：O(n)
- 模式串预处理：O(m)

##### next数组优化
```c
// 优化的next数组（nextVal）
void getNextVal(char *T, int *nextVal) {
    int i = 0, j = -1;
    nextVal[0] = -1;
    
    while (T[i] != '\0') {
        if (j == -1 || T[i] == T[j]) {
            i++;
            j++;
            
            if (T[i] != T[j]) {
                nextVal[i] = j;
            } else {
                nextVal[i] = nextVal[j];
            }
        } else {
            j = nextVal[j];
        }
    }
}
```

## KMP算法详解

### next数组的含义
`next[j]`是模式串前j个字符组成的子串中，**最长相等前后缀的长度**

### next数组的例子
```
模式串 T = "ababaa"
next数组：-1 0 0 1 2 3
位置：     0  1 2 3 4 5

分析：
- j=0: next[0] = -1
- j=1: "a"的最长相等前后缀 = 0
- j=2: "ab"的最长相等前后缀 = 0
- j=3: "aba"的最长相等前后缀 = 1（"a"）
- j=4: "abab"的最长相等前后缀 = 2（"ab"）
- j=5: "ababa"的最长相等前后缀 = 3（"aba"）
```

### KMP算法的核心思想
- 当匹配失败时，i指针不动，j指针利用next数组进行移动
- 避免了主串的回溯，提高了效率

## 考试要点

### 高频考点
1. **BF算法的时间复杂度** — 最坏情况O(n×m)
2. **KMP算法的时间复杂度** — O(n+m)
3. **next数组的计算** — 理解前缀和后缀的概念
4. **KMP vs BF** — KMP为什么更快？

### 常见陷阱
- next数组下标从0还是1开始（王道教材用1-based）
- next数组和nextVal数组的区别
- KMP算法j回退的理解
