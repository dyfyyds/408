# 分页存储管理

> 将内存和进程空间划分为大小相同的块，通过页表实现地址转换。详见 [[内存管理]]。

---

## 基本原理

- 内存和进程空间都划分为**大小相同**的块（页框/页帧）
- 进程空间划分的块称为**页**
- 页面大小通常为 4KB、8KB、16KB（2的幂次）

## 地址转换

```
逻辑地址 = 页号 + 页内偏移
  ↓
页表查询 → 页框号 + 页内偏移
  ↓
物理地址 = 页框号 × 页面大小 + 页内偏移
```

## 页表

| 字段 | 说明 |
|------|------|
| 页框号 | 该页在内存中的位置 |
| 存在位 | 该页是否在内存中 |
| 访问位 | 该页是否被访问过 |
| 修改位 | 该页是否被修改过 |
| 保护位 | 读写权限 |

## 两级页表

- 解决单级页表占用空间大的问题

```
页目录 → 页表 → 物理页框
```

## 快表（TLB，Translation Lookaside Buffer）

- 页表的高速缓存
- 存放最近访问的页表项
- 命中率可达 90% 以上
