# 进程同步与通信

## 进程同步的基本概念

### 为什么需要进程同步？
多个进程并发执行时，由于速度不可预知，可能导致执行结果不确定

### 进程同步与互斥

#### 进程同步
多个进程因一定合作关系，需要协调先后顺序

#### 进程互斥
多个进程因争抢临界资源，不能同时进入临界区

## 临界资源和临界区

### 临界资源（Critical Resource）
一次只允许一个进程访问的资源

**示例**：打印机、共享变量

### 临界区（Critical Section）
访问临界资源的代码段

### 临界区管理的原则

#### 1. 空闲让进
临界区空闲时，允许一个进程进入

#### 2. 忙则等待
临界区有进程时，其他进程必须等待

#### 3. 有限等待
等待时间有限，不能无限等待

#### 4. 让权等待
进程不能进入临界区时，释放CPU

## 实现进程互斥的方法

### 1. 软件方法

#### 单标志法
```c
int flag[2]; // flag[0]和flag[1]分别表示两个进程的标志
int turn = 0; // 轮流标志

// P0进程
while (flag[0] == 1); // 等待
flag[0] = 1;
// 临界区
turn = 1;
flag[0] = 0;

// P1进程
while (flag[1] == 1); // 等待
flag[1] = 1;
// 临界区
turn = 0;
flag[1] = 0;
```
**问题**：存在空闲让进问题

#### 双标志法
```c
bool flag[2];

// P0进程
while (flag[1]); // 等待P1不进入
flag[0] = true;
// 临界区
flag[0] = false;

// P1进程
while (flag[0]); // 等待P0不进入
flag[1] = true;
// 临界区
flag[1] = false;
```
**问题**：存在忙等问题

#### 双标志后检查法
```c
bool flag[2];

// P0进程
flag[0] = true;
while (flag[1]); // 检查P1是否要进入
// 临界区
flag[0] = false;

// P1进程
flag[1] = true;
while (flag[0]); // 检查P0是否要进入
// 临界区
flag[1] = false;
```
**问题**：存在饥饿问题

#### Peterson算法
```c
bool flag[2];
int turn = 0;

// P0进程
flag[0] = true;
turn = 1;
while (flag[1] && turn == 1);
// 临界区
flag[0] = false;

// P1进程
flag[1] = true;
turn = 0;
while (flag[0] && turn == 0);
// 临界区
flag[1] = false;
```
**优点**：解决了上述所有问题

### 2. 硬件方法

#### 中断屏蔽
```c
// 进入临界区
关闭中断（Disable Interrupt）
// 临界区代码
// 离开临界区
开中断（Enable Interrupt）
```
- **优点**：简单高效
- **缺点**：不适合多CPU，可能影响系统时钟

#### TestAndSet指令（TS指令）
```c
bool TestAndSet(bool *lock) {
    bool old = *lock;
    *lock = true;
    return old;
}

// 使用
while (TestAndSet(&lock)); // 等待
// 临界区
lock = false;
```

#### Swap指令（XCHG指令）
```c
void Swap(bool *lock, bool *old) {
    bool temp = *lock;
    *lock = *old;
    *old = temp;
}

// 使用
bool key = true;
do {
    Swap(&lock, &key);
} while (key);
// 临界区
lock = false;
```

## 信号量机制

### 什么是信号量？
信号量是操作系统提供的一种进程同步机制

### 信号量的定义
```c
typedef struct {
    int value;           // 资源数
    struct process *list; // 等待队列
} semaphore;
```

### P/V操作

#### P操作（wait，down，proberen）
- 申请资源
- value减1
- 若value < 0，进程进入等待队列

#### V操作（signal，up，verhogen）
- 释放资源
- value加1
- 若value ≤ 0，唤醒等待队列中的一个进程

### 信号量实现进程互斥
```c
semaphore mutex = 1; // 互斥信号量

// P0进程
P(mutex);  // 进入临界区前
// 临界区
V(mutex);  // 离开临界区后

// P1进程
P(mutex);  // 进入临界区前
// 临界区
V(mutex);  // 离开临界区后
```

### 信号量实现进程同步

#### 同步：A在B之前执行
```c
semaphore S = 0; // 同步信号量

// A进程
// A的一些操作
V(S); // 通知B可以执行

// B进程
P(S); // 等待A完成
// B的操作
```

## 经典同步问题

### 1. 生产者-消费者问题

#### 问题描述
- 生产者生产产品放入缓冲区
- 消费者从缓冲区取产品消费
- 缓冲区大小为n

#### 代码实现
```c
semaphore mutex = 1;    // 互斥信号量
semaphore empty = n;    // 空缓冲区数量
semaphore full = 0;     // 满缓冲区数量

// 生产者进程
while (true) {
    // 生产一个产品
    P(empty);    // 空缓冲区减1
    P(mutex);    // 进入临界区
    // 将产品放入缓冲区
    V(mutex);    // 离开临界区
    V(full);     // 满缓冲区加1
}

// 消费者进程
while (true) {
    P(full);     // 满缓冲区减1
    P(mutex);    // 进入临界区
    // 从缓冲区取产品
    V(mutex);    // 离开临界区
    V(empty);    // 空缓冲区加1
    // 消费产品
}
```

#### 关键点
- 必须使用三个信号量
- P操作顺序不能颠倒
- V操作顺序可以颠倒

### 2. 读者-写者问题

#### 问题描述
- 多个读者可以同时读
- 只有一个写者可以写
- 读写互斥，写写互斥

#### 代码实现
```c
semaphore rmutex = 1;   // 读者计数互斥
semaphore wmutex = 1;   // 写者互斥
int readcount = 0;      // 读者计数

// 读者进程
while (true) {
    P(rmutex);         // 读者计数互斥
    readcount++;
    if (readcount == 1)
        P(wmutex);     // 第一个读者写锁
    V(rmutex);
    
    // 读操作
    
    P(rmutex);
    readcount--;
    if (readcount == 0)
        V(wmutex);     // 最后一个读者解锁
    V(rmutex);
}

// 写者进程
while (true) {
    P(wmutex);         // 写者互斥
    // 写操作
    V(wmutex);
}
```

### 3. 哲学家进餐问题

#### 问题描述
- 5个哲学家，5根筷子围成一圈
- 哲学家左右各拿一根筷子才能吃饭

#### 代码实现
```c
semaphore chopstick[5] = {1, 1, 1, 1, 1};

// 哲学家i
while (true) {
    // 思考
    
    P(chopstick[i]);        // 拿左边的筷子
    P(chopstick[(i+1)%5]);  // 拿右边的筷子
    // 吃饭
    V(chopstick[i]);        // 放左边的筷子
    V(chopstick[(i+1)%5]);  // 放右边的筷子
}
```

**死锁问题**：如果每个哲学家都拿起左边的筷子，就会死锁

**解决方案**：
1. 最多允许4个哲学家同时拿筷子
2. 奇数号哲学家先拿左边，偶数号先拿右边
3. 同时拿两根筷子，否则一根都不拿

## 管程（Monitor）

### 什么是管程？
管程是一种用于进程同步的高级语言结构

### 管程的特点
- **互斥性**：管程内的过程一次只允许一个进程执行
- **同步性**：使用条件变量实现进程同步

### 条件变量
```c
condition x, y;

// 等待
x.wait();
// 唤醒
x.signal();
```

### 管程实现生产者-消费者
```c
monitor ProducerConsumer {
    int count = 0;
    condition full, empty;
    
    void produce() {
        // 生产产品
        if (count == n)
            full.wait();
        // 放入产品
        count++;
        if (count == 1)
            empty.signal();
    }
    
    void consume() {
        // 消费产品
        if (count == 0)
            empty.wait();
        // 取出产品
        count--;
        if (count == n-1)
            full.signal();
    }
}
```

## 考试要点

### 高频考点
1. **P/V操作** — 信号量的定义和P/V操作
2. **生产者-消费者问题** — 三个信号量的使用
3. **读者-写者问题** — 读者计数和读写互斥
4. **哲学家进餐问题** — 死锁预防

### 常见陷阱
- P/V操作必须成对出现
- 生产者-消费者需要三个信号量（mutex、empty、full）
- 读者-写者问题中，第一个读者加写锁，最后一个读者解锁
