# CPU与指令系统

## CPU的功能与组成

### CPU的功能
1. **指令控制**：完成取指令、分析指令、执行指令
2. **操作控制**：产生操作信号，控制其他部件
3. **时间控制**：对各种操作进行时间上的定时
4. **数据加工**：执行算术运算和逻辑运算

### CPU的基本组成

```
┌────────────────────────────────┐
│           控制器(CU)           │
├────────────┬───────────────────┤
│ 运算器(ALU) │   寄存器          │
├────────────┴───────────────────┤
│       内部总线                  │
└────────────────────────────────┘
```

1. **运算器（ALU）**：执行算术运算和逻辑运算
2. **控制器（CU）**：分析指令，产生控制信号
3. **寄存器**：高速暂存数据
4. **内部总线**：连接各部件

## 指令系统

### 指令的基本格式

#### 指令 = 操作码 + 地址码

```
┌──────────┬──────────────────┐
│ 操作码   │    地址码         │
└──────────┴──────────────────┘
```

- **操作码**：指明操作的性质和功能
- **地址码**：指明操作数的位置

### 操作码的编码

#### 1. 固定长度编码
- 所有操作码长度相同
- 优点：译码简单，速度快
- 缺点：操作码少，浪费空间

#### 2. 可变长度编码
- 操作码长度可变
- 优点：充分利用指令字长，操作码多
- 缺点：译码复杂，速度慢

### 地址码的个数

| 类型 | 格式 | 描述 |
|------|------|------|
| **零地址指令** | 只有操作码 | 不需要操作数 |
| **一地址指令** | 操作码 + 地址码 | 只需要一个操作数 |
| **二地址指令** | 操作码 + 地址码1 + 地址码2 | 需要两个操作数 |
| **三地址指令** | 操作码 + 地址码1 + 地址码2 + 结果地址 | 需要两个操作数和一个结果地址 |
| **四地址指令** | 操作码 + 地址码1 + 地址码2 + 结果地址 + 下地址 | 需要两个操作数、一个结果地址和下地址 |

### 指令长度
- 指令字长 = 操作码长度 + 地址码长度
- 等长指令：所有指令长度相同（如x86）
- 变长指令：指令长度可变（如ARM、RISC-V）

## 寻址方式

### 定义
寻址方式是指确定本条指令的数据地址以及下一条待执行指令的地址的方法

### 数据寻址方式

#### 1. 立即寻址
```
操作数 = 地址码字段的值

例：MOV AX, #1234H  ; 立即数1234H送入AX
```
- **优点**：指令执行速度快，无需访问内存
- **缺点**：操作数长度受地址码长度限制

#### 2. 直接寻址
```
操作数地址 = 地址码字段的值

例：MOV AX, [2000H]  ; 从内存2000H单元取数送AX
```
- **优点**：简单，执行速度快
- **缺点**：操作数地址受地址码长度限制

#### 3. 间接寻址
```
操作数地址 = (地址码字段的值)

例：MOV AX, [[2000H]]  ; 内存2000H单元的内容是地址，再取该地址的数送AX
```
- **优点**：扩大寻址范围
- **缺点**：执行速度慢，需两次访问内存

#### 4. 寄存器寻址
```
操作数 = 寄存器的值

例：MOV AX, BX  ; BX的内容送AX
```
- **优点**：速度快，指令短
- **缺点**：寄存器数量有限

#### 5. 寄存器间接寻址
```
操作数地址 = 寄存器的值

例：MOV AX, [BX]  ; BX的内容是地址，从该地址取数送AX
```

#### 6. 基址变址寻址
```
有效地址 = 基址寄存器 + 变址寄存器 + 形式地址

例：MOV AX, [BX + SI + 100H]
```

#### 7. 相对寻址
```
有效地址 = PC + 形式地址

例：JMP 2000H  ; 跳转到当前PC+2000H处
```
- 常用于转移指令

#### 8. 堆栈寻址
```
操作数 = 栈顶元素
```

### 寻址方式对比

| 寻址方式 | 操作数位置 | 执行速度 | 寻址范围 |
|---------|---------|---------|---------|
| **立即寻址** | 指令中 | 最快 | 受限 |
| **直接寻址** | 内存中 | 快 | 受限 |
| **间接寻址** | 内存中 | 慢 | 大 |
| **寄存器寻址** | 寄存器 | 最快 | 受限 |
| **寄存器间接寻址** | 内存中 | 快 | 受限 |
| **相对寻址** | 内存中 | 快 | 大 |

## 指令流水线

### 为什么需要流水线？

#### 串行执行的问题
- 取指 → 译码 → 执行 → 访存 → 写回
- 每个阶段都需要一个时钟周期
- 执行n条指令需要n×5个时钟周期

#### 流水线的思想
- 将指令执行过程分成多个阶段（流水级）
- 每个阶段由专门的硬件电路完成
- 理想情况下，每个时钟周期完成一条指令

### 五级流水线

```
取指（IF）→ 译码（ID）→ 执行（EX）→ 访存（MEM）→ 写回（WB）
```

#### 各级的功能
1. **取指（IF）**：从内存取指令到IR
2. **译码（ID）**：分析指令，读寄存器
3. **执行（EX）**：执行运算或计算地址
4. **访存（MEM）**：访问内存
5. **写回（WB）**：结果写回寄存器

### 流水线的性能

#### 理想情况
- n条指令执行时间 = 5 + (n-1) = n + 4 个时钟周期
- **加速比** = n / (n+4) ≈ 5

#### 实际性能
- 受**冒险（Hazard）**影响
- 需要插入气泡（bubble）来避免冲突

### 流水线的冒险与冲突

#### 1. 结构冲突（Structural Hazard）
- 硬件资源不足导致

**解决**：资源重复设置

#### 2. 数据冲突（Data Hazard）
- 下一条指令依赖上一条指令的结果

**解决**：
- **前推/旁路（Forwarding/Bypassing）**：不等写回，提前转发
- **插入气泡（Stall）**：插入NOP指令
- **调度指令**：编译器优化指令顺序

#### 3. 控制冲突（Control Hazard）
- 转移指令导致

**解决**：
- **预测分支**：猜测是否跳转
- **延迟转移**：转移指令后面插入有效指令
- **分支历史表（BHT）**

### 超标量流水线

- 每个时钟周期同时执行多条指令
- 需要多个功能部件

### 超流水线

- 将流水级进一步细分
- 流水级更多，时钟周期更短

## 考试要点

### 高频考点
1. **寻址方式** — 立即、直接、间接、寄存器寻址的特点
2. **指令格式** — 操作码和地址码的组成
3. **流水线** — 五级流水线的工作原理
4. **流水线冒险** — 结构冲突、数据冲突、控制冲突

### 常见陷阱
- 相对寻址的基准地址是PC的内容
- 立即寻址的操作数直接在指令中
- 间接寻址需要两次访问内存
- 流水线加速比 < 流水级数（受冲突影响）

### 真题示例
- 2010年408真题：各种寻址方式的特点
- 2016年408真题：指令流水线的性能计算
