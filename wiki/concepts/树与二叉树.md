# 树与二叉树（Tree & Binary Tree）

## Definition

**树（Tree）**是n个节点的有限集合，n=0时称为空树。

**二叉树（Binary Tree）**是每个节点最多有两个子树的有序树，左右子树不能颠倒。

## 树的基本概念

### 树的定义
- 有且仅有一个特定的根节点（Root）
- 除根节点外，其余节点分为互不相交的子树

### 树的基本术语
| 术语 | 定义 |
|------|------|
| **父节点** | 有子树的节点是其子树的根节点的父节点 |
| **子节点** | 一个节点含有的子树的根节点称为该节点的子节点 |
| **度** | 一个节点含有的子树的个数称为该节点的度 |
| **叶子节点** | 度为0的节点称为叶子节点或终端节点 |
| **分支节点** | 度不为0的节点称为分支节点或非终端节点 |
| **孩子节点** | 节点的子树的根称为该节点的孩子节点 |
| **兄弟节点** | 同一个父节点的子节点之间互称兄弟节点 |

### 树的存储结构
1. **双亲表示法** — 适合查找父节点
2. **孩子表示法** — 适合查找孩子节点
3. **孩子兄弟表示法（二叉树表示法）** — 可表示任意树

## 二叉树的基本概念

### 二叉树的性质

#### 性质1
在二叉树的第i层上至多有$2^{i-1}$个节点（i≥1）

#### 性质2
深度为k的二叉树至多有$2^k-1$个节点（k≥1）

#### 性质3
对任何一棵二叉树T，若其终端节点数为$n_0$，度为2的节点数为$n_2$，则
$$n_0 = n_2 + 1$$

#### 性质4
具有n个节点的完全二叉树的深度为
$$\lfloor log_2 n \rfloor + 1$$

#### 性质5
如果对一棵有n个节点的完全二叉树（深度为$\lfloor log_2 n \rfloor + 1$）的节点按层序编号，则：
- 若i=1，则节点i是二叉树的根
- 若i>1，则节点i的父节点是$\lfloor i/2 \rfloor$
- 若2i>n，则节点i无左孩子，否则左孩子是2i
- 若2i+1>n，则节点i无右孩子，否则右孩子是2i+1

### 满二叉树与完全二叉树

#### 满二叉树（Full Binary Tree）
- 一颗深度为k且有$2^k-1$个节点的二叉树
- 每层都"满"，没有空缺

#### 完全二叉树（Complete Binary Tree）
- 深度为k的二叉树，除第k层外，其余各层都是满的
- 第k层的节点都集中在左边
- 可以用顺序存储结构（数组）

## 二叉树的遍历

### 前序遍历（PreOrder：根-左-右）
```c
void PreOrder(BiTree T) {
    if (T) {
        visit(T);              // 访问根节点
        PreOrder(T->lchild);   // 前序遍历左子树
        PreOrder(T->rchild);   // 前序遍历右子树
    }
}
```

### 中序遍历（InOrder：左-根-右）
```c
void InOrder(BiTree T) {
    if (T) {
        InOrder(T->lchild);    // 中序遍历左子树
        visit(T);              // 访问根节点
        InOrder(T->rchild);    // 中序遍历右子树
    }
}
```

### 后序遍历（PostOrder：左-右-根）
```c
void PostOrder(BiTree T) {
    if (T) {
        PostOrder(T->lchild);  // 后序遍历左子树
        PostOrder(T->rchild);  // 后序遍历右子树
        visit(T);              // 访问根节点
    }
}
```

### 层序遍历（LevelOrder）
```c
void LevelOrder(BiTree T) {
    LinkQueue Q;
    InitQueue(Q);
    BiTree p;
    
    EnQueue(Q, T);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, p);
        visit(p);
        
        if (p->lchild) EnQueue(Q, p->lchild);
        if (p->rchild) EnQueue(Q, p->rchild);
    }
}
```

### 遍历顺序与表达式
```
树结构：
    A
   / \
  B   C

前序：A → B → C
中序：B → A → C
后序：B → C → A
```

### 根据遍历序列构造二叉树
- 必须提供**中序序列**和**任意其他一种遍历序列**
- 前序+中序 → 可以唯一确定二叉树
- 后序+中序 → 可以唯一确定二叉树
- 前序+后序 → **不能**唯一确定二叉树

## 线索二叉树

### 为什么需要线索二叉树？
- 二叉树遍历需要O(n)时间复杂度
- 如果频繁需要找前驱后继，效率较低
- 利用空指针域，存储前驱/后继信息

### 线索二叉树的节点结构
```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;  // 左右线索标志
    // ltag=0：lchild指向左孩子
    // ltag=1：lchild指向前驱
    // rtag=0：rchild指向右孩子
    // rtag=1：rchild指向后继
} ThreadNode, *ThreadTree;
```

### 线索化的过程
```c
void InThreading(ThreadTree &p, ThreadTree &pre) {
    if (p) {
        InThreading(p->lchild, pre);  // 线索化左子树
        
        if (!p->lchild) {
            p->lchild = pre;  // 前驱线索
            p->ltag = 1;
        }
        if (pre && !pre->rchild) {
            pre->rchild = p;  // 后继线索
            pre->rtag = 1;
        }
        pre = p;  // 更新pre
        
        InThreading(p->rchild, pre);  // 线索化右子树
    }
}
```

## 二叉搜索树（BST）

### 定义
对于树中的任意节点：
- 左子树所有节点的值 < 当前节点值
- 右子树所有节点的值 > 当前节点值
- 左右子树也都是二叉搜索树

### 中序遍历二叉搜索树
得到有序序列（升序）

### 查找
```c
// 递归查找
BiTree SearchBST(BiTree T, KeyType key) {
    if (!T || T->data.key == key)
        return T;
    if (key < T->data.key)
        return SearchBST(T->lchild, key);
    else
        return SearchBST(T->rchild, key);
}
```

## 平衡二叉树（AVL树）

### 定义
任意节点的左子树和右子树的高度差的绝对值不超过1

### 平衡因子（BF）
- BF = 左子树高度 - 右子树高度
- 平衡因子只能是：-1、0、1

### 不平衡的四种情况
1. **LL型** — 右旋
2. **RR型** — 左旋
3. **LR型** — 先左旋，再右旋
4. **RL型** — 先右旋，再左旋

## 哈夫曼树（Huffman Tree）

### 定义
带权路径长度（WPL）最小的二叉树

### 哈夫曼编码
- 最优前缀码
- 字符出现频率越高，编码越短
- 编码唯一可译

### 构造哈夫曼树的步骤
1. 构造n个叶子节点，每个节点有权值
2. 选取两个权值最小的节点，合并为一个新节点
3. 新节点的权值 = 两个子节点权值之和
4. 重复步骤2-3，直到只剩一个节点

## 考试要点

### 高频考点
1. **二叉树的性质** — 特别是$n_0 = n_2 + 1$
2. **三种遍历序列** — 前序、中序、后序的相互推导
3. **线索二叉树** — 线索化过程和查找前驱/后继
4. **二叉搜索树** — 查找、插入、删除
5. **哈夫曼树** — WPL计算、哈夫曼编码

### 常见陷阱
- 已知前序+后序不能唯一确定二叉树
- 中序遍历二叉搜索树得到有序序列
- 完全二叉树的节点编号规律

- 2011年408真题：二叉树的中序遍历序列
- 2016年408真题：线索二叉树的前驱后继
