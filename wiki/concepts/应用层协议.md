# 应用层协议（Application Layer）

## 应用层概述

### 应用层的功能
- 直接为用户的应用进程提供服务
- 处理特定的应用程序细节

### 网络应用模型

#### 1. 客户/服务器模型（C/S）
- 服务器：提供服务
- 客户：请求服务

#### 2. 对等网络模型（P2P）
- 每个节点既是客户端又是服务器
- 无中心服务器

## DNS（域名系统）

### 什么是DNS？
将域名解析为IP地址的系统

### 域名结构

```
主机名.三级域名.二级域名.顶级域名

例如：www.tsinghua.edu.cn
```

| 域名 | 说明 |
|------|------|
| **com** | 商业组织 |
| **edu** | 教育机构 |
| **org** | 非营利组织 |
| **gov** | 政府部门 |
| **net** | 网络组织 |
| **cn** | 中国 |

### DNS服务器的层次结构

#### 根域名服务器
- 知道所有顶级域名服务器的IP地址

#### 顶级域名服务器（TLD）
- 负责管理所有二级域名
- 如.com、.edu、.org等

#### 权威域名服务器
- 负责一个区域的域名解析

#### 本地域名服务器
- 每个ISP都有一个本地DNS服务器
- 主机首先向本地DNS服务器查询

### DNS查询过程

#### 迭代查询
```
主机 → 本地DNS → 根DNS → 顶级DNS → 权威DNS
```

#### 递归查询
```
主机 → 本地DNS → 根DNS → 顶级DNS → 权威DNS → 返回结果给主机
```

### DNS报文格式

#### DNS查询报文
```
┌──────┬──────┬──────┬──────┬──────┬──────┐
│ 标识 │ 标志 │ 问题数 │ 回答数 │ 权威数 │ 附加数 │
└──────┴──────┴──────┴──────┴──────┴──────┘
│                   问题区域                    │
├──────────────────────────────────────────────┤
│                   回答区域                    │
├──────────────────────────────────────────────┤
│                   权威区域                    │
├──────────────────────────────────────────────┤
│                   附加区域                    │
└──────────────────────────────────────────────┘
```

### 资源记录（RR）

```
域名 生存时间 类别 类型  RDATA

例如：
www.example.com.  3600  IN  A  192.168.1.1
```

## HTTP（超文本传输协议）

### 什么是HTTP？
HTTP是Web应用层协议，用于传输超文本

### HTTP的特点
- **无状态**：每个请求都是独立的
- **可靠传输**：基于TCP协议
- **应用层协议**：默认端口80

### HTTP报文格式

#### 请求报文
```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html

```

#### 响应报文
```
HTTP/1.1 200 OK
Date: Mon, 01 Jan 2024 00:00:00 GMT
Content-Type: text/html
Content-Length: 1234

<html>...</html>
```

### HTTP请求方法

| 方法 | 功能 |
|------|------|
| **GET** | 请求指定页面 |
| **POST** | 提交数据（如表单） |
| **PUT** | 上传文件 |
| **DELETE** | 删除指定资源 |
| **HEAD** | 只请求响应头 |
| **OPTIONS** | 查询服务器支持的请求方法 |

### HTTP状态码

| 类别 | 状态码 | 说明 |
|------|--------|------|
| **1xx** | 100-199 | 信息性状态码 |
| **2xx** | 200-299 | 成功状态码 |
| **3xx** | 300-399 | 重定向状态码 |
| **4xx** | 400-499 | 客户端错误状态码 |
| **5xx** | 500-599 | 服务器错误状态码 |

#### 常见状态码

| 状态码 | 说明 |
|--------|------|
| **200 OK** | 请求成功 |
| **301 Moved Permanently** | 永久重定向 |
| **302 Found** | 临时重定向 |
| **304 Not Modified** | 资源未修改（缓存） |
| **400 Bad Request** | 请求错误 |
| **401 Unauthorized** | 未授权 |
| **403 Forbidden** | 禁止访问 |
| **404 Not Found** | 资源不存在 |
| **500 Internal Server Error** | 服务器内部错误 |
| **502 Bad Gateway** | 网关错误 |
| **503 Service Unavailable** | 服务不可用 |

### HTTP/1.0 vs HTTP/1.1 vs HTTP/2.0

#### HTTP/1.0
- 非持久连接（每次请求都需要建立TCP连接）
- 短连接

#### HTTP/1.1
- 持久连接（Keep-Alive）
- 支持管道化（Pipeline）
- 支持Host头（虚拟主机）

#### HTTP/2.0
- 二进制帧（二进制分帧层）
- 多路复用（Multiplexing）
- 头部压缩（HPACK）
- 服务器推送（Server Push）

## HTTPS（安全HTTP）

### 什么是HTTPS？
HTTP + SSL/TLS（安全套接层）

### HTTPS的工作流程

```
客户端 → 服务端：ClientHello（支持的加密算法）
  ↓
服务端 → 客户端：ServerHello + 证书 + ServerHelloDone
  ↓
客户端验证证书
  ↓
客户端 → 服务端：生成随机数（预主密钥），用公钥加密后发送
  ↓
服务端用私钥解密，得到预主密钥
  ↓
双方根据预主密钥生成会话密钥
  ↓
双方确认使用会话密钥
  ↓
后续通信使用会话密钥加密
```

### 对称加密 vs 非对称加密

| 对比维度 | 对称加密 | 非对称加密 |
|---------|---------|-----------|
| **密钥数量** | 一个密钥 | 两个密钥（公钥+私钥） |
| **速度** | 快 | 慢 |
| **安全性** | 密钥分发困难 | 安全性高 |
| **常用算法** | AES、DES、3DES | RSA、ECC |

### SSL/TLS握手
- 握手过程协商加密算法和会话密钥
- 使用非对称加密传输对称密钥
- 后续通信使用对称加密

## FTP（文件传输协议）

### FTP的特点
- **有状态**：需要维护连接状态
- **控制连接**：21端口，传输控制命令
- **数据连接**：20端口，传输文件数据

### FTP的工作模式

#### 主动模式（PORT）
- 客户端监听某个端口，发送PORT命令
- 服务器主动连接客户端的20端口

#### 被动模式（PASV）
- 客户端发送PASV命令
- 服务器监听某个端口，告知客户端
- 客户端主动连接服务器的该端口

### FTP的传输模式
- **ASCII模式**：传输文本文件
- **二进制模式**：传输二进制文件

## SMTP（简单邮件传输协议）

### 什么是SMTP？
用于发送电子邮件的协议

### 端口
- 25端口（SMTP）
- 465端口（SMTPS，加密）

### 邮件发送流程
```
发件人 → SMTP客户端 → SMTP服务器 → DNS查询 → 收件人SMTP服务器 → 收件人
```

## DHCP（动态主机配置协议）

### 什么是DHCP？
自动分配IP地址的协议

### DHCP的工作流程（DORA四步）

```
1. Discover（发现）
   客户端广播：我需要IP地址

2. Offer（提供）
   服务器响应：我给你这个IP地址

3. Request（请求）
   客户端广播：我要使用这个IP地址

4. Acknowledge（确认）
   服务器确认：同意分配
```

### DHCP的续租
- 当租期过了50%时，客户端向服务器请求续租

## 考试要点

### 高频考点
1. **DNS** — 域名解析过程、递归查询 vs 迭代查询
2. **HTTP** — 请求方法、状态码、HTTP/1.0 vs HTTP/1.1
3. **HTTPS** — SSL/TLS握手、对称加密 vs 非对称加密
4. **FTP** — 控制连接 vs 数据连接、主动模式 vs 被动模式
5. **DHCP** — DORA四步过程

### 常见陷阱
- HTTP是无状态协议
- FTP是唯一同时使用两个端口的协议（21和20）
- DNS默认使用UDP，区域传输使用TCP
