# 缓冲技术

> 解决 CPU 与 I/O 设备速度不匹配问题的技术。详见 [[文件系统与IO管理]]。

---

## 为什么需要缓冲？

- 解决速度不匹配问题
- 减少中断次数
- 提高 CPU 和 I/O 设备的并行性

## 缓冲的类型

### 1. 单缓冲
- 在用户区和设备之间设置一个缓冲区
- 缓冲时间：$T = max(C, T') + M$
  - $C$：CPU 计算时间
  - $T'$：I/O 传输时间
  - $M$：从缓冲区到内存的传输时间

### 2. 双缓冲
- 设置两个缓冲区
- 一个缓冲区输入时，另一个缓冲区输出
- 吞吐率提高一倍

### 3. 循环缓冲
- 多个缓冲区组成循环队列
- 适合生产者和消费者速度不匹配的情况

### 4. 缓冲池（Buffer Pool）
- 系统设置多个公共缓冲区
- 可以动态分配
- 提高缓冲区利用率

## SPOOLing 技术

### 思想
- 用磁盘模拟打印机
- 应用程序输出时写入磁盘，打印机慢慢打印

### 组成
1. **输入井**：磁盘上的输入缓冲区
2. **输出井**：磁盘上的输出缓冲区
3. **输入进程**：将输入数据写入输入井
4. **输出进程**：从输出井读取数据输出到打印机

### 特点
- 实现了虚拟设备功能
- 提高了 I/O 速度
- 将独占设备变成共享设备
