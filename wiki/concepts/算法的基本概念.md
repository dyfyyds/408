# 算法的基本概念

## Definition

**算法（Algorithm）**是对特定问题求解步骤的一种描述，是指令的有限序列。

算法是程序的核心，程序 = 数据结构 + 算法

## 算法的5个重要特性

### 1. 有穷性（Finiteness）
- 算法必须在执行有限步之后终止
- 每个步骤都必须在有穷时间内完成

### 2. 确定性（Definiteness）
- 算法的每一步骤必须有确切的定义
- 相同的输入只能得到相同的输出

### 3. 可行性（Feasibility）
- 算法中描述的操作都可以通过已实现的基本运算执行有限次来实现

### 4. 输入（Input）
- 有零个或多个输入
- 取自某个特定的对象的集合

### 5. 输出（Output）
- 有一个或多个输出
- 是算法对输入计算后得到的结果

## 算法设计的要求

### 正确性（Correctness）
- 算法能够正确解决特定的问题
- 通过调试和测试验证

### 可读性（Readability）
- 便于人们理解
- 便于调试和修改

### 健壮性（Robustness）
- 对异常输入的反应和处理能力
- 不会产生莫名其妙的结果

### 效率与低存储量需求
- **时间复杂度**：算法执行时间
- **空间复杂度**：算法执行过程中所需存储空间

## 时间复杂度分析

### 定义
时间复杂度是问题规模 n 的函数，分析算法的时间效率

### 渐进复杂度
- **最坏时间复杂度**：最坏情况下算法的时间复杂度
- **平均时间复杂度**：期望情况下算法的时间复杂度
- **最好时间复杂度**：最好情况下算法的时间复杂度

### 常见时间复杂度
从小到大依次为：
```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

### 例题
```c
// 时间复杂度 O(1)
int a = 10;

// 时间复杂度 O(n)
for(int i = 0; i < n; i++) {
    printf("Hello");
}

// 时间复杂度 O(n²)
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        printf("Hello");
    }
}

// 时间复杂度 O(log n)
for(int i = 1; i < n; i = i * 2) {
    printf("Hello");
}
```

## 空间复杂度分析

### 定义
空间复杂度 = 算法所需存储空间 = 存储原数据 + 辅助空间

### 原地工作（In-place）
若额外空间相对于输入数据量是常数，则称算法是原地工作的

### 例题
```c
// 空间复杂度 O(1)
int sum(int n) {
    int s = 0;
    for(int i = 1; i <= n; i++)
        s += i;
    return s;
}

// 空间复杂度 O(n)
int* createArray(int n) {
    int *arr = (int*)malloc(n * sizeof(int));
    return arr;
}
```

## 考试要点

### 高频考点
1. **时间复杂度的计算** — 循环次数分析
2. **空间复杂度的计算** — 辅助空间分析
3. **最坏、平均、最好时间复杂度** — 理解三者的区别

### 常见陷阱
- 不要只看循环次数，要注意嵌套结构
- 多个输入参数时，时间复杂度可能依赖多个变量
- 递归算法的时间复杂度分析（递归树、主方法）

- 2010年408真题：计算以下代码的时间复杂度
