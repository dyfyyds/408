# Cache原理（高速缓存）

## 为什么需要Cache？

### 速度差异问题
- CPU速度：~1ns（纳秒级）
- 主存速度：~100ns
- **速度差达100倍以上**

### Cache的作用
利用**局部性原理**，将热点数据放在速度更快的Cache中

### 局部性原理

#### 时间局部性
最近访问过的内容，不久后还会被访问

#### 空间局部性
访问某个存储单元后，其附近的存储单元也会被访问

## Cache的基本原理

### Cache与主存的映射

```
CPU访问地址 → 查找Cache
  ├── Cache命中（Hit）→ 直接从Cache读取
  └── Cache未命中（Miss）→ 从主存读取，并加载到Cache
```

### 命中率与平均访问时间

#### 命中率（Hit Rate）
$$命中率 = \frac{Cache命中次数}{总访问次数}$$

#### 平均访问时间
$$t_a = h \times t_c + (1-h) \times t_m$$

- $h$：命中率
- $t_c$：Cache访问时间
- $t_m$：主存访问时间

#### 例题
```
已知：
- Cache访问时间：10ns
- 主存访问时间：100ns
- 命中率：95%

平均访问时间 = 0.95 × 10 + 0.05 × 100 = 9.5 + 5 = 14.5ns

Cache加速比 = 主存访问时间 / 平均访问时间 = 100 / 14.5 ≈ 6.9
```

## Cache的三种映射方式

### 假设条件
- 主存容量：256MB = 2^28 B
- Cache容量：1MB = 2^20 B
- 块大小：64B = 2^6 B

#### 计算
- 块内地址位数：6位
- Cache块数：2^20 / 2^6 = 2^14 = 16384块

### 1. 直接映射（Direct Mapping）

#### 基本思想
每个主存块只能映射到Cache中的一个**特定位置**

#### 映射公式
$$Cache块号 = 主存块号 \pmod{Cache块数}$$

#### 地址格式
```
┌──────────┬──────────┬──────┐
│   标记   │  Cache块号 │ 块内地址 │
│  (Tag)   │  (Index)  │ (Offset)│
└──────────┴──────────┴──────┘

总28位 = 14位标记 + 8位块号 + 6位块内地址
```

#### 工作原理
1. 根据地址的块号字段找到Cache中的对应行
2. 将该行的标记与地址的标记字段比较
3. 若相等且有效位=1，则Cache命中

#### 优缺点
- **优点**：实现简单，查找速度快
- **缺点**：冲突概率高，Cache利用率低

#### 例题
```
例：主存地址为 0000 0000 0000 0000 0000 0000 0000（28位）

块内地址（6位）：000000
Cache块号（8位）：0000 0000
标记（14位）：0000 0000 0000 00

该地址映射到Cache的第0块，标记为0x0000
```

### 2. 全相联映射（Fully Associative Mapping）

#### 基本思想
主存块可以映射到Cache中的**任意位置**

#### 地址格式
```
┌────────────┬──────┐
│    标记    │ 块内地址 │
│   (Tag)    │(Offset)│
└────────────┴──────┘

总28位 = 22位标记 + 6位块内地址
```

#### 工作原理
1. 将地址的标记字段与Cache中**所有行的标记**同时比较
2. 若某行的标记匹配且有效位=1，则Cache命中

#### 优缺点
- **优点**：冲突概率低，Cache利用率高
- **缺点**：硬件复杂，查找慢（需要并行比较所有标记）

#### 应用场景
- 小容量Cache（如TLB）

### 3. 组相联映射（Set Associative Mapping）

#### 基本思想
- 将Cache分成若干**组（Set）**
- 每个组包含若干**块（Line/Way）**
- 主存块先映射到特定组，再映射到组内任意块

#### 映射公式
$$组号 = 主存块号 \pmod{组数}$$

#### 地址格式（2路组相联）
```
┌──────────┬──────────┬──────┐
│   标记   │   组号   │ 块内地址 │
│  (Tag)   │  (Index)  │(Offset)│
└──────────┴──────────┴──────┘

总28位 = 15位标记 + 7位组号 + 6位块内地址
```

#### 工作原理
1. 根据组号字段找到Cache中的对应组
2. 将该组内所有行的标记与地址的标记字段比较
3. 若某行的标记匹配且有效位=1，则Cache命中

#### 常见的组相联
- **2路组相联**：每组2块
- **4路组相联**：每组4块
- **8路组相联**：每组8块

#### 优缺点
- **优点**：折中方案，兼顾速度和利用率
- **缺点**：实现比直接映射复杂，但比全相联简单

#### 实际应用
- 现代CPU的L1、L2、L3 Cache普遍采用组相联

### 三种映射方式对比

| 对比维度 | 直接映射 | 全相联映射 | 组相联映射 |
|---------|---------|-----------|-----------|
| **灵活性** | 差 | 好 | 中 |
| **硬件成本** | 低 | 高 | 中 |
| **查找速度** | 快 | 慢 | 中 |
| **冲突率** | 高 | 低 | 中 |
| **实际应用** | 简单系统 | 小Cache | 现代CPU |

## Cache的写策略

### 写命中（Write Hit）

#### 1. 写直达（Write Through）
- 写Cache的同时，立即写主存
- 优点：Cache与主存一致性好
- 缺点：写操作慢，总线带宽占用大

#### 2. 写回（Write Back）
- 写Cache时，不立即写主存
- 设置**修改位（Dirty Bit）**
- 只有当Cache块被替换时，才写回主存
- 优点：写操作快
- 缺点：需要判断是否修改过

### 写未命中（Write Miss）

#### 1. 写分配（Write Allocate）
- 先加载主存块到Cache
- 再在Cache中修改

#### 2. 写不分配（No Write Allocate）
- 直接写入主存，不加载到Cache

## Cache的替换算法

### 问题
当Cache满时，需要替换某个块

### 1. 先进先出算法（FIFO）
- 优先替换最早进入Cache的块
- 实现简单（使用队列）
- 不符合局部性原理

### 2. 最近最少使用算法（LRU，Least Recently Used）
- 优先替换最长时间未被访问的块
- 符合局部性原理
- 效果最好

#### LRU的实现
- **计数器法**：为每个块设置计数器
- **栈法**：用栈记录访问顺序
- **硬件实现**：使用移位寄存器

#### LRU示例（4路组相联）
```
访问序列：A B C D B A D E B A D

Cache初始状态：[空] [空] [空] [空]

访问A：[A] [空] [空] [空]
访问B：[A] [B] [空] [空]
访问C：[A] [B] [C] [空]
访问D：[A] [B] [C] [D]
访问B：[A] [B] [C] [D] → B最近访问
访问A：[A] [B] [C] [D] → A最近访问
访问D：[A] [B] [C] [D] → D最近访问
访问E：[E] [B] [C] [D] → A被替换（最久未访问）
访问B：[E] [B] [C] [D]
访问A：[E] [A] [C] [D] → B被替换
访问D：[E] [A] [C] [D]
```

### 3. 随机算法（Random）
- 随机选择替换的块
- 实现简单
- 效果不稳定

### 4. 最不经常使用算法（LFU）
- 替换访问次数最少的块
- 需要维护计数器

### 替换算法对比

| 算法 | 实现复杂度 | 效果 | 实际应用 |
|------|-----------|------|---------|
| **FIFO** | 简单 | 差 | 很少使用 |
| **LRU** | 复杂 | 好 | 广泛使用 |
| **Random** | 简单 | 不稳定 | 简单系统 |
| **LFU** | 中等 | 较好 | 部分使用 |

## 多级Cache

### 为什么需要多级Cache？
- 单级Cache容量有限
- 速度与容量的矛盾

### 多级Cache结构
```
L1 Cache（最快，最小，最贵）
  ↓ Miss
L2 Cache（较快，较大）
  ↓ Miss
L3 Cache（较慢，更大）
  ↓ Miss
主存
```

### 各级Cache的特点

| 级别 | 速度 | 容量 | 命中率 |
|------|------|------|--------|
| **L1** | ~1-4 cycles | 32-64KB | ~95% |
| **L2** | ~10-20 cycles | 256KB-2MB | ~98% |
| **L3** | ~40-60 cycles | 4-32MB | ~99% |

## 考试要点

### 高频考点
1. **三种映射方式** — 直接映射、全相联映射、组相联映射
2. **地址格式** — 根据Cache参数计算标记、块号、块内地址
3. **命中率计算** — 平均访问时间、加速比
4. **替换算法** — LRU算法的执行过程

### 常见陷阱
- 块大小一定是2的幂次
- 块内地址位数 = log₂(块大小)
- 直接映射的冲突问题
- LRU需要硬件支持
