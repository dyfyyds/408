# 进程与线程

## 进程的基本概念

### 什么是进程？
进程是操作系统进行资源分配和调度的基本单位

### 进程的实体
进程由**程序段、数据段、PCB（进程控制块）**三部分组成

#### PCB（Process Control Block）
进程控制块是进程存在的唯一标志

##### PCB包含的内容
1. **进程描述信息**
   - 进程标识符PID
   - 用户标识符UID

2. **进程控制和管理信息**
   - 进程当前状态
   - 进程优先级
   - 代码运行入口地址
   - 程序的外存地址
   - 进入内存时间

3. **资源分配信息**
   - 占用内存大小
   - I/O设备
   - 文件

4. **CPU现场信息**
   - 寄存器值
   - 栈指针
   - 程序计数器

### 进程的特征
1. **动态性** — 进程是程序的一次执行过程，有生命周期
2. **并发性** — 多个进程可以同时运行
3. **独立性** — 进程是系统资源分配和调度的独立单位
4. **异步性** — 进程按独立速度执行
5. **结构性** — 每个进程都包含PCB

## 进程的状态与转换

### 进程的三种基本状态

```
      ┌───→ 就绪态
运行态 ─┼───→ 阻塞态
      └───→ 终止态
```

#### 1. 运行态（Running）
- 占有CPU，正在CPU上运行

#### 2. 就绪态（Ready）
- 已经具备运行条件，但CPU被其他进程占用，暂时等待
- 一旦获得CPU，立即执行

#### 3. 阻塞态（Blocked/Waiting）
- 等待某个事件的发生（如I/O完成）
- 无法立即执行，除非等待的事件发生

### 进程状态转换

```
运行态 ──→ 就绪态（时间片用完/被更高优先级抢占）
运行态 ──→ 阻塞态（等待I/O/信号量/消息）

阻塞态 ──→ 就绪态（等待的事件发生）

就绪态 ──→ 运行态（调度程序分配CPU）

运行态 ──→ 终止态（exit()系统调用）
```

### 其他进程状态

#### 创建态（New）
- 进程正在被创建，操作系统分配PCB等资源

#### 终止态（Terminated）
- 进程执行结束，释放资源

#### 挂起态（Suspended）
- 进程被换出到外存
- **挂起就绪态**：进程在外存，就绪，等待调入内存
- **挂起阻塞态**：进程在外存，阻塞，等待事件

## 进程控制

### 原语（Primitive）
用于进程控制的底层操作，具有**原子性**（不可中断）

### 进程控制原语

#### 1. 进程创建（Create）
```c
pid = fork();  // 创建子进程
```
- 申请空白PCB
- 为进程分配资源
- 初始化PCB
- 将进程插入就绪队列

#### 2. 进程终止（Terminate）
```c
exit(0);  // 进程终止
```
- 从PCB集合中找到终止进程的PCB
- 释放进程占有的资源
- 将PCB从所在队列中移除

#### 3. 进程阻塞（Block）
```c
wait();  // 阻塞进程
```
- 找到要阻塞进程的PCB
- 将进程从就绪队列移出
- 改变进程状态为阻塞态
- 将PCB插入等待队列

#### 4. 进程唤醒（Wake Up）
```c
signal();  // 唤醒进程
```
- 在事件等待队列中找到PCB
- 将PCB从等待队列移除
- 将进程状态改为就绪态
- 将PCB插入就绪队列

#### 5. 进程切换（Switch）
- 保存当前进程的CPU现场到PCB
- 从就绪队列选择新进程
- 恢复新进程的CPU现场

## 进程调度

### 调度的层次

#### 1. 高级调度（作业调度）
- 从外存调入进程到内存
- 发生频率最低

#### 2. 中级调度（内存调度）
- 将内存中的进程换出到外存（挂起）
- 内存紧张时进行
- 发生频率介于高级和低级之间

#### 3. 低级调度（进程调度）
- 从就绪队列选择进程，分配CPU
- 发生频率最高

### 进程的挂起与激活

#### 挂起原语（Suspend）
- 根据请求者ID和进程ID找到PCB
- 检查活动状态，改为静止状态
- 根据状态移入相应队列

#### 激活原语（Active）
- 根据请求者ID和进程ID找到PCB
- 检查静止状态，改为活动状态
- 根据状态移入相应队列

## 线程

### 为什么引入线程？
- 进程是资源分配的基本单位
- 传统进程调度开销大
- 并发粒度不够细

### 什么是线程？
线程是CPU调度和分派的基本单位

### 线程与进程对比

| 对比维度 | 进程 | 线程 |
|---------|------|------|
| **资源分配** | ✅ 拥有资源 | ❌ 不拥有资源（共享进程资源） |
| **调度单位** | ❌ 资源分配单位 | ✅ CPU调度单位 |
| **并发性** | 进程间并发 | 线程间并发 |
| **开销** | 创建、切换开销大 | 创建、切换开销小 |
| **通信** | 进程间通信（IPC） | 线程间直接读写共享数据 |

### 线程的实现方式

#### 1. 用户级线程（ULT）
- 线程管理工作由用户态完成
- 内核不知道线程的存在
- 优点：切换快，开销小
- 缺点：一个线程阻塞会导致整个进程阻塞

#### 2. 内核级线程（KLT）
- 线程管理工作由内核完成
- 优点：并行性好，一个线程阻塞不影响其他线程
- 缺点：切换开销大

#### 3. 组合方式
- 线程创建在用户空间
- 调度和同步在内核
- 大多数现代操作系统采用

## 进程同步与通信

### 进程同步的基本概念

#### 临界资源
- 一次只允许一个进程访问的资源

#### 临界区
- 访问临界资源的代码段

#### 同步
- 多个进程因一定合作关系，需要协调先后顺序

#### 互斥
- 多个进程因争抢临界资源，不能同时进入临界区

### 实现进程互斥的方法

#### 1. 软件方法
- **单标志法** — 存在空闲让进问题
- **双标志法** — 存在忙等问题
- **双标志后检查法** — 存在饥饿问题
- **Peterson算法** — 解决以上问题

#### 2. 硬件方法

##### 中断屏蔽
- 关中断 → 进入临界区 → 开中断
- 优点：简单高效
- 缺点：不适合多CPU

##### TestAndSet指令（TS指令）
```c
bool TestAndSet(bool *lock) {
    bool old = *lock;
    *lock = true;
    return old;
}
```

##### Swap指令（XCHG指令）
```c
void Swap(bool *lock, bool *old) {
    bool temp = *lock;
    *lock = *old;
    *old = temp;
}
```

#### 3. 信号量机制（Semaphore）

### 信号量机制

#### 整型信号量
- 用一个整数S表示资源数

#### 记录型信号量（常用）
```c
typedef struct {
    int value;           // 资源数
    struct process *list; // 等待队列
} semaphore;

// P操作（wait）：申请资源
void P(semaphore S) {
    S.value--;
    if (S.value < 0) {
        block(S.list);  // 当前进程进入等待队列
    }
}

// V操作（signal）：释放资源
void V(semaphore S) {
    S.value++;
    if (S.value <= 0) {
        wakeup(S.list); // 唤醒等待队列中的进程
    }
}
```

#### 利用信号量实现进程互斥
```c
semaphore mutex = 1; // 互斥信号量

P(mutex);  // 进入临界区前
// 临界区代码
V(mutex);  // 离开临界区后
```

### 经典同步问题

#### 1. 生产者-消费者问题
- 生产者生产产品放入缓冲区
- 消费者从缓冲区取产品消费

```c
semaphore mutex = 1;    // 互斥信号量
semaphore empty = n;    // 空缓冲区数量
semaphore full = 0;     // 满缓冲区数量

// 生产者进程
while (true) {
    // 生产一个产品
    P(empty);    // 空缓冲区减1
    P(mutex);    // 进入临界区
    // 将产品放入缓冲区
    V(mutex);    // 离开临界区
    V(full);     // 满缓冲区加1
}

// 消费者进程
while (true) {
    P(full);     // 满缓冲区减1
    P(mutex);    // 进入临界区
    // 从缓冲区取产品
    V(mutex);    // 离开临界区
    V(empty);    // 空缓冲区加1
    // 消费产品
}
```

#### 2. 读者-写者问题
- 多个读者可以同时读
- 只有一个写者可以写
- 读写互斥，写写互斥

#### 3. 哲学家进餐问题
- 5个哲学家，5根筷子
- 哲学家左右各拿一根筷子才能吃饭

## 进程通信

### 进程通信的类型

#### 1. 共享存储
- 多个进程共享一块内存空间
- 需要通过信号量或互斥实现同步

#### 2. 消息传递
- 进程通过发送/接收消息通信
- 分为**直接通信**和**间接通信**

#### 3. 管道通信
- 半双工通信，只能单向传输
- 采用先进先出方式

#### 4. 套接字（Socket）
- 用于不同主机之间的进程通信

## 考试要点

### 高频考点
1. **进程与线程的区别** — 资源分配、调度单位、开销
2. **进程状态转换** — 三态模型、五态模型
3. **信号量机制** — P/V操作、实现互斥和同步
4. **经典同步问题** — 生产者-消费者、读者-写者

### 常见陷阱
- 信号量的P/V操作必须成对出现
- 生产者-消费者问题需要三个信号量（mutex、empty、full）
- 线程共享进程的资源

- 2011年408真题：进程状态转换
- 2014年408真题：信号量实现进程同步
