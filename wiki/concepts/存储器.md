# 存储器（Memory）

## 存储器的层次结构

### 存储器的分类

#### 按存储介质分类
- **半导体存储器**：主存、Cache（易失性）
- **磁表面存储器**：磁盘、磁带（非易失性）
- **光存储器**：光盘（非易失性）

#### 按存取方式分类
- **随机存储器（RAM）**：可按地址随机访问
- **只读存储器（ROM）**：只能读不能写
- **串行访问存储器**：磁盘、磁带

#### 按在计算机中的作用分类
- **主存储器（Main Memory）**：主存、内存
- **辅助存储器（Auxiliary Memory）**：辅存、外存
- **高速缓冲存储器（Cache）**：介于CPU和主存之间

### 存储器的层次结构

```
寄存器 → Cache → 主存 → 辅存
```

**金字塔结构（从上到下）：**
- 速度越来越慢
- 容量越来越大
- 每位价格越来越低

**局部性原理：**
- **时间局部性**：最近访问过的内容，不久后还会被访问
- **空间局部性**：访问某个存储单元后，其附近的存储单元也会被访问

## 主存储器

### 主存储器的基本概念

#### 主存的组成
- **存储体**：存储信息的单元集合
- **地址寄存器（MAR）**：存放访问的存储单元地址
- **数据寄存器（MDR）**：存放从存储体读/写的数据

#### 主存的主要技术指标

##### 存储容量
- 按字编址：存储单元数 = 容量 / 字长
- 按字节编址：存储单元数 = 容量（字节数）

##### 存储速度
- **存取时间**：从启动到完成读写所需时间
- **存取周期**：连续启动两次独立操作所需时间
- **存取频率**：1 / 存取周期

### SRAM和DRAM

| 对比维度 | SRAM | DRAM |
|---------|------|------|
| **存储原理** | 触发器 | 电容 |
| **集成度** | 低 | 高 |
| **速度** | 快 | 慢 |
| **功耗** | 高 | 低 |
| **价格** | 高 | 低 |
| **是否需要刷新** | ❌ 不需要 | ✅ 需要刷新 |
| **应用** | Cache | 主存 |

### ROM（只读存储器）

| 类型 | 特点 | 应用 |
|------|------|------|
| **MROM** | 掩膜ROM，出厂时写入 | 固化程序 |
| **PROM** | 可编程ROM，一次写入 | 小批量生产 |
| **EPROM** | 可擦除可编程，紫外线擦除 | 开发阶段 |
| **EEPROM** | 电可擦除可编程 | 存储参数 |
| **Flash** | 闪存，块擦除 | U盘、SSD |

## Cache（高速缓存）

### Cache的基本原理

#### 为什么需要Cache？
- CPU速度远快于主存（速度差100倍以上）
- Cache速度快、容量小，价格高
- 利用**局部性原理**，将热点数据存放在Cache中

#### Cache的工作流程
```
CPU访问地址 → 检查Cache是否命中
  ├── 命中（Hit）：直接从Cache读取数据
  └── 未命中（Miss）：从主存读取，并将该块数据加载到Cache
```

### Cache和主存的映射方式

#### 1. 直接映射（Direct Mapping）
- 每个主存块只能映射到Cache中的一个特定位置
- 优点：实现简单，查找速度快
- 缺点：冲突概率高

```
主存块号 mod Cache块数 = Cache块号

地址格式：
┌──────┬──────┬──────┐
│ 标记 │ 块内地址 │ 块号 │
└──────┴──────┴──────┘
```

#### 2. 全相联映射（Fully Associative Mapping）
- 主存块可以映射到Cache中的任意位置
- 优点：冲突概率低，Cache利用率高
- 缺点：硬件复杂，查找慢

```
地址格式：
┌──────┬──────┐
│ 标记 │ 块内地址 │
└──────┴──────┘
```

#### 3. 组相联映射（Set Associative Mapping）
- 将Cache分成若干组，每个组包含若干块
- 主存块先映射到特定组，再映射到组内任意块
- 实际使用最广泛

```
地址格式：
┌──────┬──────┬──────┐
│ 标记 │ 组号 │ 块内地址 │
└──────┴──────┴──────┘
```

### 命中率与平均访问时间

#### 命中率（Hit Rate）
$$命中率 = \frac{Cache命中次数}{总访问次数}$$

#### 未命中率（Miss Rate）
$$未命中率 = 1 - 命中率$$

#### 平均访问时间
$$t_a = 命中率 \times t_c + 未命中率 \times t_m$$

- $t_c$：Cache访问时间
- $t_m$：主存访问时间

### Cache的写策略

#### 写直达（Write Through）
- 写Cache的同时，立即写主存
- 优点：Cache和主存一致性好
- 缺点：写操作时间长

#### 写回（Write Back）
- 写Cache时，不立即写主存
- 只有当Cache块被替换时，才写回主存
- 优点：写操作快
- 缺点：需要设置**修改位（dirty bit）**

### 替换算法

#### 1. 先进先出算法（FIFO）
- 优先替换最早进入Cache的块
- 算法简单，但不符合局部性原理

#### 2. 最近最少使用算法（LRU）
- 优先替换最长时间未被访问的块
- 符合局部性原理，效果最好
- 实现复杂（需要计数器或栈）

#### 3. 随机算法（Random）
- 随机选择替换的块
- 实现简单，但效果不稳定

## 虚拟存储器

### 虚拟存储器的基本概念

#### 定义
将主存和辅存统一编址，构成一个大的存储空间

#### 特点
- **虚拟存储空间远大于物理主存**
- 程序员可以使用比主存大得多的虚拟地址空间
- 由操作系统和硬件配合实现

#### 实现方式
- 基于**局部性原理**
- 暂时不用的部分放在辅存
- 需要的部分再调入主存

### 页式虚拟存储器

#### 基本原理
- 将虚拟空间和主存空间都划分为大小相同的页
- 通过**页表**实现虚拟地址到物理地址的转换

#### 页表
- 每个进程都有一个页表
- 页表的每个表项（PTE）包含：
  - **有效位**：该页是否在主存中
  - **页框号**：主存中的物理页号
  - **访问位**：记录是否被访问过（用于置换算法）
  - **修改位**：记录是否被修改过（用于写回策略）

#### 地址转换
```
虚拟地址 → 页号 + 页内偏移
  ↓
查页表 → 页框号 + 页内偏移
  ↓
物理地址 → 物理页号 + 页内偏移
```

#### 快表（TLB，Translation Lookaside Buffer）
- 页表的一种缓存，存放最近访问的页表项
- 命中时，直接得到页框号，无需访问内存中的页表
- 速度快但容量小

### 段式虚拟存储器

#### 基本原理
- 按程序逻辑模块分段
- 每个段有自己的段名和段长
- 通过**段表**实现地址转换

#### 段表项
- 段号
- 段长
- 段基址（段在主存中的起始地址）
- 有效位

#### 页式 vs 段式

| 对比维度 | 页式 | 段式 |
|---------|------|------|
| **划分依据** | 物理页大小（等长） | 程序的逻辑模块（不等长） |
| **地址空间** | 一维 | 二维 |
| **便于共享** | 差 | 好 |
| **便于保护** | 差 | 好 |
| **碎片问题** | 内部碎片 | 外部碎片 |

### 段页式虚拟存储器

- 结合段式和页式的优点
- 先分段，每段再分页
- 地址结构：段号 + 页号 + 页内偏移

### 页面置换算法

#### 1. 先进先出算法（FIFO）
- 先进入主存的页面先被替换
- 实现简单，但效果较差（可能出现Belady异常）

#### 2. 最近最少使用算法（LRU）
- 最近最久未被使用的页面被替换
- 最接近OPT算法，效果最好
- 需要硬件支持（计数器或栈）

#### 3. 最不经常使用算法（LFU）
- 访问次数最少的页面被替换
- 需要维护计数器

#### 4. 时钟置换算法（Clock）
- LRU的近似算法
- 用访问位和修改位来判断

## 考试要点

### 高频考点
1. **Cache的工作原理** — 命中率计算
2. **Cache的三种映射方式** — 直接映射、全相联、组相联
3. **页面置换算法** — FIFO、LRU、Clock
4. **虚拟存储器的实现** — 页式、段式、段页式
5. **TLB** — 快表的作用

### 常见陷阱
- Cache未命中时，会加载整个块（不只是一个字）
- LRU算法不会出现Belady异常
- FIFO算法可能出现Belady异常（分配页框越多，缺页率反而越高）
